# 💡 [BDS 플랫폼] 기술적 안정성 컴포넌트 라이브러리 및 UX 가이드라인 V1

## 📌 개요: '안정성 프리미엄'의 시각화 목표
*   **핵심 메시지:** "우리는 실패를 예측하고 대비합니다. 따라서 당신은 안심할 수 있습니다." (Confidence through Resilience)
*   **적용 범위:** 결제 흐름, 데이터 전송/검증 과정, 중요 정보 조회 페이지 등 기술적 상호작용이 발생하는 모든 UX 지점.
*   **사용 원칙:** Red 계열의 '위험' 경고는 최소화하고, **Deep Blue (신뢰)**와 **Warning Orange (주의)**를 핵심 액션 플로우에 사용한다.

---

## 🟢 Component 1: 성공 상태 컴포넌트 (Success State)
**[지표 연동]:** Success Rate 95% 달성 시 노출되는 최종 확정 화면 및 완료 메시지.

| 요소 | 규격/코드 | 디자인 원칙 | 예시 흐름 |
| :--- | :--- | :--- | :--- |
| **아이콘** | SVG: 체크마크 (✓) + 보호막 아이콘 | 딥 블루 계열의 입체적인 '보호' 느낌 강조. | 성공 메시지 배경 전체에 은은한 파란색 그라데이션 적용. |
| **메시지 텍스트** | H2, Bold, Deep Blue (`#003366`) | 단순 "성공했습니다" 대신, "안정적으로 거래가 완료되었습니다." 등 신뢰 증명 문구 사용. | *[예시]* "데이터 전송이 안정적으로 완료되어 고객님의 수익 인증에 성공하였습니다." |
| **보조 정보** | Badge/Tag | **'95% Success Guarantee'** 등의 기술적 지표를 시각적인 배지로 강조 (회색 배경, Deep Blue 텍스트). | 이 섹션을 통해 플랫폼의 '기술력'을 간접적으로 홍보. |

## 🟠 Component 2: 경고 및 실패 컴포넌트 (Failure/Warning State)
**[지표 연동]:** 시스템 오류, 입력값 오류 등 사용자 액션이 필요한 경우 노출되는 화면. **절대 Red를 사용하지 않는다.**

| 요소 | 규격/코드 | 디자인 원칙 | 예시 흐름 |
| :--- | :--- | :--- | :--- |
| **경고 박스** | 배경: Light Orange (`#FFF8E1`), 테두리: Warning Orange (`#FFB300`) | 경고가 '위험'이 아닌 '주의 및 개선 필요'임을 명시. | 오류 메시지 상단에 큰 주황색 삼각형 아이콘과 함께 "잠시 확인해 주세요" 문구 배치. |
| **오류 설명** | Body, Deep Blue (`#003366`) | 단순 에러 코드 나열 금지. 사용자 관점에서 *무엇을* 잘못했는지, 그리고 *왜* 문제가 발생했는지를 친절하게 설명한다. | *[나쁜 예]* "Error Code 404." / *[좋은 예]* "결제 시스템과 통신이 일시적으로 원활하지 않습니다. 잠시 후 다시 시도해 주세요." |
| **다음 액션 유도** | Button, Warning Orange (`#FFB300`) | 버튼 자체를 주황색으로 처리하여 '여기서 할 수 있는 다음 행동'을 명확히 제시한다. (예: [재시도하기], [직접 확인하기]) |

## 🔄 Component 3: 복구 및 재시도 컴포넌트 (Recovery/Retry Loop)
**[지표 연동]:** Retry Rate 80%를 시각화하는 애니메이션과 로직.

| 요소 | 규격/코드 | 디자인 원칙 | 예시 흐름 |
| :--- | :--- | :--- | :--- |
| **로딩 상태** | Skeleton Loader + Progress Bar | 단순 회전 로딩을 지양하고, '진행 중'이라는 느낌을 주며 신뢰도를 유지한다. Deep Blue 색상 기반의 부드러운 진행 바 사용. | "데이터를 안정적으로 불러오는 중..." 문구와 함께 진행률 표시. |
| **재시도 버튼** | Button, Warning Orange (`#FFB300`) | 재시도는 '실패'가 아니라 '시스템이 자동으로 복구를 시도하는 적극적 행동'임을 강조한다. 텍스트: "다시 연결하기" 또는 "80% 확률로 성공합니다." (재치 있는 문구 사용 가능). |
| **애니메이션** | Smooth Transition | 재시도를 누르는 순간, 짧은 로딩 애니메이션과 함께 '시스템 점검 중'이라는 메시지를 보여주어 기술적 노력을 시각화한다. |

---

## 🖥️ 적용 가이드라인 요약 (Quick Check)
1.  **색상 팔레트:** Deep Blue (`#003366`), Warning Orange (`#FFB300`), Light Backgrounds. **(Red는 절대 사용 금지)**
2.  **폰트 계층 구조:** H1/H2 (굵고 신뢰감 있는 산세리프체) > 본문 (가독성 높은 세리프 또는 산세리프).
3.  **톤 앤 매너:** 전문적이고, 차분하며, 해결책을 제시하는 **'안정적인 컨설턴트'**의 어조를 유지한다.
