# 🚨 BDS 마케팅 에셋 최종 통합 검증 체크리스트 (V1.0)

## 🎯 목표: Pain(문제 인식) → Solution(해결책 제시)의 감정적 전환 일관성 확보
**확인 범위:** 랜딩 페이지, 인스타그램 콘텐츠, 오류/피드백 컴포넌트 전체
**핵심 원칙:** 모든 자산은 '소상공인의 불안 해소'에 초점을 맞추고, 시각적으로 **Pain Red $\rightarrow$ Solution Green** 그라데이션을 유지해야 함.

---

### 📝 I. [메시지/카피] 일관성 검증 (Writer & Designer 공동 점검)

| 항목 | 세부 내용 | Pass/Fail | 비고 및 액션 아이템 |
| :--- | :--- | :--- | :--- |
| **[A] 문제 정의의 공감성** | 소상공인이 겪는 어려움(불안, 막막함)이 구체적인 데이터와 사례로 연결되었는가? (추측 방지) | [ ] | 예: "열정 부족" $\rightarrow$ "데이터 로직 부재"로 전환 명확화. |
| **[B] Solution의 명료성** | BDS 플랫폼이 제공하는 해결책(핵심 가치)이 단 하나의 강력한 메시지로 압축되었는가? (복잡성 제거) | [ ] | 핵심 문구: "품질 검증된 중간 유통 허브" 강조. |
| **[C] CTA의 행동 유도력** | 모든 콘텐츠 마지막에 다음 단계로 나아갈 명확하고 간결한 행동 촉구가 존재하는가? (저장/클릭 유도) | [ ] | 예: "지금 BDS에서 나의 데이터 진단받기" |

### 🎨 II. [비주얼 디자인 시스템] 일관성 검증 (Designer 주도 점검)

**핵심 토큰:**
*   **Pain Red:** `#D94E4E` (위험, 경고, 불안정 - *문제 제기 시 사용*)
*   **Transition:** `#FFC107` (주의, 고민, 전환점 - *데이터 진단/개선 필요성 제시 시 사용*)
*   **Solution Green:** `#388E3C` (안전, 성장, 성공 - *BDS 솔루션/성과 예측 시 사용*)

| 에셋 종류 | 점검 포인트 | 적용 여부 | 디테일 확인 사항 및 수정 지침 |
| :--- | :--- | :--- | :--- |
| **[A] 랜딩 페이지 (LP)** | Pain $\rightarrow$ Solution 흐름에 따른 컬러 전환이 자연스러운가? | [ ] | 1. *문제 제기 섹션:* 적색 계열의 아이콘, 경고 문구 사용. / 2. *솔루션 섹션:* 녹색/청록색 기반의 성공 그래프, 체크리스트 활용. |
| **[B] 오류 카드 컴포넌트** | 에러 메시지 자체도 '학습 기회'로 전환되는가? (Fail $\rightarrow$ Learn) | [ ] | 1. *오류 발생 시:* Red Tone으로 경고. / 2. *해결책 제시:* Green Tone의 명확한 재시도 버튼 및 가이드 제공. (`ErrorCardComponent_DesignSystem_v3` 기반 확인). |
| **[C] 인스타그램 콘텐츠** | 이미지/영상 내 모든 그래픽 요소가 Red $\rightarrow$ Green 그라데이션 원칙을 따르는가? | [ ] | 1. *Before:* 어둡고 흐릿한 느낌 (Red Tone) / 2. *After:* 밝고 구조화된 느낌 (Green Tone). 폰트 크기/배치도 일관되어야 함. |
