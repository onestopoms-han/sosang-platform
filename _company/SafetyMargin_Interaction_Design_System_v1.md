# 🎨 Safety Margin Interaction Design System 가이드라인 v1.0 (Designer 초안)\n\n## 🎯 목표\n'안전마진(Safety Margin)' 지표가 사용자에게 가장 직관적이고 안정적으로 전달되도록, 데이터 상태 변화에 따른 시각/애니메이션 스펙을 확정한다.\n\n## 🔑 핵심 원칙\n1. **직관성:** 색상과 애니메이션의 변화는 '경고'와 '안정화'라는 감정적 메시지와 일치해야 한다.\n2. **최소한의 마찰:** 상태 변화에 따른 과도한 애니메이션은 피하고, 데이터가 바뀌었음을 명확하게 인지시키는 미니멀 인터랙션(Micro-interaction)을 사용한다.\n3. **기술 최우선:** 모든 스펙은 코다리 개발팀이 정의한 API 호출 주기와 UI/UX의 성능 제약 조건을 최우선으로 따른다.\n\n## 📊 Safety Margin 상태별 디자인 및 애니메이션 스펙\n\n### 1. Critical State (위험 단계: Red/Warning)\n*   **Trigger:** `SafetyScore < Threshold_C` (예: 지난 7일간 데이터 감소율 > X%)\n*   **시각적 변화:** 배경 전체에 *Deep Blue*가 아닌, 경고를 의미하는 **Orange 계열의 그라데이션 오버레이** 적용.\n*   **애니메이션 스펙:**\n    *   **진입 (Entering):** Score 수치가 임계점을 넘어서는 순간, 해당 영역이 `Pulse` 애니메이션(1초 주기로 미세하게 깜빡임)을 실행하며 사용자 주의를 집중시킨다. \n    *   **유지 (Maintaining):** 정적인 경고 상태 유지.\n*   **텍스트/피드백:** "🔴 위험! 안전마진이 임계치 이하입니다. 즉각적인 활동 계획이 필요합니다."\n\n### 2. Caution State (주의 단계: Yellow)\n*   **Trigger:** `Threshold_L <= SafetyScore < Threshold_C`\n*   **시각적 변화:** 컴포넌트 경계선에 **Soft Orange 테두리** 적용 및 배경색을 미묘하게 어둡게 조정.\n*   **애니메이션 스펙:**\n    *   **진입 (Entering):** Score 수치가 임계 범위로 진입하는 순간, 수치(`Score Value`)가 `Smooth Bounce` 효과(0.3초)를 통해 부드럽게 자리 잡는다.\n    *   **유지 (Maintaining):** 정적인 주의 상태 유지.\n*   **텍스트/피드백:** "🟡 주의: 안전마진이 감소 추세입니다. 활동을 재점검하고 보완 계획을 세우세요."\n\n### 3. Optimal State (최적 단계: Deep Blue)\n*   **Trigger:** `SafetyScore >= Threshold_H` (예: 지난 7일간 데이터 증가율 > Y%)\n*   **시각적 변화:** 기본 브랜드 컬러인 **Deep Blue 계열의 배경 및 테두리** 유지.\n*   **애니메이션 스펙:**\n    *   **진입 (Entering):** Score 수치가 최적 범위로 돌아오는 순간, `Fade-in` 애니메이션(0.5초)과 함께 긍정적인 느낌을 주는 **Micro Sparkle 효과**(작은 입자 반짝임)가 한 번 발생한다.\n    *   **유지 (Maintaining):** 가장 안정되고 신뢰감을 주는 상태 유지.\n*   **텍스트/피드백:** "🔵 안전마진 확보! 현재 활동이 매우 안정적이며, 지속적인 성장이 예상됩니다."\n\n### 🔄 Transition Interaction 스펙 (가장 중요)\n\n| FROM State | TO State | Trigger Event | Animation Sequence | Duration | Developer Note (코다리 확인 필요) |\n| :---: | :---: | :---: | :---: | :---: | :---: |\n| Optimal $\to$ Critical | **데이터 급락** | API Call Cycle Completion (Error/Drop Data) | 1. Deep Blue $\to$ Orange Gradient Overlay (0.3s). 2. Score Value가 Bounce하며 빨간색으로 변환(0.5s). 3. Pulse 애니메이션 시작.\n| Caution $\to$ Optimal | **데이터 회복** | API Call Cycle Completion (Recovery Data) | 1. Soft Orange 테두리 제거(0.2s). 2. 배경이 Deep Blue로 `Fade-in`(0.4s). 3. Micro Sparkle 효과 실행.\n| Critical $\to$ Caution | **활동 시작/개선** | User Action (예: 콘텐츠 업로드) | 경고 오버레이가 점진적으로 투명해지며(Opacity Gradient), 색상이 Orange에서 Yellow로 부드럽게 전환된다(Color Interpolation). | 0.7s | 사용자 액션 발생 시, 시스템이 즉시 스코어 업데이트를 처리할 수 있는지 확인 필요. |\n\n--- \n**[다음 행동]**: 코다리에게 이 `v1.md` 초안을 제공하고, 특히 **Transition Interaction 스펙**의 기술적 구현 가능성과 데이터 트리거 지점(API Hook)에 대한 검토를 요청한다.\n
