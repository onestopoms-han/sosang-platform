## II. ⚙️ Trust Widget 상세 사양
Trust Widget은 '안전마진(Safety Margin)'을 시각적으로 표현하는 핵심 컴포넌트입니다.

### 1. 기본 구조 및 좌표 명세

    *   *좌표:* X: 80px ~ 280px / Y: 40px ~ 120px
    *   *높이:* 데이터 바인딩에 따라 동적 변경 (Min Height: 30px, Max Height: 70px)
*   **Key Element 2: Risk Label:** 점수 아래, 현재 리스크 등급(A/B/C)을 텍스트와 아이콘으로 표시.

본 설계안은 리스크 등급 (Risk Level: A/B/C) 을 플랫폼의 핵심 상태 (State) 로 삼아, 각 상태에 따라 최적화된 수익 모델과 기능 우선순위를 정의합니다. 이를 통해 **Trust Widget** 과 **PainGauge**가 단순히 UI 컴포넌트가 아닌, 사용자의 심리적 리스크를 관리하여 결제를 유도하는 '비즈니스 엔진'으로 작동하도록 설계됩니다.

## 1️⃣ State-based Architecture: 리스크 등급별 구조 및 수익 모델
플랫폼의 상태는 **사용자가 인지할 수 있는 리스크 (Risk Perception)**에 따라 정의됩니다. 아래 표는 각 상태별 기능 우선순위와 수익 모델을 명시합니다.

| 리스크 상태 | 정의 | 주요 사용자 감성 | **기능 우선순위** | **수익 모델 (Revenue Model)** |
| :--- | :--- | :--- | :--- | :--- |
| **🟢 A: 안전마진 상태**<br>(Trust Maximized) | 공급망 검증 완료, 실시간 거래 데이터 공유 가능, 정산 100% 보장. | "완전 신뢰", "편의성" | - PainGauge: 0 (최소 노출)<br>- Trust Widget: "인증됨" 표시 유지<br>- 결제 버튼: 즉시 활성화 | **정기 구독료 + 프리미엄 수수료**<br>• 월 5~10 만 원 (플랫폼 유지비)<br>• 거래 건당 2% (수수료 waived for 3 months) |
| **🟡 B: 검증 중 상태**<br>(Risk Mitigation) | 기본 정보 제공, 일부 데이터 검증 중, 신용 점수 기반 평가. | "신중", "호기심" | - PainGauge: '검증 진행' 단계 표시<br>- Trust Widget: 출처 링크 노출<br>- 결제 버튼: 조건부 활성화 (예: 30 일 무료 체험) | ** Freemium 전환 + 데이터 판매**<br>• 무료 체험 기간 후 월 2~5 만 원 (기본 플랜)<br>• 공급망 검증 데이터 리포트 유료 구독<br>• PainGauge 기반 리스크 분석 툴 사용료 |
| **🔴 C: 고위험 상태**<br>(Risk Avoidance) | 정보 부족, 부정 거래 의심, 정산 보장 불가. | "불안", "피해" | - PainGauge: '주의' 경고 표시<br>- Trust Widget: 출처 링크 필수 클릭<br>- 결제 버튼: 지연 또는 차단 | **분리된 수수료 + 보험 파트너십**<br>• 거래 건당 5~10% (고위험 상환 수수료)<br>• 보험 가입 유도 수수료 (보험사 제휴)<br>• PainGauge '위험 분석' 리포트 유료 판매 |

## 2️⃣ Trust Widget & PainGauge 의 비즈니스 목표 연결성
각 위젯은 단순한 UI 요소를 넘어, **사용자의 심리적 리스크를 관리하여 결정을 유도**하는 역할을 수행합니다. 아래는 두 위젯이 각 상태 (A/B/C) 에서 어떻게 작동하고, 그것이 곧 수익으로 이어지는지 설명합니다.

### 🔵 Trust Widget: 신뢰 기반의 결제 전환
- **기능**: 플랫폼이 제공하는 인증 정보를 시각적으로 표시하며, 출처 링크를 통해 정밀성을 증명합니다.
- **비즈니스 연결**:
  - **상태 A**: 사용자의 "완전 신뢰"감을 고취시켜, 즉시 결제 행동을 유도합니다. 이는 높은 거래 건수 (High Volume) 와 낮은 수수료 수익을 가져옵니다.
  - **상태 B**: 출처 링크 클릭 시 검증 데이터를 제공하여, 사용자가 '안전하게' 플랫폼을 이용하도록 설득합니다. 이는 Freemium 플랜의 전환율을 높입니다.
  - **상태 C**: "주의" 경고와 함께 출처 링크를 강제 노출시켜, 사용자가 위험을 인지하고도 플랫폼을 신뢰하게 만듭니다. 이는 고위험 수수료 모델 (High Margin) 을 실현합니다.

### 🟢 PainGauge: 리스크 기반의 행동 유도
- **기능**: 사용자의 리스크 수준 (재정적 불안, 공급망 불안 등) 을 실시간으로 시각화하여, 적절한 솔루션을 제안합니다.
- **비즈니스 연결**:
  - **상태 A**: PainGauge 은 "0" 상태로 유지되어, 사용자에게 불필요한 심리적 부하를 줄입니다. 이는 장기적인 플랫폼 충성도를 높여 구독료 수익을 안정화시킵니다.
  - **상태 B**: '검증 진행' 단계가 표시되어, 사용자가 검증 데이터를 유료 리포트로 구매하도록 유도합니다. 이는 데이터 기반의 추가 수익원 (Data Revenue) 을 창출합니다.
  - **상태 C**: '주의' 경고가 표시되어, 사용자가 위험을 인지하고도 플랫폼이 제공하는 보험이나 분산 투자 솔루션을 선택하도록 안내합니다. 이는 고위험 수수료와 파트너십 수익을 가져옵니다.

### 2. 🗺️ 사용자 여정 지도 (User Journey Map & Flow)

소상공인 A가 플랫폼을 처음 접하여 최종적으로 거래를 완료하는 가상의 시나리오를 통해 흐름을 설계했습니다.

#### Step 1: 진입 단계 (Initial Landing / Profile View)
*   **사용자 상태:** 높은 불확실성, 리스크 인지 전.
*   **디자인 목표:** 사용자에게 *무언가 부족하다*는 느낌(Gap)을 주어 PainGauge를 노출시킨다.
*   **시각적 흐름:**
    1.  페이지 로딩 시 → **PainGauge (B State)**가 화면 중앙 상단에 '주의' 수준으로 배치된다.
    2.  PainGauge 하단에는 "이 정보는 BDS 플랫폼을 통해 검증될 필요가 있습니다"라는 문구와 함께 Trust Widget의 초기 버전(출처 미확인)이 배치된다.

#### Step 2: 문제 인지 단계 (Problem Identification)
*   **사용자 행동:** PainGauge를 클릭하거나, 신뢰도가 낮은 데이터를 발견한다.
*   **디자인 목표:** 단순 경고가 아닌, **왜 이 데이터가 불안정한지** 근거를 제시하여 사용자가 플랫폼의 가치를 이해하게 한다.
*   **시각적 흐름:**
    1.  PainGauge 클릭 $\rightarrow$ 모달(Modal) 팝업 또는 아코디언 확장 $\rightarrow$ 해당 리스크에 대한 **데이터 기반 설명 (Why)**이 펼쳐진다.
    2.  Trust Widget은 단순히 "출처 불명"을 보여주는 대신, "신뢰도 확보를 위해 [플랫폼 기능 X]가 필요합니다."라고 구체적인 해결책(Solution)과 함께 노출된다.

#### Step 3: 신뢰 구축 단계 (Validation & Trust Building)
*   **사용자 행동:** 플랫폼의 검증 과정(예: API 연동, 서류 업로드 등)에 자발적으로 참여하거나, 다른 성공 사례를 본다.
*   **디자인 목표:** 리스크가 감소하고 있음을 시각적/감성적으로 확신시킨다.
*   **시각적 흐름:**
    1.  PainGauge의 수치가 **🟡 B $\rightarrow$ 🟢 A**로 녹색 계열로 부드럽게 변한다 (애니메이션 필수).
    2.  Trust Widget이 "검증 완료" 상태로 전환되며, 그 과정에서 확보된 신뢰 근거(예: '공급망 인증서', '정산 보장 시스템')가 **Primary Blue** 박스로 강조되어 배치된다.

#### Step 4: 행동 유도 단계 (Conversion)
*   **사용자 상태:** 높은 확신, 구매 준비 완료.
*   **디자인 목표:** 주저할 틈 없이 명확하고 강력한 CTA를 제시한다.
*   **시각적 흐름:**
    1.  페이지 최하단 또는 사이드바의 CTA 버튼이 **Growth Green**으로 활성화되고, "안전마진 기반 구매하기"와 같은 구체적인 문구로 전환된다.

#### B. Trust Widget 컴포넌트 (The Credibility Engine)

#### C. CTA 버튼 (Action Button)
*   **State-Based 활성화:** CTA의 색상과 텍스트는 반드시 현재 PainGauge의 상태와 연동되어야 한다.
    *   (B State $\rightarrow$ A State 전환 전): `[임시] 무료 리포트 신청` (Blue/Grey, 제한적 액션)
    *   (A State 도달 후): `[최종] 안전마진으로 거래하기` (Green, 최대 활성 CTA)

### 📑 BDS 플랫폼 핵심 로직 요약 (CEO 브리핑용 3단계 플로우)

저희 플랫폼은 단순히 정보를 보여주는 것을 넘어, 소상공인이 느끼는 '불확실성'이라는 리스크를 관리하며 매출로 연결하는 심리적 구조를 가지고 있습니다.

#### 🚩 Step 1: 문제 포착 (Pain Detection & Doubt)
*   **목표:** 사용자에게 잠재된 위험(Risk Gap)을 인지시켜 관심을 유도합니다.
*   **핵심 컴포넌트:** **PainGauge** (위험 지수 게이지)
*   **작동 방식:** 플랫폼 진입 시, 데이터 기반으로 '주의' 또는 '경고' 등급의 리스크를 시각화하여 노출합니다. ("이 정보는 믿을 만한가요?"라는 질문을 던지게 합니다.)

#### 🔍 Step 2: 신뢰 구축 (Evidence Gathering & Trust Building)
*   **목표:** PainGauge가 제기한 의문점을 명확하고 객관적인 증거로 해소하여 불안감을 해소합니다.
*   **핵심 컴포넌트:** **Trust Widget** (신뢰 인증 위젯) + State-based Architecture
*   **작동 방식:** 리스크를 낮추는 과정(A $\rightarrow$ B $\rightarrow$ C 상태 전환)을 거치며, '출처 링크'와 '검증된 데이터'를 제시합니다. 사용자가 '이건 믿을 만하다'고 느끼도록 안내하는 단계입니다.

#### 🚀 Step 3: 행동 유도 (Conversion & Action)
*   **목표:** 충분한 신뢰가 쌓인 시점에 가장 빠르고 명확하게 다음 단계를 제시하여 결제를 유도합니다.
*   **핵심 컴포넌트:** **CTA (Call To Action)** 및 최적화된 수익 모델 연결
*   **작동 방식:** 모든 리스크 요소(PainGauge)가 Green 상태로 돌아오고, 신뢰(Trust Widget)가 확립되면, 가장 매력적인 플랜이나 서비스 구독 버튼을 '최종적으로' 노출합니다.

---

이 3단계 흐름을 개발팀의 최종 목표는 **State-based Design System**으로 구현하여 모든 페이지에서 일관성 있게 작동시키는 것입니다.
