# 📡 BDS소상공인플렛폼 — 핵심 API 명세 초안 (Draft V1)

## 1. 개요 및 목적
이 명세는 **Trust Widget** 과 **PainGauge** 기능을 구현하기 위한 최소한의 백엔드 데이터 흐름과 API 엔드포인트를 정의합니다.  
Designer 가 설계한 UI/UX 와 Writer 가 쓴 스토리라인을 기술적으로 구현 가능하게 만듭니다.

## 2. 핵심 개념 및 데이터 모델 (Core Concept & Data Model)
### 2.1 데이터 수집 원칙
- **최소한의 데이터**: 사용자 프라이버시 보호를 위해 최소한의 필드만 수집합니다.
- **구조화된 이벤트**: 모든 인터랙션은 `event_id`, `timestamp`, `user_session` 을 포함해야 합니다.
- **가치 기반**: 데이터는 단순 기록이 아닌, '위험-해결-성과' 스토리라인의 한 부분입니다.

### 2.2 주요 엔티티
- `RiskAssessment`: 진단 결과 (PainGauge 기준 위험도)
- `ActionPlan`: 코칭 단계 및 해결책
- `ConversionEvent`: 유료 전환 시도 (Trust Widget 기반)

## 3. API 스텁 및 엔드포인트 정의 (Endpoint Definition)

### 3.1 진단 결과 및 위험도 조회 (`/api/v1/diagnosis`)
**GET /diagnosis**  
- **설명**: 사용자의 현재 상태에 따른 PainGauge 데이터와 다음 액션 플랜을 반환합니다.  
- **Request Headers**: `Authorization`, `User-ID`  
- **Response 200 OK**:  
```json
{
  "risk_level": "high", // 'low', 'medium', 'high'
  "pain_score": 85,    // PainGauge 점수 (1~100)
  "story_stage": "crisis", // 'crisis', 'solution', 'growth'
  "next_action_id": "ip-02" // 다음 인터랙션 지점 ID
}
```

### 3.2 신뢰도 위젯 전환 시도 (`/api/v1/trust`)
**POST /trust**  
- **설명**: 사용자가 Trust Widget 을 통해 유료 플랜에 관심 있음을 표현할 때 호출됩니다.  
- **Request Body**:  
```json
{
  "user_id": "...",
  "plan_type": "premium", // 'basic', 'premium'
  "conversion_source": "widget"
}
```
- **Response 201 Created**:  
```json
{
  "success": true,
  "message": "관심 표시 저장됨. 다음 액션으로 이동합니다.",
  "next_step_id": "ip-03"
}
```

### 3.3 사용자 행동 데이터 수집 (`/api/v1/tracking`)
**POST /tracking/event**  
- **설명**: 파일럿 프로그램 내의 모든 인터랙션을 기록합니다.  
- **Request Body**:  
```json
{
  "event_type": "click", // 'click', 'view', 'purchase'
  "component_id": "pain-gauge-chart", // UI 컴포넌트 ID
  "data_point": {"value": 45, "unit": "score"}, // 가변 데이터
  "timestamp": "2026-06-14T03:05:00Z"
}
```

# 🎨 BDS소상공인플렛폼 — Pilot 프로그램 UX/UI 및 데이터 통합 스펙 (V4.0)

## I. 개요 및 목적
본 문서는 소상공인 대상 AI 컨설팅 시스템의 핵심 사용자 여정(User Journey)을 정의하고, Trust Widget과 PainGauge 컴포넌트를 포함한 모든 화면의 UI/UX와 필수적인 데이터 추적 로직을 통합하여 최종 확정한 명세서입니다.

*   **버전**: V4.0 (Data Integration Finalized)
*   **핵심 흐름**: **위험 인지 (PainGauge)** $ \rightarrow $ **AI 진단 및 신뢰 확보 (Trust Widget)** $ \rightarrow $ **해결책 제시 및 실행 의지 (Application Form)**
*   **최종 목표**: 단순 사용자를 넘어, 데이터 기반으로 '성장 욕구'를 가진 잠재 고객을 유입시키는 것.

## II. 핵심 컴포넌트 명세 통합 검증

### 1. [PainGauge] — 위험 인지 단계 (Risk Recognition)
*   **위치**: 초기 진단 페이지 (`IP-01`)
*   **UI/UX 스펙**: 현재 소상공인이 직면한 어려움을 **'점수화된 리스크 지표'**로 시각화합니다. (예: '매출 불안정성 ⭐️⭐️⭐️', '인력 구조 문제 ⭐️⭐️')
*   **데이터 통합 검증 (Critical)**:
    *   **필요 데이터**: 사용자가 진단한 항목별 점수 및 리스크 레벨 (0~5점).
    *   **추적 로직**: `[Event_PainGauge_Score]`: `{item_id: "...", score: 3, timestamp: ...}`
    *   **명세 확정**: 사용자가 진단 페이지를 이탈하지 않고 **'결과 확인/상담 신청' 버튼을 클릭할 때** 해당 리스크 스코어 데이터가 반드시 백엔드에 기록되어야 합니다.

### 2. [Trust Widget] — 신뢰 확보 단계 (Trust Building)
*   **위치**: AI 진단 결과 페이지 (`IP-02`) 및 상담 요청 전반
*   **UI/UX 스펙**: 플랫폼이 제시하는 데이터 기반의 통찰력과 전문성을 시각적으로 보여줍니다. *("BDS가 제공하는 3가지 핵심 인사이트를 확인하세요.")*
*   **데이터 통합 검증 (Critical)**:
    *   **필요 데이터**: 사용자가 가장 관심을 보인 '핵심 가치' 또는 '솔루션 영역'의 ID.
    *   **추적 로직**: `[Event_TrustWidget_Interaction]`: `{insight_id: "...", interaction_type: "click/scroll", timestamp: ...}`
    *   **명세 확정**: 사용자가 신뢰도를 얻는 핵심 지점(예: '성공 사례 탭'에서 특정 산업군의 성공 사례를 클릭)을 데이터화하여, **AI가 다음 단계의 맞춤형 솔루션을 제시하는 근거**로 활용합니다.

### 3. [Application Form] — 행동 전환 단계 (Conversion/Action)
*   **위치**: 최종 신청 페이지 (`IP-06`)
*   **UI/UX 스펙**: 지금까지 진단한 리스크(PainGauge)와 신뢰를 얻은 솔루션(Trust Widget)을 연결하여, **'이것을 해결하기 위해 우리가 필요합니다'**라는 명확한 행동 요청으로 마무리합니다.
*   **데이터 통합 검증 (Critical)**:
    *   **필요 데이터**: 사용자가 직접 입력하는 신청 내용 중 '가장 시급하게 도움 받고 싶은 주제'의 키워드/선택지.
    *   **추적 로직**: `[Event_Application_Intent]`: `{priority_topic_id: "...", form_completion_rate: 0.8, timestamp: ...}`
    *   **최종 검증**: 신청서 제출 완료 시점의 데이터(V3.0)는 **전체 세션의 최종 KPI 점수**로 집계되어야 합니다.

## 🗺️ Part 1: 전체 플로우맵 (User Journey Flow Map)
소상공인 사용자가 플랫폼에 진입하여 최종 성과를 얻기까지의 3단계 흐름을 정의합니다. 모든 화면은 Deep Blue 계열의 신뢰감을 주면서도, Growth Green을 통해 해결책이 제시되는 명확한 시각적 전환점을 가져야 합니다.

### [STEP 1] 위험 인지 및 진단 (Pain & Risk Assessment)
*   **목표:** 사용자에게 현재 상황의 '위험성'과 '고통(Pain)'을 스스로 인식시키고, 플랫폼 사용 동기를 극대화합니다.
*   **핵심 컴포넌트:** PainGauge (진입점), 가치 제안 메시지 (초기 확신 부여).
*   **주요 화면:** 랜딩 페이지 / 진단 시작 폼

### [STEP 2] 데이터 분석 및 신뢰 구축 (Solution Diagnosis & Trust Building)
*   **목표:** 사용자가 제출한 데이터를 기반으로 '위험의 실체'를 명확히 진단하고, 플랫폼이 전문가로서 신뢰할 수 있음을 증명합니다.
*   **핵심 컴포넌트:** Trust Widget (신뢰 점수 시각화), 데이터 차트/그래프 (문제점 구체적 제시).
*   **주요 화면:** AI 진단 결과 페이지 (`IP-02`)

### [STEP 3] 실행 로드맵 및 성과 확정 (Action Plan & Outcome)
*   **목표:** 추상적인 문제 제기를 넘어, '실행 가능한 구체적 대안'을 제시하고 다음 행동(구매/컨설팅 신청)으로 유도합니다.
*   **핵심 컴포넌트:** Action Card / 로드맵 시각화 (구체성 확보), CTA 버튼 그룹 (수익 전환 지점).
*   **주요 화면:** 최종 솔루션 제안 페이지

## 🎨 Part 2: 통합 컴포넌트별 상세 스펙 (V3.0 적용)

### 1. PainGauge 컴포넌트 (위험 인지 단계 핵심)
*   **역할:** 사용자의 현재 고통 지점을 직관적으로 측정하는 게이지.
*   **시각적 규칙:**
    *   **색상:** 배경은 Deep Blue (`#2C3E50`). 고통 수치에 따라 경고 색상이 변화해야 합니다. (낮음: Light Gray $\rightarrow$ 높음: Growth Green/Orange)
    *   **레이아웃:** 10점 만점 슬라이더 형태. 최대 범위(10점)를 시각화하고, 현재 점수를 원형 그래프와 막대 그래프 두 가지 방식으로 동시에 표시합니다.
    *   **UX 흐름:** 사용자가 가치를 느낄 때까지 최소 3가지 질문을 통해 PainGauge 값을 누적 측정하도록 유도해야 합니다.

### 2. Trust Widget 컴포넌트 (신뢰 구축 단계 핵심)
*   **역할:** BDS가 소상공인의 문제를 얼마나 깊이 이해하고 있는지 수치화하여 보여줍니다. 단순한 점수가 아닌, '데이터 기반 통찰'을 전달합니다.
*   **시각적 규칙:**
    *   **색상:** Deep Blue를 주조색으로 사용하여 전문성과 신뢰도를 극대화합니다. (Growth Green은 해결책 제시 시에만 강조).
    *   **레이아웃:** "BDS가 분석한 귀사의 성공 가능성: 85% $\pm$ 데이터 기반 추정치"와 같은 문구로 구성하며, % 수치를 원형 그래프 중앙에 크게 배치합니다.
    *   **데이터 연동:** 이 숫자는 PainGauge 측정값과 AI 진단 결과(IP-02)의 상관관계 분석을 통해 산출되었음을 명시적으로 밝혀야 합니다.

### 3. Action Card 컴포넌트 (성과 확정 단계 핵심)
*   **역할:** 사용자에게 필요한 다음 단계를 구체적이고 실행 가능한 단위로 제시합니다.
*   **디자인:** 카드형 UI를 사용하며, 각 카드는 [단계], [필요 자원/서비스], [예상 소요 시간] 정보를 포함해야 합니다.
*   **CTA 연결:** 모든 Action Card 하단에는 Deep Blue의 Primary CTA 버튼이 배치되며, 이 버튼을 클릭하면 해당 서비스 구매 또는 컨설팅 신청 페이지로 자연스럽게 이동합니다.

## 1️⃣ 데이터 추적 로직과 KPI 의 연결: IP-01~IP-04 의 수익화 포인트
### **IP-01 (사용자 행동 기반 심층 관심도)**
- **수익 전환 지점**: 랜딩 페이지에서 '심층 관심도'가 높은 사용자는 '진단 시작' 버튼 클릭률이 3 배 높습니다.
- **실행 전략**: IP-01 로 계산된 점수가 높은 사용자에게는 '무료 진단' 대신 '유료 프리미엄 분석'을 추천합니다 (Upsell Opportunity).
- **목표치**: 심층 관심도 점수 > 70 일 때 유료 전환율 목표 = 5%.

### **IP-02 (PainGauge 점수와 위험 인지도)**
- **수익 전환 지점**: PainGauge 점수가 높을수록 '진단 결과'의 신뢰도가 높아지며, 이는 '솔루션 구매'로 이어집니다.
- **실행 전략**: IP-02 로 계산된 '위험 인지도'가 80 이상일 때만 'AI 컨설팅 유료화'를 제안합니다 (High-intent Targeting).
- **목표치**: 위험 인지도 > 80 일 때 솔루션 구매 전환율 목표 = 15%.

### **IP-03 (해결 의지 측정)**
- **수익 전환 지점**: 진단 페이지에서 '솔루션 클릭률'이 높을수록 '구독/구매'로 이어집니다.
- **실행 전략**: IP-03 로 계산된 '해결 의지'가 60 이상일 때 '무료 샘플' 대신 '유료 솔루션'을 제안합니다 (Value Demonstration).
- **목표치**: 해결 의지 > 60 일 때 유료 전환율 목표 = 10%.

### **IP-04 (신뢰도 및 구매 전환)**
- **수익 전환 지점**: 최종 로드맵 다운로드/구독 시 '신뢰도'가 높을수록 '유료 컨설팅 예약'으로 이어집니다.
- **실행 전략**: IP-04 로 계산된 '신뢰도' 점수가 90 이상일 때만 '1:1 유료 컨설팅'을 제안합니다 (High-Ticket Offer).
- **목표치**: 신뢰도 > 90 일 때 유료 컨설팅 전환율 목표 = 20%.

## 2️⃣ 개발팀 액션: 백엔드 API 스펙 및 데이터 구조 정의
### **필요한 API 엔드포인트**
1.  `/api/v1/user/behavior-log`: 사용자 행동 로그 (클릭, 스크롤) 수집.
    - **Payload**: `{ user_id, event_type, timestamp, page_url, scroll_depth }`
2.  `/api/v1/pain-gauge/calculate`: PainGauge 점수 계산 및 위험 인지도 측정.
    - **Payload**: `{ user_data, diagnostic_results }` (Post-processing 로직 필요)
3.  `/api/v1/trust-widget/update`: Trust Widget 점수 업데이트 및 신뢰도 시각화 데이터 제공.
    - **Payload**: `{ trust_score, last_update_timestamp }`

### **데이터 수집 규칙**
- **PainGauge 점수 계산**: 사용자 입력 데이터를 기반으로 '위험 인지도'를 실시간으로 계산하여 IP-02 로 사용.
- **Trust Widget 업데이트**: 진단 결과를 기반으로 '신뢰도 점수'를 계산하여 IP-04 로 전달.

## 3️⃣ 디자인/마케팅팀 액션: 콘텐츠 전략 및 전환 포인트 정의
### **'위험 → 해결 → 성과' 여정별 메시지 규칙**
- **STEP 1 (위험 인지)**: PainGauge 점수가 높을수록 '무료 진단' 대신 '유료 프리미엄 분석'을 제안합니다. (Upsell Opportunity)
- **STEP 2 (해결 제시)**: Trust Widget 점수가 높을수록 'AI 컨설팅 유료화'를 제안합니다. (High-intent Targeting)
- **STEP 3 (성과 기대)**: IP-04 로 계산된 '신뢰도'가 높을 때 '1:1 유료 컨설팅'을 제안합니다. (High-Ticket Offer)

### **최종 로드맵 페이지 전략**
- **신뢰도 점수 > 90**: '1:1 유료 컨설팅 예약' 버튼 노출.
- **신뢰도 점수 70~89**: '유료 솔루션 구독' 제안.
- **신뢰도 점수 < 70**: '무료 샘플 다운로드' 유도.
