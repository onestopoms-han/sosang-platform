# 🚀 Pitch Deck & LP 목업 제작 스토리텔링 체크리스트
*(근거 자료: Writer님의 고객 여정 시나리오 v1.md)*

## 🎯 섹션 A: 문제 제기 (The Pain Point) — 감성 자극 및 공감대 형성
**목표:** 소상공인이 느끼는 *막연한 불안함*을 구체적인 '위험'으로 정의하고, 기존 해결책의 한계를 각인시킵니다.

| 요소 | 필수 스토리텔링 내용 (Hook) | 시각화 가이드라인 |
| :--- | :--- | :--- |
| **핵심 메시지** | "기술 도입 자체가 아니라, **안정적인 운영**이 문제입니다." | 좌절하는 소상공인의 모습 (이미지/일러스트). 실패 사례 데이터 차트. |
| **Pain Point 1** | 매출 급감, 시장 변화에 대한 무력감 ("나는 뭘 잘못했나?") | '위기'를 상징하는 어둡고 혼란스러운 색채(Warning Orange 활용 가능). |
| **Pain Point 2** | 기술 실패의 비용 (다운타임, 데이터 유실 등) | 돈이 새어나가거나 시스템이 꺼지는 듯한 시각적 메타포. |
| **기존 해결책 한계** | 기능만 강조하는 경쟁사/일반 SaaS 툴은 '결과'를 보장하지 못함. | 복잡하게 얽힌 기술도(Diagram)와 그 끝에 표시된 물음표(?). |

## ✨ 섹션 B: 솔루션 제시 (The Solution) — 신뢰 구축 및 차별화
**목표:** BDS 플랫폼을 단순한 '툴'이 아닌, '위험 관리 컨설턴트'이자 '안전망'으로 포지셔닝합니다.

| 요소 | 필수 스토리텔링 내용 (Hook) | 시각화 가이드라인 |
| :--- | :--- | :--- |
| **핵심 전환 메시지** | "필요한 것은 기능이 아니라, **검증된 안정성(Stability)**입니다." | 어둠에서 밝은 빛으로 전환되는 대비적인 이미지. (Deep Blue 톤 강조). |
| **USP 전면 노출** | 기술적 안정성 지표: Success Rate 95%, Retry Rate 80%를 메인 카드로 배치. | 가장 눈에 띄는 대형 숫자와 그래프. 신뢰도 게이지, '안전 등급' 아이콘 사용 필수. |
| **작동 원리 (Mechanism)** | BDS가 어떻게 기술적 위험을 예측하고 예방하는지 간결하게 플로우차트화. | 간결한 순서도로 설명하며, 각 단계마다 안정성 지표를 언급할 것. |

## 🛡️ 섹션 C: 증명 및 신뢰 확보 (Proof & Trust) — 설득력 강화
**목표:** "우리 말이 아닌, 데이터가 말하게" 만듭니다. 가장 중요한 부분입니다.

| 요소 | 필수 스토리텔링 내용 (Hook) | 시각화 가이드라인 |
| :--- | :--- | :--- |
| **기술 검증 자료** | 최소 기능 단위 시뮬레이터의 E2E 테스트 결과 데이터(Uptime%, 평균 복구 시간 등). | 실제 개발 보고서처럼 보이는 깔끔한 표 또는 대시보드 목업. (전문성 강조). |
| **고객 성공 사례** | '위기'를 겪었던 소상공인 → BDS 도입 후 '안정적 성장'을 이룬 과정. | Before & After 대비형 스토리텔링. 구체적인 매출/효율 개선 수치 제시. |
| **컨설팅 관점 강조** | "BDS는 기술 제공자가 아니라, 사장님의 성공을 돕는 컨설턴트입니다." | 전문적이고 신뢰감 있는 전문가 이미지 또는 심볼 마크. |

## ✅ 섹션 D: 행동 유도 (The CTA) — 다음 스텝 제시
**목표:** 고객이 망설이지 않고 다음 단계로 나아가게 합니다.

| 요소 | 필수 스토리텔링 내용 (Hook) | 시각화 가이드라인 |
| :--- | :--- | :--- |
| **CTA 1 (낮은 장벽)** | "무료 불안감 진단 받기" (`bds-platform.co.kr/diagnosis`) | 버튼 형태의 명확하고 큰 CTA. 심리적 부담이 적음을 강조. |
| **CTA 2 (높은 가치)** | "맞춤형 데모 신청: 내가 필요한 기술 컨설팅 받기" | 상담원과의 대화 또는 개인화된 결과물을 보여주는 목업. |
