# 🎨 V3.0 디자인 시스템 & 상태 매핑 최종 구현 지침 (Frontend Specification)

## 1. 브랜드 컬러 시스템 확정

디자인 시스템의 핵심 색상과 상태(State)에 따른 동적 색상을 명확히 정의합니다.

| 항목 | 역할 | HEX 코드 | 용도 및 설명 |
| :--- | :--- | :--- | :--- |
| **Primary (신뢰/기본)** | 기본 배경, 안정적인 상태 | `#004D66` (Deep Blue) | 시스템의 기본 색상. 신뢰와 안정감을 나타냄. |
| **Success (성장/긍정)** | 긍정적 상태, 성공 시 | `#3CB371` (Growth Green) | 목표 달성 및 긍정적인 성장을 나타냄. |
| **Warning (경고)** | 주의 필요 상태 | `#FF8C00` (Orange) | 잠재적인 위험 또는 주의가 필요한 상태를 명확히 알림. |
| **Critical (위기)** | 즉각적인 조치가 필요한 심각한 상태 | `#DC143C` (Crimson Red) | 가장 높은 수준의 위기 상황을 나타내며, 즉각적인 행동을 유도함. |

## 2. 상태 기반 시각화 매핑 (State-to-Style Mapping)

사용자의 신뢰도(Trust Level) 상태에 따라 UI 컴포넌트가 동적으로 변화하는 규칙을 정의합니다. 이 매핑은 Trust Widget 및 PainGauge의 핵심 디자인 요소에 적용되어야 합니다.

### 2.1. Trust Widget 구현 지침

| Trust Level | 시각적 효과 (Primary/Background) | 텍스트 색상 | 상호작용 피드백 |
| :--- | :--- | :--- | :--- |
| **High Trust** (80점 이상) | `Primary` (`#004D66`) 배경 + 녹색 테두리 | `#3CB371` (Growth Green) | 안정적이고 긍정적인 느낌 강조. |
| **Medium Trust** (50% ~ 79%) | `Primary` 배경 + 주황색 경고 바 | `#FF8C00` (Orange) | 주의를 환기하며 개선이 필요함을 시사. |
| **Low Trust** (20% ~ 49%) | `Primary` 배경 + 빨간색 경고 바 | `#DC143C` (Crimson Red) | 즉각적인 조치가 필요함을 강조. |
| **Critical** (<20%) | 전체 영역에 `Critical` 색상 (`#DC143C`) 적용 | 흰색 (`#FFFFFF`) | 최대 위험 상태로 명확히 구분하고 강력한 행동 유도. |

### 2.2. PainGauge 구현 지침 (위기 수준 시각화)

PainGauge는 현재 사용자가 느끼는 고통(Pain)의 수준을 직관적으로 보여주어야 하며, 색상 변화를 통해 위기감을 전달해야 합니다.

*   **Low Pain (안정):** 녹색 계열 (`#3CB371` 근접)
*   **Medium Pain (주의):** 주황색 계열 (`#FF8C00` 근접)
*   **High Pain (위기):** 빨간색 계열 (`#DC143C` 근접)

## 3. 컴포넌트 개발팀 전달 상세 지침

### 3.1. 핵심 디자인 원칙

1.  **명확성 우선:** 모든 상태 변화는 사용자가 즉시 이해할 수 있도록 색상과 레이아웃으로 명확하게 구분되어야 합니다.
2.  **행동 유도 (Action-Oriented):** `Critical` 상태에서는 시각적 경고를 넘어, 텍스트나 버튼에 대한 다음 행동(예: "즉시 컨설팅 신청")을 강제적으로 제시해야 합니다.
3.  **일관성 유지:** 모든 컴포넌트(`Trust Widget`, `PainGauge`)에서 위에서 정의된 색상 코드를 **절대적으로** 사용해야 하며, 다른 보조 색상(Secondary Colors)은 시스템의 핵심 메시지 전달에 집중합니다.

### 3.2. API 연동 지침 (Backend State Mapping)

프론트엔드는 백엔드로부터 수신하는 상태 값(`trust`, `warning`, `critical`)을 기반으로 위에서 정의한 색상 매핑 로직을 **Client-Side**에서 실행하여 UI를 렌더링해야 합니다.

*   **Input:** `{ trust: 75, warning: false, critical: false }`
*   **Logic:** `if (trust >= 80) { applyHighTrustStyle(); } else if (trust >= 50) { applyMediumTrustStyle(); } ...`

이 지침을 바탕으로 프론트엔드 팀은 컴포넌트를 개발하고 API 연동 로직을 구현해 주십시오.
