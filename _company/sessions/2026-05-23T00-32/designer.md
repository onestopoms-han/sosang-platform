# 📐 C-02 상세 페이지 핵심 섹션 디자인 확정안

**테마:** 손실 최소화 (Loss Minimization)
**핵심 비주얼 목표:** 막연한 불안을 데이터 기반의 명확하고 신뢰할 수 있는 해결책으로 전환시킨다.

## 1. 전체 컬러 및 타이포그래피 시스템 적용 (Visual System Reinforcement)

이 섹션에서는 브랜드 정체성(BDS소상공인플렛폼)에 맞춘 시각적 언어를 일관되게 사용합니다.

| 요소 | 설정값 | 설명 |
| :--- | :--- | :--- |
| **Primary Color (신뢰)** | `#007AFF` (Bright Blue) | 데이터의 정확성과 전문성을 상징하며, CTA 버튼에 집중적으로 사용. |
| **Secondary Color (경고/Pain)** | `#FF3B30` (Red Accent) | 손실, 위험, 주목해야 할 문제점을 강조하는 데 사용. 대비를 극대화하여 긴급성을 부여. |
| **Background Color** | `#F8F9FA` (Light Gray) | 깨끗하고 데이터에 집중할 수 있는 여백을 제공. |
| **Typography (제목)** | Pretendard Bold / SemiBold | 명확성과 권위를 나타냄. |
| **Typography (본문)** | Pretendard Regular / Medium | 가독성을 최우선으로 하여 복잡한 데이터를 쉽게 이해하도록 함. |

## 2. 섹션별 레이아웃 및 비주얼 콘셉트 상세

### 섹션 A: Pain Point 제시 (후크)
**목표:** 소상공인의 현재 고통과 불안감을 즉각적으로 공감하게 만든다. (A. Pain Point 제기 단계)

| 구성 요소 | 디자인 레이아웃 및 시각적 콘셉트 | 적용되는 스토리텔링 |
| :--- | :--- | :--- |
| **헤드라인** | **"매출은 올랐는데, 통장은 왜 비어있을까요?"** (가장 크고 굵게) | 직접적인 질문으로 주의를 집중. (Pain Point의 명시) |
| **메인 비주얼** | **'불안의 시각화 그래프'** : 중앙에 큰 숫자와 함께, 매출(상승 화살표)과 비용(하락/누수 흐름)을 대비시키는 인포그래픽 스타일의 다이어그램. (Secondary Color `#FF3B30` 사용) | 감성적 공감 + 데이터 기반 문제 제기. |
| **서브 텍스트** | "당신의 숨겨진 지출은 어디에 있습니까? 경험으로만 추측하던 불안을 이제 숫자로 확인하세요." | Pain Point의 구체화. |
| **레이아웃 특징** | **대비 강조 (Contrast Focus)**: 배경은 밝게 유지하되, 문제점을 나타내는 그래프 영역에만 강렬한 빨간색과 파란색 대비를 사용하여 시각적 충격을 준다. |

### 섹션 B: Solution 소개 (핵심 가치)
**목표:** BDS 플랫폼이 제공하는 '손실 최소화' 프레임워크가 어떻게 이 문제를 해결하는지 명확하게 제시한다. (B. Solution 제시 단계)

| 구성 요소 | 디자인 레이아웃 및 시각적 콘셉트 | 적용되는 스토리텔링 |
| :--- | :--- | :--- |
| **헤드라인** | **"손실 최소화 공식: 데이터 기반의 성장 로드맵"** (Primary Color `#007AFF` 사용) | 해결책에 대한 확신과 권위를 부여. (Solution 제시) |
| **핵심 가치 소개** | **3단계 구조 박스**: '진단' $\rightarrow$ '분해' $\rightarrow$ '실행'의 세 단계로 구성된 깔끔한 카드형 레이아웃. 각 단계는 아이콘(🔍, 🧩, 🚀)으로 명확히 구분. | 문제 분해 및 해결 프로세스의 논리적 흐름 강조. |
| **핵심 기능 소개** | **‘Diagnosis Service’ 시각화**: 진단 결과가 어떻게 구체적인 실행 방안(예: 재고 최적화, 마케팅 예산 조정)으로 변환되는지 보여주는 플로우 차트 삽입. | 플랫폼의 핵심 기능 사용법을 직관적으로 안내. |
| **레이아웃 특징** | **명확한 흐름 (Flow)**: 위에서 아래로 명확하게 이어지는 선형 구조를 사용하여, 사용자가 다음 단계로 자연스럽게 이동하도록 유도한다. 여백(Whitespace)을 충분히 확보하여 복잡함을 줄인다. |

## 3. 최종 디자인 가이드 요약 (Designer Guideline Summary)

1.  **시각적 흐름:** **Red Alert (Pain)** $\rightarrow$ **Blue Clarity (Solution)** 의 명확한 대비를 통해 사용자의 시선을 자연스럽게 플랫폼의 해결책으로 이동시킨다.
2.  **톤앤매너:** 전문적이고, 공감하며, 데이터에 기반한 신뢰감을 주는 디자인을 유지한다. 장식적인 요소는 최소화하고 정보 전달의 명료성에 집중한다.
3.  **CTA 배치:** 각 섹션이 끝날 때마다 다음 단계로 나아갈 수 있는 명확하고 강조된 CTA 버튼(Primary Color 사용)을 배치하여, 사용자가 망설임 없이 다음 행동을 취하도록 설계한다.
