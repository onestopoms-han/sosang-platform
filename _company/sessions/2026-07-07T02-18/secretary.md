## 📋 핵심 통합 테스트 완료 체크리스트 (Master Checklist)

플랫폼의 핵심 가치인 '불확실성 감소 경험' 판매와 '통제권 회복'이라는 감성적 서사를 기술적으로 검증하는 데 중점을 두었습니다.

### 1. 데이터 및 로직 안정성 (Backend Focus)
*   [ ] **위험 예측 엔진 연동 테스트:** 실시간 위험 데이터 입력 시, 백엔드에서 계산된 '불확실성 감소 점수'가 정확하게 API 응답 스키마에 반영되는지 검증. (특히 `Validated` 플래그 확인)
*   [ ] **동적 가격 책정 로직 검증:** 사용자 리스크 단계(0%~15%)에 따른 수수료($Y$) 계산 로직이 모든 경계값에서 오류 없이 작동하는지 테스트.
*   [ ] **감정 지표 연동 테스트:** API 응답 시 포함되는 '감정적 상태 지표'가 UI/UX 전환 트리거와 정확하게 매핑되는지 확인.
*   [ ] **시스템 안정성(Stress Test):** 급격한 데이터 변화 또는 예외 상황 발생 시, 시스템이 다운되지 않고 '회복(Recovery)' 상태로 안전하게 전환되는지 검증.

### 2. UI/UX 및 상호작용 일관성 (Frontend Focus)
*   [ ] **Trust Widget 기능 테스트:** `PainGauge` 및 `Trust Widget` 컴포넌트가 모든 리스크 등급(A/B/C)에 따라 지정된 색상 팔레트와 애니메이션을 완벽하게 구현하는지 확인.
*   [ ] **Recovery Flow 검증:** Deep Blue(위기)에서 Deep Indigo/Gold-Amber(회복)로의 UI 전환 시, 설정된 속도 및 애니메이션이 기술 사양서(`Trust_Widget_Technical_Spec_V1.0`)에 맞게 부드럽고 안정적으로 구현되는지 확인.
*   [ ] **핵심 서사 일관성:** 모든 화면과 인터랙션에서 '불확실성 감소'라는 핵심 가치 전달이 기능적 오류 없이 감성적 흐름을 따라가는지 점검.
