# 📊 Dashboard Prototype Specification (V3.0)

## 1. 디자인 시스템 요약 (Reference: Trust Widget & PainGauge V3.0)

본 프로토타입은 **신뢰도(Trust)**와 **위험도(Risk)**의 시각적 표현에 모든 디자인 자원을 집중하여, 소상공인이 직관적으로 상태를 인지하고 행동을 유도하도록 설계되었습니다.

### 1.1. 컬러 팔레트 (Color Palette)

| 상태 | Trust Widget 색상 | PainGauge 색상 | 배경/기본 텍스트 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| **Green (정상)** | `#4CAF50` (Primary Success) | `#4CAF50` | `#F9F9F9` (Light Gray Background) | 신뢰/안정 상태. 긍정적 행동 유도. |
| **Amber (주의)** | `#FFC107` (Warning) | `#FFC107` | `#FFF8E1` (Light Yellow Background) | 주의 필요 상태. 검토 및 조치가 필요함. |
| **Red (위험)** | `#F44336` (Error/Danger) | `#F44336` | `#FFEBEE` (Light Red Background) | 위험 상태. 즉각적인 조치 요구. |

### 1.2. 타이포그래피 및 레이아웃 (Typography & Layout)

*   **주요 정보:** 산세리프 계열 (예: Pretendard 또는 Inter). 데이터 값은 크고 굵게 표시하여 시각적 강조.
*   **레이아웃 구조:** 3단 그리드 기반의 카드형(Card-based) 레이아웃을 채택하여 정보의 위계질서를 명확히 합니다.

## 2. 핵심 대시보드 컴포넌트 상세 스펙

### 2.1. Trust Widget (신뢰도 표시 영역)

**목표:** 현재 데이터 신뢰도를 한눈에 파악하고, 다음 행동을 유도합니다.

*   **위치:** 대시보드 상단 중앙 또는 좌측 상단 고정 위치.
*   **구성 요소:**
    1.  **Title:** "현재 사업 신뢰도" (Bold, 중형 폰트)
    2.  **Gauge Visualization:** 원형 게이지(Progress Ring)를 사용하여 현재 상태(`Green`/`Amber`/`Red`)의 채워진 정도를 시각화합니다.
    3.  **Status Indicator:** 선택된 색상으로 배경 및 테두리 강조. (예: Green일 경우, 녹색 배경에 흰색 텍스트).
    4.  **Action Prompt:** 상태에 따라 아래 액션 버튼을 동적으로 표시합니다.

| TrustLevel | 시각적 표현 | 행동 유도 메시지 예시 |
| :--- | :--- | :--- |
| **Green** | 녹색 게이지, 긍정적인 아이콘 | "사업 안정, 다음 단계로 이동" (Primary Button) |
| **Amber** | 주황색 게이지, 경고 아이콘 | "데이터 점검 필요, 컨설팅 요청하기" (Secondary Button) |
| **Red** | 빨간색 게이지, 위험 아이콘 | "위험 신호 감지, 긴급 지원 요청" (Urgent Button) |

### 2.2. PainGauge (위험도 표시 영역)

**목표:** 현재 소상공인이 직면한 잠재적 위험 수준을 정량화하여 심각성을 인지시킵니다.

*   **위치:** Trust Widget의 보조 위치 또는 별도의 섹션으로 배치.
*   **구성 요소:**
    1.  **Title:** "잠재적 위험 지수" (Bold, 중형 폰트)
    2.  **Gauge Visualization:** 수평 막대(Linear Bar) 또는 단계별 인디케이터를 사용하여 위험 수준(`0`~`3`)을 시각적으로 표현합니다.
    3.  **Risk Level Display:** 현재 위험 등급 숫자 표시 (예: `2/3`).
    4.  **Contextual Description:** 해당 위험도에 따른 구체적인 해석 제공.

| RiskLevel | 시각적 표현 | 설명 예시 |
| :--- | :--- | :--- |
| **0** | 낮은 막대, Green 색상 | "안정적입니다. 지속적인 성장을 목표로 합니다." |
| **1** | 중간 막대, Amber 색상 | "주의 구간 진입. 재정 흐름 점검이 필요합니다." |
| **2** | 높은 막대, Red 색상 | "위험 수준. 즉각적인 리스크 관리 컨설팅을 권장합니다." |
| **3** | 최대 막대, Red 색상 | "심각한 위험. 긴급 지원 및 구조 조정이 필요합니다." |

## 3. 핵심 대시보드 레이아웃 (Wireframe Concept)

**구조:** 3단 그리드 기반의 위에서 아래로 정보가 흐르는(Top-down flow) 명확한 계층 구조를 사용합니다.

| 섹션 | 내용 | 디자인 강조점 | 목적 |
| :--- | :--- | :--- | :--- |
| **A. 헤더 (Header)** | 사용자 이름, 알림, 빠른 액션 버튼 | 미니멀리즘 유지 | 접근성 및 브랜드 인식 |
| **B. 핵심 상태 시각화 (Trust/Pain)** | **Trust Widget**과 **PainGauge**를 가장 크게 배치 | 색상 대비 극대화 | 사용자의 즉각적인 상태 인지 |
| **C. 주요 KPI 카드 (Summary Cards)** | 월별 매출 추이, 고객 신뢰도 변화 그래프 등 핵심 데이터 요약 | 차트(Chart) 중심의 시각화 | 데이터 기반 의사결정 지원 |
| **D. 행동 유도 영역 (Actionable Insights)** | AI 기반 예측 분석 결과, 다음 권장 조치 목록 | 명확한 Call-to-Action 버튼 | 구체적인 다음 단계 제시 |

### 3.1. 프로토타입 흐름 정의 (Interaction Flow)

1.  **사용자 접근:** 사용자가 대시보드에 로그인하면, 시스템은 자동으로 **Trust Widget**과 **PainGauge**의 현재 상태를 즉시 로드합니다.
2.  **상태 기반 필터링:** Trust/Pain 값에 따라 전체 대시보드의 톤(전체 배경색 또는 강조 색상)이 해당 상태의 색상(Green, Amber, Red)으로 미묘하게 변화하여 사용자의 감정적 반응을 즉각적으로 반영합니다.
3.  **행동 유도:** **Red (위험)** 상태가 감지될 경우, 시스템은 C 섹션의 KPI 카드 대신 D 섹션의 '긴급 지원 요청' 버튼을 가장 눈에 띄게(Primary Color) 표시하여 사용자의 시선을 즉시 위험 해결 경로로 유도합니다.
