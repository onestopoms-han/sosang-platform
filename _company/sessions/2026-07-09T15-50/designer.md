## 🚀 최종 인터랙티브 UX Flow 설계: 실행 책임 여정 (Execution Accountability Journey)

### 1. 전체 흐름 개요 (Journey Overview)

플랫폼은 사용자를 **'불안감(Uncertainty)'**에서 **'실행 책임(Accountability)'**으로 이끄는 3단계의 시각적 여정을 제공합니다. 각 단계의 완료는 다음 단계로 나아가는 명확한 '통제권 회복'을 의미하며, 이는 시각적 보상으로 즉각화됩니다.

**Flow State:** $S1 \rightarrow S2 \rightarrow S3$ (Diagnosis $\rightarrow$ Problem Recognition $\rightarrow$ Action Plan)

### 2. 핵심 컴포넌트 스펙 및 동적 변화 가이드

#### A. 메인 게이지: 실행 책임 바 (Accountability Bar)
이것은 전체 여정의 진행 상황을 나타내는 가장 중요한 시각 요소입니다.

*   **컴포넌트 명:** Accountability Progress Bar
*   **기능:** $S1$에서 시작하여 $S3$ 완료 시까지 단계별 진행률을 표시합니다.
*   **시각적 변화 (색상 코드):**
    *   **$S1$ (진단):** 짙은 회색/어두운 파란색 (`#4a5568` - 불안정성 표현)
    *   **$S2$ (문제 인식):** 주황색/경고색 (`#D97706` - 인지 단계, 주의 필요)
    *   **$S3$ (실행 계획):** 성공 녹색 (`#38A169` - 실행 가능성 확보, 통제권 회복 시작)
    *   **완료 상태:** 밝은 청록색/골드 혼합 (`#6B7280` + `gold`)

*   **애니메이션 가이드:** 각 단계 완료 시, 이전 단계의 바가 부드럽게 채워지며 다음 단계의 색상으로 전환되는 **'누적 채움(Cumulative Fill)' 애니메이션**을 적용합니다. 마우스를 올리면 각 단계별 목표와 행동 지침이 툴팁 형태로 즉시 표시되어야 합니다.

#### B. 액션 블루프린트 상세 뷰 (Action Blueprint Detail View)
$S3$ 단계에서 사용자가 실제로 실행할 항목들을 보여주는 인터페이스입니다.

*   **구조:** 계층적 트리 구조(Tree Structure)를 기본으로 하되, 각 행동 지침(Action Step)은 독립적인 카드 형태로 제시됩니다.
*   **컴포넌트 스펙:**
    *   **Header (Step ID):** `Step X: [Writer 카피라이팅 기반 목표]` (굵고 명확하게)
    *   **Goal Card:** 사용자가 달성해야 할 구체적인 목표 (`goal` 필드)를 강조합니다.
    *   **Accountability Widget:** 각 Action Step에는 해당 행동이 플랫폼의 핵심 가치와 어떻게 연결되는지 표시합니다.
        *   `Metric`: `FINANCIAL_IMPACT` 또는 `TIME_SAVED` (코다리 Schema 기반)
        *   `Target`: 목표 값 (`target_value`)과 현재 진행 상태 (`status: PENDING/IN_PROGRESS/COMPLETED`).
    *   **Action Detail Panel:** `Instruction` 필드에 대해, 사용자가 클릭하면 **'실행 책임'을 강화하는 구체적인 행동 지침(Writer 카피라이팅 기반)**이 펼쳐지도록 설계합니다.

### 3. 인터랙션 및 시각적 보상 흐름 상세 (Interaction Flow)

| 단계 | 사용자 경험 목표 | 시각적 변화 (UI/UX) | 핵심 피드백 (Copywriting 연계) |
| :--- | :--- | :--- | :--- |
| **S1: 진단** | 불안정성 인지 및 데이터 수용 | Risk Gauge Widget이 고정되어 표시됨. 바는 짙은 회색으로 채워짐. | "현재 상황의 객관적 진단" (데이터 기반 제시) |
| **$S1 \rightarrow S2$ 전이** | 문제에 대한 공감 형성 | 게이지가 주황색으로 변하며, 다음 단계로 이동할 때 '다음 질문' 버튼 활성화. | "당신의 불안정성을 명확히 인식했습니다." (문제 인식 촉구) |
| **S2: 문제 인식** | 해결책의 필요성 인식 | 중간 Accountability Bar가 채워지며 주황색에서 녹색으로 서서히 전환 시작. | "이러한 문제를 해결하기 위한 구체적인 로드맵이 필요합니다." (해결책 제시) |
| **$S2 \rightarrow S3$ 전이** | 실행 의지 확보 및 통제권 회복 시작 | Accountability Bar가 녹색으로 급격히 채워지며, 부드러운 상승 애니메이션 발생. | "당신의 행동 계획은 이제 구체화되었습니다." (실행 책임 부여) |
| **S3: 실행 계획** | 명확한 행동 지침 수용 | Action Blueprint 상세 뷰가 활성화되며, 개별 Action Step의 `status` 필드에 따라 카드 색상이 변경됨. | "다음 행동을 시작하세요. 이 행동이 당신의 통제권을 회복시킵니다." (행동 책임감 부여) |
| **S3 완료** | 성취감 극대화 | 전체 Accountability Bar가 완전한 녹색으로 채워지고, 중앙에 'MISSION COMPLETE: EXECUTION ACCOUNTABILITY ACHIEVED' 메시지 및 축하 애니메이션. | "성공적으로 실행 로드맵을 완성했습니다." (최종 보상) |

### 4. 디자인 가이드라인 요약

1.  **톤앤매너:** 정보 나열이 아닌, **여정(Journey)**과 **실행(Action)**에 초점을 맞춘 동적이고 지지적인 톤을 유지합니다.
2.  **색상 사용:** 불안정성($S1$) $\rightarrow$ 경고($S2$) $\rightarrow$ 확신($S3$)의 심리적 흐름에 맞춰 색상을 명확하게 변화시켜야 합니다 (회색/주황/녹색).
3.  **애니메이션 우선순위:** 정적인 레이아웃보다 **상태 변화(State Transition)**와 **누적 채움(Cumulative Fill)** 애니메이션을 최우선으로 설계하여 사용자의 심리적 보상을 즉각적으로 전달합니다.
