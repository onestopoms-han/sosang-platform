## 📝 Trust Widget & PainGauge 최종 UI/UX 사양 (V3.0) 요약

### 1. 핵심 목표 재확인
*   **목표:** 소상공인이 자신의 노력을 투자했을 때 얻는 경제적 이익(ROI)과 심리적 고통(Pain)을 직관적으로 인지하고, 다음 행동을 유도한다.
*   **핵심 가치:** 신뢰성 (Trust), 통찰력 (Insight), 행동 촉구 (Action).

### 2. 데이터 매핑 및 시각화 전략 (API Output Schema 기반)

새로운 API 출력 스키마가 제공되면, 아래와 같이 데이터를 UI 요소에 매핑합니다.

| 지표 그룹 | 입력 데이터 (예시: 신규 API 필드) | 시각화 방법 (Visualization) | 상호작용 (Interaction) |
| :--- | :--- | :--- | :--- |
| **Trust Widget** | `trust_score` (0-100), `control_group_enabled` | 게이지(Gauge) 또는 원형 진행도(Progress Ring) | 마우스 오버 시 상세 설명 팝업, 그룹 비교 시 차트 전환 |
| **PainGauge** | `pain_level` (1-10), `reduction_rate` (%) | 심리적 상태를 나타내는 색상 변화 및 아이콘 | 슬라이더(Slider) 조작을 통한 데이터 범위 탐색 |
| **ROI 연동** | `calculated_roi`, `base_cvr`, `aov` (기준값 대비) | 핵심 지표 대시보드 형태로 배치 | ROI가 기준치보다 높을 경우, 긍정적인 색상(녹색 계열) 강조 |

### 3. 컴포넌트별 시각적 명세 (Visual Specification)

#### A. Trust Widget 디자인 상세
*   **레이아웃:** 카드 형태 또는 몰입형 모달 형태로 구성하여 정보의 무게감을 부여합니다.
*   **컬러 팔레트:** 신뢰(Trust)를 상징하는 딥 블루 계열(`#0A4C9A`)과 성과(Result)를 나타내는 밝은 청록색/녹색(`#38A169`)을 주 색상으로 사용합니다.
*   **타이포그래피:** ROI 수치는 가장 크고 굵게 표시하며, 나머지 설명 텍스트는 명확성을 위해 간결한 산세리프체(예: Inter 또는 Pretendard)를 사용합니다.
*   **인터랙션 상세:**
    1.  사용자가 Trust Score에 마우스를 올리면, **`trust_score`의 변화 추이 그래프**가 실시간으로 로드되어 표시됩니다.
    2.  `control_group_enabled` 값에 따라, 해당 데이터가 A/B 테스트 그룹 내에서 어떻게 분포되었는지 시각적으로 분할하여 보여줍니다 (Bar Chart 또는 Stacked Bar).

#### B. PainGauge 디자인 상세
*   **레이아웃:** 단순성과 감정적 연결성을 극대화합니다.
*   **색상 체계:** Pain Level에 따라 색상을 명확히 구분합니다.
    *   Low (1-3): 녹색 계열 (안정)
    *   Medium (4-7): 노란색/주황색 계열 (경고)
    *   High (8-10): 빨간색 계열 (위험)
*   **시각 요소:** 숫자 외에 해당 Pain Level을 시각적으로 표현하는 **추상적인 아이콘 또는 애니메이션 효과**를 추가하여 감정적 몰입도를 높입니다.

### 4. 디자인 시스템 통합 제안
새로운 API 스키마가 확정되면, 이 사양을 기반으로 다음 요소를 업데이트합니다.
1.  **컬러/타이포 리소스:** Trust 및 Pain 상태에 따른 동적 색상 매핑 테이블 생성.
2.  **컴포넌트 라이브러리:** `TrustWidget` 컴포넌트와 `PainGauge` 컴포넌트에 데이터 바인딩 로직(Data Binding Logic)을 명세화하여 개발팀에 전달합니다.
