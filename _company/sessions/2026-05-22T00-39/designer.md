# 비교 모달 디자인 사양서 (v1.1 - 최종 개발용)

## 1. 구조 및 레이아웃 정의
**컴포넌트 이름:** ComparisonModal
**목표:** Basic와 Premium 플랜의 핵심 가치 차이를 시각적으로 극대화하여 사용자가 즉시 선택하도록 유도한다.

### 1.1. 전체 레이아웃 (Container)
- **컨테이너 크기:** 최대 너비 800px, 최대 높이 700px. 중앙 정렬(Center Alignment).
- **패딩 (Padding):** 내부 콘텐츠 영역에 최소 40px의 내부 패딩을 적용하여 시각적 여백 확보.
- **배경:** 부드러운 그림자 효과(Soft Shadow)를 사용하여 모달이 배경과 분리되도록 처리.

### 1.2. 헤더 영역 (Header)
- **제목:** "플랜 비교하기" (H2, Font Size: 24px). 왼쪽 정렬.
- **닫기 버튼 (Close Button):** 우측 상단에 배치. 아이콘(X), 클릭 시 모달 닫기.

### 1.3. 플랜 비교 영역 (Comparison Area)
이 영역은 **Tabbed View** 방식을 채택하여 정보의 밀도를 높인다.
- **탭 네비게이션:** 상단에 Basic, Premium 두 개의 탭을 배치. 선택된 탭 아래로 정보가 표시된다.
- **정보 카드 레이아웃 (Card Layout):** 각 플랜(Basic/Premium)은 독립적인 '카드' 형태로 디자인한다.

#### 1.3.1. 개별 플랜 카드 상세 사양
**A. Basic Plan Card:**
- **배경 색상:** Light Gray (#F8F9FA).
- **핵심 가치 강조:** '경제적 위협 감소', '기본 기능 제공' 등의 키워드를 아이콘과 함께 명확히 제시.
- **가격 표시:** **₩9,900/월** (가장 크고 굵게, Primary Color 사용).
- **추가 정보:** 포함되는 핵심 기능 목록 (Bullet Points).
- **CTA 버튼:** 'Basic 선택' (Primary Button 스타일).

**B. Premium Plan Card (선택된 상태):**
- **배경 색상:** Accent Blue (#007BFF) 또는 Light Shade. (사용자에게 가치를 부여하는 색상 사용).
- **핵심 가치 강조:** '시간 및 시행착오 감소', '전문 컨설팅 제공' 등 **가치 중심의 문구**를 굵게 처리하여 강조.
- **가격 표시:** **₩29,900/월** (가장 크고 굵게, Primary Color 사용).
- **추가 정보:** 포함되는 모든 기능 목록 및 컨설팅 가치 명시.
- **CTA 버튼:** 'Premium 선택' (**Primary Action Button** 스타일 - 가장 눈에 띄게).

### 1.4. 데이터 연동 (API Mapping)
- **입력 데이터:** `diagnosis_input_schema.ts`에서 도출된 기능 목록과 가격 정보를 기반으로 동적으로 카드 내용을 채운다.
- **핵심:** API 응답 시, 각 플랜에 해당하는 **시각적 스타일링 변수**를 함께 반환하도록 JSON 스키마를 업데이트한다.

## 2. 상호작용 및 상태 (Interaction & State)
- **로딩 상태:** 데이터 로딩 중에는 전체 모달을 50% 오버레이하고 중앙에 '데이터 로딩 중...' 스피너 표시.
- **선택 후 피드백:** 사용자가 플랜을 선택하면, 선택된 카드에 명확한 시각적 하이라이트(Border, Background Change)를 즉시 적용하여 선택 완료를 알린다.

### 2.1. 최종 검토 및 개발 지침 (Developer Notes)
- **개발팀 참고 사항:**
    1.  모든 간격은 `rem` 또는 `px` 단위로 일관성을 유지해야 합니다. 특히 카드 간의 수직 간격은 최소 32px 이상을 확보하여 가독성을 높여야 합니다.
    2.  Premium 플랜 선택 시, 배경색과 CTA 버튼 색상에 **브랜드 강조 색상(Accent Color)**을 적극적으로 활용하여 Premium 옵션의 가치를 극대화해야 합니다.
    3.  API 응답은 단순 데이터뿐만 아니라, 각 플랜의 디자인 변수(예: `card_style: 'basic' | 'premium'`)를 포함하도록 수정해야 합니다.
