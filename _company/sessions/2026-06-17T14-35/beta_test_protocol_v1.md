# Beta_Test_Protocol_V1.0 — 소상공인 성공 여정 추적 프로토콜

## 📌 1. 목적 및 배경
- **목적**: BDS 플랫폼의 실제 효과를 측정 가능한 지표로 증명하고, 소상공인의 '성장 여정'을 시각화하여 신뢰를 확보함.
- **핵심 가치**: 단순 기능 사용이 아닌, 사장님의 **성장 단계별 니즈**를 충족시키는 솔루션으로서의 가치를 입증.

## 📌 2. 핵심 KPI (성공 여정 추적 지표)
| 지표명 | 정의 | 목표치 (1 개월 기준) | 측정 주기 |
|--------|------|---------------------|-----------|
| **Journey Completion Rate** | 프로토콜을 온보딩부터 실행까지 완료한 사용자 비율 | 60% 이상 | 매일 |
| **NPS (Net Promoter Score)** | "이 플랫폼으로 매출이 늘었다"라고 생각하는 사장님 비율 | +50 이상 | 월별 |
| **Revenue Impact** | 프로토콜 사용 후 실제 매출 증가분 추정치 | +20% 이상 | 3 개월累计 |
| **Feature Adoption Rate** | 핵심 기능 (예: AI 리스크 예측) 의 활성 사용률 | 40% 이상 | 매일 |

## 📌 3. 프로토콜 단계별 액션 플랜
### 단계 1: 온보딩 (Day 1-2)
- **행동**: 'Pilot' 앱 설치 → 5 분 퀴즈 → 성공 여정 맵 시각화
- **목표**: 사장님이 자신의 가게를 '성장 여정의 한 명인 캐릭터'로 인식하도록 유도.

### 단계 2: 학습 및 실험 (Day 3-7)
- **행동**: 교육 커리큘럼의 1 차 모듈 (데이터 기반 성장) → AI 리스크 예측 도구 사용 체험
- **목표**: '안전 마진' 개념을 이해하고, 실제 데이터를 입력하여 '위험도'를 확인하는 경험.

### 단계 3: 추적 및 피드백 (Day 8-14)
- **행동**: 성공 여정 대시보드에 자신의 성장 곡선 기록 → 다른 사장님과 비교 (익명)
- **목표**: '성장 가능'이라는 동기를 부여하고, 실제 매출 영향력을 시각화.

### 단계 4: 확장 및 공유 (Day 15-30)
- **행동**: 여정 완료 보상 → 성공 사례 포털에 업로드 → 다른 소상공인에게 추천
- **목표**: 플랫폼의 사회적 가치를 증명하고, 추가 사용자 유입을 유도.

## 📌 4. 데이터 수집 계획
- **수집 방법**: 앱 내 이벤트 로깅, 설문조사 (NPS), 매출 변화 추종 (사용자 동의 하)
- **보안**: 소상공인 개인정보 보호법 준수 (익명화 처리)

## 📌 5. 기대 효과 및 다음 단계
- **기대 효과**: 플랫폼의 '실제 가치'를 수치로 증명하여 투자자와 정부의 관심을 확보.
- **다음 단계**: 프로토콜 실행 → 데이터 분석 → 커리큘럼 수정 → 대량 배포 준비
