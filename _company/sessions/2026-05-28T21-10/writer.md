## 2. 최종 스토리라인: 데이터 기반의 감정적 여정 (Data-Driven Emotional Journey)

개발팀 (코다리) 이 구현해야 할 핵심 로직은 단순한 UI/UX가 아니라, **사용자가 시스템과 상호작용할 때 느끼는 감정적 변화**를 기술적으로 반영하는 것입니다.

### 📖 스토리라인의 3 단계
1.  **Pain 인식 단계 (Data Warning):** 사용자의 비즈니스 데이터 (예: 매출 하락) 가 빨간색으로 표시되고, 경고 모션이 발생하며, '불안'을 유발합니다.
2.  **Solution 기대 단계 (Data Healing):** BDS 시스템이 분석 결과를 제시하고, 파란색의 안정감 있는 UI 로 전환되며, '희망'과 '해결'을 제공합니다.
3.  **Action 유도 단계 (Data Empowerment):** 사용자가 구체적인 솔루션 (예: 대출 신청, 마케팅 제안) 을 선택하면, 데이터 기반의 성공 사례가 나타나며, '신뢰'와 '동기부여'를 심어줍니다.

### 🧩 핵심 메시지 (Core Message)
**"BDS 는 당신의 비즈니스 데이터를 이해하고, 그 고통을 공감하며, 해결책을 제시하는 디지털 컨설턴트입니다."**

## 3. 개발 구현 가이드: 스토리라인과 기술 로직의 연결점

코다리 에이전트가 이제 바로 코드를 작성할 때, 아래 사항을 반드시 고려해야 합니다. 이 내용은 디자인 시스템과 개발 명세서 간의 **브릿지 (Bridge)** 역할을 하는 문서입니다.

### 🧩 1. 데이터 → 색상/모션 매핑
```javascript
// 예시: 매출 하락률 (-20%) vs 색상 전환
if (data.revenue_change < -15) {
    // Pain 단계 로직
    setUIColor('danger'); // 빨강 계열
    applyMotionToken('shake_warning'); // 진동 효과
} else if (data.revenue_change >= 0) {
    // Solution 단계 로직
    setUIColor('primary_blue'); // 파랑 계열
    applyMotionToken('smooth_fade_in'); // 부드러운 페이드 인
}
```

### 🧩 2. 사용자 행동 → 시스템 피드백
사용자가 특정 버튼 (예: '분석하기') 을 누를 때, 시스템이 보여주는 응답은 단순한 로딩 화면이 아니라 **감정적 여정에 맞는 메시지**를 전달해야 합니다.

| 사용자 행동 | 기대되는 반응 (시스템) | 기술적 구현 포인트 |
| :--- | :--- | :--- |
| 데이터 입력 후 분석 요청 | "분석 중입니다..." → "문제를 찾았습니다." | 로딩 스피너는 경고 아이콘으로, 결과물은 파랑 배경에 하이라이트 |
| 솔루션 제안 화면 클릭 | "해결책을 확인해보세요" → "신청 완료!" | 모션 토큰 `slide_up` 과 함께, 성공 메세지가 떠오르는 애니메이션 |

### 🧩 3. 타이포그래피 및 레이아웃 규칙
- **타이포그래피:** 개발팀에서 UI 컴포넌트를 만들 때, 디자인 시스템에 정의된 폰트 스타일 (예: Bold 가독성, 중간 대비) 을 코드 변수로 활용하세요.
- **레이아웃:** Pain → Solution 의 흐름을 시각적으로 강조하기 위해, **스크롤 시점에 따라 배경색상이 점진적으로 변하는 패럴랙스 효과**를 구현할 수 있습니다.
