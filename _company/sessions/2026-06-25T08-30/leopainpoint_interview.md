# 📋 BDS 소상권플렛폼 - 잠재 고객 Pain Point/WTP 인터뷰 프로세스 (v1.0)
**작성일:** 2026-06-25  
**책임자:** 레오 (Head of YouTube)  
**목표:** 초기 베타 사용자의 진짜 고민 (Pain Point) 과 지불 의사 (WTP) 를 숫자로 측정하여 플랫폼 MVP 검증

## 1. 인터뷰 대상 선정 및 연락
- **대상:** 골목상권 소매점, 식당, 서비스업 종사자 중 매출 불안정성을 호소하는 사장님 5 명
- **선정 기준:** 기존 BDS 소상권플렛폼 SNS 페이지에서 'AI 리스크', '매출 예측' 관련 콘텐츠를 적극적으로 공유한 사용자 (추정 100 명 중 상위 5 명)
- **연락 방법:** 메신저/이메일 초안 발송

**[초안: 인터뷰 요청 메시지]**  
"사장님, 안녕하세요. 최근 BDS 소상권플렛폼에서 제공하는 AI 리스크 예측 시스템의 초기 베타 테스트를 위해, 실제 운영 데이터를 공유해달라고 제안드립니다. 15 분만 투자하시면, 사장님의 가게가 겪는 진짜 고민을 우리가 직접 해결하는 데 기여할 수 있습니다. 자세한 내용은 아래 링크 (TBD) 를 확인해주세요."

## 2. 인터뷰 질문지 (Interview Script)
**[Pain Point 측정]**  
- Q1: "현재 매출 예측을 위해 가장 많이 사용하는 도구/방식은 무엇인가요? (예: 엑셀, 손수 계산, 직감)"  
- Q2: "가장 불안한 순간은 언제ですか? (예: 월 말, 재고盘点 시, 경쟁사 개점 시)  
- Q3: "고객이 어떤 경로로 가게에 방문할지 예측하기 어려운 점은 무엇인가요?"

**[WTP 측정]**  
- Q4: "AI 기반 매출 예측 시스템 도입을 위해 월 5 만 원~10 만 원 정도의 비용을 지불한다면, 어떤 기능을 가장 우선시하고 싶나요? (예: 재고 관리 자동화, 고객 유입 경로 분석)"  
- Q5: "이 시스템이 당신의 가게 매출을 2% 증가시킨다면, 한 달에 몇 만 원의 수익으로 상환 가능한가요?"

## 3. 데이터 수집 스키마 및 저장소
**[스키마 정의]**  
```sql
CREATE TABLE interview_response (
    id SERIAL PRIMARY KEY,
    respondent_id VARCHAR(10),
    pain_point TEXT,
    wtp_amount DECIMAL(10,2),
    feature_preference TEXT,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**[ETL 파이프라인 기획]**  
- **입력:** 메신저/이메일 응답 → 텍스트 파일 저장 (JSONL)  
- **처리:** 파이썬 스크립트로 정제 및 DB 로 복사  
- **출력:** `sessions/2026-06-25T08-30/interview_data_v1.jsonl`
