### 1. 실패 원인 잠재적 후보군 (Hypothesis Generation)

테스트 실패는 보통 다음 세 가지 지점에서 발생합니다. 저는 이 세 가지 영역에 집중하여 예상되는 문제점을 목록화했습니다.

1.  **데이터 계약 불일치 (Schema Mismatch):**
    *   `data_schema.ts`와 백엔드(`routers/local_connection_router.py`) 또는 계산 로직(`roiCalculator.ts`) 간의 데이터 필드 타입, 필수 여부, 혹은 구조가 일치하지 않을 때 발생합니다. (가장 흔한 원인)
2.  **API 연결 실패 (Integration Failure):**
    *   `web_preview.py`가 프론트엔드에서 요청하는 API 엔드포인트나 데이터 형식에 대해 백엔드가 예상치 못한 에러(500 Internal Server Error 등)를 반환하거나, 데이터 파이프라인(`data_schema.ts` → DB/API)에서 누락이나 잘못된 변환이 발생했을 때 발생합니다.
3.  **비즈니스 로직 오류 (Logic Flaw):**
    *   `roiCalculator.ts`와 같은 핵심 비즈니스 로직 내에서 특정 입력값(예: 0으로 나누기, 잘못된 조건 분기)에 대해 예외 처리 없이 시스템이 붕괴되거나 논리적으로 잘못된 결과를 반환할 때 발생합니다.

### 2. 시스템 무결성 확보를 위한 재수립 로드맵 (Recovery Roadmap)

실패 원인을 특정하기 전에, **시스템 안정성을 선행**하는 방향으로 테스트 인프라와 검증 단계를 강화하는 로드맵을 수립하겠습니다. 이 로드맵은 '테스트 코드 자동화 프레임워크 구축' 목표에 직접적으로 기여합니다.

#### Phase 1: 진단 및 격리 (Diagnosis & Isolation)
*   **목표:** 실패 지점을 최소화하고, 테스트 환경의 독립성을 확보한다.
*   **액션 1. 로그 수집 표준화:** `lint_test.py`와 `web_preview.py` 실행 시 발생하는 모든 **풀 스택 로그(서버 측 응답 코드, 데이터베이스 트랜잭션 로그, 프론트엔드 에러)를 하나의 통합된 형식으로 기록하는 모듈을 추가**합니다.
*   **액션 2. Mocking 강화:** 실제 API 호출 대신, `web_preview.py`에서 사용할 수 있는 **모든 외부 API 응답(특히 데이터베이스 조회 결과)**에 대해 완벽하게 제어할 수 있는 Mocking 레이어를 구축하여 테스트를 격리합니다.

#### Phase 2: 계약 기반 검증 강화 (Contract-Based Validation)
*   **목표:** 데이터 흐름의 '계약'이 깨지지 않았는지 확인한다.
*   **액션 3. 스키마 레벨 검증 자동화:** `data_schema.ts`에 정의된 JSON Schema를 기반으로, 백엔드 API 응답과 프론트엔드가 수신한 데이터가 **항상 이 스키마를 준수하는지 확인하는 중간 레이어 검증 함수**를 구현합니다.
*   **액션 4. ROI 로직 단위 테스트 심화:** `roiCalculator.ts`의 모든 경계값(Edge Cases) 및 예외 처리 시나리오에 대한 **명시적인 단위 테스트 케이스**를 추가하여, 비즈니스 로직 자체의 무결성을 100% 보장합니다.

#### Phase 3: E2E 통합 검증 자동화 (Full E2E Automation)
*   **목표:** 실제 사용자 여정(User Journey)을 시뮬레이션하여 시스템 전체의 안정성을 최종 확인한다.
*   **액션 5. 통합 테스트 스위트 확장:** 기존 `SIT_Test_Suite_v1.0`에 더해, **'손실 최소화' 프레임워크 기반의 사용자 행동 시나리오(예: 특정 데이터 입력 후 경고 메시지 확인)를 포함하는 E2E 테스트 케이스**를 추가하고 자동화합니다.
*   **액션 6. 피드백 루프 구축:** 실패 발생 시, **자동으로 Phase 1에서 수집한 통합 로그를 기반으로 에러의 근본 원인(Schema 오류인지, 로직 오류인지)을 추론**하고, 해당 원인에 맞는 수정 사항을 자동으로 제안하는 자가 검증 루프를 완성합니다.
