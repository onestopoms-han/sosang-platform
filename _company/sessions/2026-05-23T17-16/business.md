# 🚀 MVP 출시 준비: 기술-비주얼 검증 보고서 (V1.0)

## 🎯 목표 정의: '가치 증명' 경험의 구현
본 단계의 핵심 목표는, 소상공인이 플랫폼을 이용하며 느끼는 **‘AI 기반 리스크 제거’**라는 가치를 디자인적으로 극대화하고, 이 과정에서 발생하는 모든 데이터를 기술적으로 추적하여 KPI 대시보드에 실시간으로 반영하는 것입니다.

| 측면 | 구현 내용 (Synthesis) | 목적/기대 효과 |
| :--- | :--- | :--- |
| **비즈니스 목표** | 소상공인 어려움 파악 $\rightarrow$ AI 진단 $\rightarrow$ 유료 솔루션(Premium Conversion) 구매. | 단순 정보 제공을 넘어, *필요한 상품*을 판매하는 구조 확립 (ROI 확보). |
| **사용자 경험 (UX)** | Pain Point 발견(진단) $\xrightarrow{\text{Deep Blue}}$ → 해결책 제시 및 가치 확인 $\xrightarrow{\text{Growth Green}}$ → 결제 및 전환. | 사용자가 스스로 '돈을 지불할 만한 가치가 있다'고 느끼게 하는 **가치 증명 플로우** 완성. |
| **기술 구조 (API)** | 진단 시나리오 데이터 수집 $\rightarrow$ 유료 서비스 계산 로직 실행 $\rightarrow$ 결제 게이트웨이 연동. | 모든 상호작용을 추적 가능한 데이터(매출, 사용 시간, 진단 항목)로 변환하여 KPI의 근거 확보. |

## 🛠️ 기술-비주얼 통합 매핑 (The Connection Points)
가장 중요한 검증 지점은 **'데이터가 디자인 요소에 어떻게 영향을 미치는가'** 입니다. 현재까지의 산출물을 연결하여 필수적으로 확인해야 할 세 가지 상호작용 포인트를 정의합니다.

### 1. 데이터 입력 $\rightarrow$ 핵심 메시지 생성 (진단 단계)
*   **기술 요구사항:** 소상공인이 직접 데이터를 입력하는 과정(API 요청).
*   **디자인 반영:** `KPI_Metric_Card`의 'A. 핵심 지표'에 표시될 **Pain Point 점수 또는 위험도 지수**가 이 데이터로부터 도출되어야 합니다. (기술 결과물 $\rightarrow$ 시각적 경고/강조)
*   **검증 항목:** 입력 오류 처리, 진단 로직의 투명한 피드백(왜 이 점수가 나왔는지).

### 2. 결제 플로우와 KPI 연결 (전환 단계)
*   **기술 요구사항:** 유료 서비스 구매 완료 API 호출 및 트랜잭션 로그 기록.
*   **디자인 반영:** `KPI_Metric_Card`의 'B. 상태/변화율'이 **실시간으로 누적 매출액과 전월 대비 성장률**을 표시해야 합니다. (기술 데이터 $\rightarrow$ 시각적 성과 증명)
*   **검증 항목:** 결제 성공/실패 여부에 따른 사용자 인터페이스(UI) 피드백, 재시도 로직의 안정성.

### 3. 대시보드의 정보 계층화 (관리자 페이지)
*   **기술 요구사항:** 다양한 소스(API, CRM 등)에서 들어오는 데이터를 통합하여 정제하는 백엔드 데이터 레이크 구축.
*   **디자인 반영:** `KPI_Metric_Card`에 표시되는 모든 지표는 단순히 숫자가 아니라, **시간의 흐름(D. 추이 그래프)**을 보여주는 시각적 맥락 위에 존재해야 합니다. (데이터 통합 $\rightarrow$ 스토리텔링)
