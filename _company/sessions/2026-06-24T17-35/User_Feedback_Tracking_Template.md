# 📄 User Feedback Tracking Template (BDS 소상공인 인터뷰용)

> **목적:** 단순한 경험 공유를 넘어, BDS 플랫폼의 핵심 기능을 정의할 정량적이고 분류 가능한 데이터 포인트를 수집한다.
> **사용 원칙:** 모든 피드백은 '감정(Emotion)'과 '행동/수치(Metric)'로 분리하여 기록한다.

---

## 🎯 섹션 I: 기본 정보 및 맥락 (Metadata)
| 항목 | 내용 | 비고 |
| :--- | :--- | :--- |
| **인터뷰 대상자** | 이름 / 업종 / 사업 규모 (직원 수, 연 매출대) | |
| **진행 날짜/시간** | YYYY-MM-DD / HH:MM | |
| **핵심 Pain Point 키워드** | 3~5개 요약 (예: 재고 관리, 현금 흐름 예측, 인력 수급) | 인터뷰 중 가장 많이 언급된 단어 기록 |

## 🔍 섹션 II: 핵심 문제 분석 및 측정 (Pain & Metric)
*이 부분이 플랫폼의 '진단(Diagnosis)' 데이터가 됩니다.*

| 질문 영역 | 인터뷰 내용 요약 (Quotes/Storytelling) | Pain Point 유형 분류 | 심각도 (Severity, 1-5점) | 현재 불편함으로 인한 비용 추정 |
| :--- | :--- | :--- | :--- | :--- |
| **재무 불안정** (매출 예측 등) | "[인터뷰 내용 구체적으로 기입]" | A. 현금 흐름 / B. 매출 변동성 / C. 원가 관리 | [점수] | (예: 월 50만 원의 기회비용 손실) |
| **운영 비효율** (재고/인력 등) | "[인터뷰 내용 구체적으로 기입]" | A. 시스템 미흡 / B. 인력 관리 어려움 / C. 공급망 문제 | [점수] | (예: 주당 5시간의 노동 시간 손실) |
| **시장 적응** (트렌드/경쟁사 등) | "[인터뷰 내용 구체적으로 기입]" | A. 트렌드 예측 불가 / B. 경쟁 우위 확보 어려움 | [점수] | (예: 신규 마케팅 비용 낭비) |

## ✨ 섹션 III: 해결책 및 니즈 분석 (Solution & WTP)
*이 부분이 플랫폼의 '기능(Feature)'과 '지불 의사(WTP)'가 됩니다.*

| 질문 영역 | 인터뷰 대상자의 반응/의견 | 가설적 필요 기능 정의 (Need) | 기대 효과 (Expected Benefit) | 지불 의사 여부 (Willing to Pay?) |
| :--- | :--- | :--- | :--- | :--- |
| **데이터 분석 요구** | "만약 이런 데이터가 있다면 좋겠다." | [예: 예측 대시보드 기능] | [예: 3개월치 비용 절감 예상] | (Y/N) / 근거: |
| **플랫폼 사용 경험** | (기존에 사용하던 솔루션의 문제점) | [예: 통합 관리 시스템 필요] | [예: 업무 시간 10시간 단축] | (Y/N) / 근거: |
