# 🚀 프론트엔드 컴포넌트 개발 최종 지침 (Trust & PainGauge)

## 1. 디자인 시스템 요약 (V3.0 핵심 사양)

모든 컴포넌트는 다음의 기본 규칙을 준수해야 합니다.

| 요소 | 정의 | 세부 사항 |
| :--- | :--- | :--- |
| **Primary Color** | 브랜드 메인 컬러 | `#1E88E5` (Blue 600) |
| **Neutral Background** | 기본 배경 색상 | `#F9F9F9` (Light Gray) |
| **Typography Scale** | 타이포그래피 계층 | H1(32px), Body(14px), Caption(12px) |
| **Spacing Unit** | 간격 단위 | 8px 기반 그리드 시스템 적용 |

## 2. 상태별 색상 및 스타일 매핑 (Backend $\rightarrow$ Frontend Visuals)

백엔드에서 반환되는 `trust`, `warning`, `critical` 상태에 따라 컴포넌트의 시각적 요소(배경, 테두리, 텍스트)는 아래와 같이 **강제** 적용되어야 합니다.

| 상태 (`status`) | 역할 (Role) | 배경 색상 (Background Color) | 강조 색상 (Accent Color) | 텍스트 색상 (Text Color) | 경고 표시 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **trust** | 신뢰 (Trust) | `#E8F5E9` (Light Green) | `#4CAF50` (Green 600) | `#1B5E20` (Dark Green) | ✅ 안정 |
| **warning** | 주의 (Warning) | `#FFFDE7` (Light Yellow) | `#FFC107` (Amber 600) | `#F57C00` (Deep Orange) | ⚠️ 주의 |
| **critical** | 위험 (Critical) | `#FFEBEE` (Light Red) | `#F44336` (Red 600) | `#B71C1C` (Dark Red) | 🚨 위험 |

## 3. 컴포넌트별 구체적인 구현 지침

### A. Trust Widget 지침

**목표:** 사용자의 현재 신뢰도를 명확하게 시각화하고 행동을 유도합니다.

1.  **레이아웃:** 중앙 정렬된 카드 형태.
2.  **신뢰도 표시 (Score Display):** 점수(예: 85/100)는 가장 큰 폰트(H1, 36px 이상)로 표시하며, 배경색에 따라 강조 색상을 적용합니다.
    *   `trust`: 배경 `#E8F5E9`, 테두리 `2px solid #4CAF50`
    *   `warning`: 배경 `#FFFDE7`, 테두리 `2px solid #FFC107`
    *   `critical`: 배경 `#FFEBEE`, 테두리 `2px solid #F44336`
3.  **메시지 표시:** 상태에 따른 메시지는 굵은 폰트(Body, 16px)로 표시하며, 강조 색상을 사용하여 긴급성을 전달합니다. (예: Critical 시 메시지는 붉은색 텍스트로 명확히 구분)

### B. PainGauge 지침

**목표:** 소상공인이 겪는 어려움의 정도를 직관적으로 측정하고 심각도를 표현합니다.

1.  **시각화 방식:** 원형 게이지(Donut Chart 또는 Progress Bar) 형태로 구현하며, 채워지는 색상은 **상태별 강조 색상**을 따릅니다.
2.  **색상 매핑:**
    *   `trust`: 녹색 계열로 점진적인 채움 (0% $\rightarrow$ 100%)
    *   `warning`: 노란색/주황색으로 경고 영역 표시
    *   `critical`: 빨간색으로 위험 영역 표시
3.  **중앙 값:** 게이지 중앙에는 현재 점수(Score)를 명확히 표시하며, 상태에 따라 폰트 색상도 위 표의 '텍스트 색상' 규칙을 따릅니다.

## 4. 디자인 가이드라인 요약 (Designer's Takeaway)

프론트엔드 개발팀은 이 지침을 **디자인 시스템 컴포넌트 라이브러리**에 직접 반영하여, 백엔드 API 응답에 따라 동적으로 색상과 레이아웃이 변경되는 **반응형 위기 인지 프레임워크**를 구축해야 합니다.

*   **핵심:** 모든 시각적 변화는 백엔드의 `status` 필드에 100% 의존해야 하며, 하드코딩된 색상은 허용되지 않습니다.
