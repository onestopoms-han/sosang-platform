*   **핵심 가치:** 대한민국 소상공인의 성공을 위한 신뢰 구축 및 불안감 해소.
*   **KPI:** PainGauge 지수 $\le 0.3$ 달성 시, 사용자에게 '안전마진 확보'의 감성적 확신 전달.
*   **요구사항:** 모든 상태 변화(Loading, Error, Success)에서 **명확한 피드백과 공감적 애니메이션**을 제공하여 신뢰를 높인다.

## 🧩 Part I: PainGauge 컴포넌트 (핵심 위젯) 명세

### 1. 기본 상태 및 시각화
*   **UI 요소:** 원형 게이지, 현재 지수 표시 (`X / 10`), 안전마진 임계점 라인.
*   **컬러 팔레트 (감성적 연관):**
    *   안전 영역 ($\le 0.3$): `#4CAF50` (Deep Green - 안정감)
    *   주의 영역 ($> 0.3 \sim 0.7$): `#FFC107` (Amber Yellow - 경고/관심 유도)
    *   위험 영역 ($> 0.7$): `#F44336` (Deep Red - 즉각적 위험)

### 2. 인터랙션 및 애니메이션 로직 (QA 승인 항목)
| 상태 | 트리거 | 애니메이션 명세 | 기술적 요구사항 | 감성적 목표 |
| :--- | :--- | :--- | :--- | :--- |
| **Initial Load** | 컴포넌트 마운트 | 게이지가 0%에서 시작하여, 초기 데이터 값에 맞춰 부드럽게 채워짐 (Easing: `ease-out` over 500ms). | API 응답 지연 시 Skeleton Loader를 먼저 보여주고, 로딩 완료 후 이 애니메이션을 실행해야 함. | 시스템이 '작동'하고 있음을 즉시 인지시키고 기다림의 지루함을 줄임. |
| **Value Update** | 백엔드 데이터 수신/변경 | 게이지가 이전 값에서 새 값으로 이동할 때, 부드러운 트랜지션(Transition)과 함께 '점진적 변화'를 보여줘야 함. (속도: 400ms). | Value Change 이벤트 발생 시만 트리거. 단순히 값을 덮어쓰는 방식 금지. | 사용자가 자신의 상태 변화를 **직관적으로 체감**하게 하여 행동의 필요성을 인지시킴. |
| **Safety Achieved** | PainGauge $\le 0.3$ 달성 | 게이지가 안전 영역에 진입하는 순간, 미세한 '펄스 효과'(Pulse Effect)와 함께 Green/Gold 색상으로 전환되며 시각적 보상을 제공해야 함. | `PainGaugeValue` Prop이 특정 임계값 이하로 떨어질 때만 트리거. 애니메이션 강도(Scale: 1.05x $\rightarrow$ 1.0x)를 명확히 정의. | 심리적 안도감과 성취감을 부여하며, 플랫폼 사용에 대한 동기부여 제공. |

## 🚧 Part II: 공통 컴포넌트 및 Fallback 로직 (최종 검증)

### 1. Skeleton Loader 시스템
*   **목표:** API 응답 지연/오류 시나리오에서 '시스템 정지' 느낌을 완벽히 제거한다.
*   **애니메이션 명세:** 단순한 회색 블록이 아닌, **가벼운 깜빡임(Subtle Pulse)** 효과를 적용해야 함. (CSS `animation: pulse 1.5s infinite;` 활용).
*   **Fallback 전략:** 데이터가 로딩되는 영역 전체에 Skeleton을 적용하여 사용자가 어느 부분이 채워질지 예측하게 해야 한다.

### 2. Error State & Empty State 처리 (감성적 Fallback)
| 상태 | 발생 원인 | UI/UX 명세 | 필수 카피라이팅 가이드라인 |
| :--- | :--- | :--- | :--- |
| **API Error** | 백엔드 통신 오류, Rate Limit 초과 등 | 표준적인 빨간색 경고창 대신, '안전마진' 관점에서 접근하는 부드러운 토스트(Toast) 알림을 사용. | "현재 시스템이 불안정합니다. 잠시 후 다시 시도해주세요." (기술적 문제보다 **사용자에게 공감**하는 톤). |
| **Empty State** | 조회할 데이터 없음 (예: 작성된 스토리가 없을 때) | 단순히 '없음'이라고 표시하지 않고, 이 공간을 채우기 위한 **명확한 다음 액션 버튼(CTA)**과 예시 이미지를 제공해야 함. | "아직 스토리가 없으신가요? 첫 번째 성공 사례를 기록하고 안전마진을 높여보세요!" (미래지향적 동기 부여). |
