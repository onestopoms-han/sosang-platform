## 🎯 워크숍 핵심 메시지 (Key Messages for Sign-off Workshop)
이 메시지를 발표 자료의 모든 페이지에 녹여야 합니다.

1. **가치 명제:** BDS는 단순 결제 도구가 아니라 **'상생의 성장 안전망'**입니다. 기술적 리스크를 인간의 감정 변화로 연결합니다.  
2. **기술 신뢰도:** API 설계안 (Mapping Table) 과 디자인 시스템 사양은 '투명성'을 증명하는 핵심 자산입니다. 불일치를 없애고 일관성을 확보했습니다.  
3. **시각적 경험:** Trust Widget 과 PainGauge 의 시각적 흐름이 사용자의 '불안감 → 안정감' 전환을 시각화합니다. 캐릭터 'The Growth Partner'가 그 연결고리입니다.

## 📖 통합 발표 스토리라인 초안 (Storyline Draft)

### **[1] 오프닝: 현재 소상공인의 진짜 고민**
- **후크:** "데이터는 있지만, 행동은 없습니다. 그 간극을 메우는 도구가 필요합니다."  
- **스토리:** PainGauge 과 Trust Widget 의 데이터를 시각화하여 '불안감'과 '기대감'의 경로를 보여줍니다. 기술이 감정과 연결되는 순간을 묘사합니다.  
- **CTA:** "이제부터, 우리는 데이터와 감정을 동시에 읽어냅니다."  

### **[2] 솔루션 소개: 성장 안전망 BDS**
- **가치 제안:** "단순 기능이 아닌, 사용자의 '행동'을 예측하고 '안전'을 제공합니다."  
- **스토리보드:** 
  - API 설계안 (Mapping Table) 을 통해 기술적 리스크를 해결했습니다.  
  - 디자인 시스템 사양을 통해 시각적 일관성을 확보했습니다.  
  - 두 결과가 만나 '고통 → 안전망'이라는 가치로 구체화됩니다.  
- **시각적 연결:** 코다리의 Mapping Table 과 디자이너의 디자인 시스템이 한 화면에 겹쳐지는 애니메이션 아이디어를 제시합니다.  

### **[3] 핵심 기능 증명: 기술 × 시각**
- **PainGauge:** 사용자의 '불안감'을 수치로 측정하고, 기술적으로 안정된 API 로 해결책을 제안합니다.  
- **Trust Widget:** 신뢰도를 시각적 요소 (색상, 애니메이션) 로 전달하여 심리적 안정감을 줍니다.  
- **The Growth Partner:** 캐릭터가 두 기능을 연결하며 사용자를 안내합니다.  

### **[4] Q&A 및 Sign-off 전략**
- **핵심 메시지 재확인:** "기술적 안정성과 인간적인 감정이 완벽하게 결합된 플랫폼입니다."
