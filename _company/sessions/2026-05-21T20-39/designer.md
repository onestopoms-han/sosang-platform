## 📐 최종 UI/UX 흐름 및 컴포넌트 구조 확정 보고서

본 문서는 `AI_Diagnosis_Final_Design_Spec.md`의 내용을 기반으로, AI 진단 리포팅 페이지의 시각적 구현에 필요한 모든 흐름과 구조를 개발팀에게 전달하기 위한 최종 명세입니다.

### 1. 전체 사용자 여정 지도 (User Journey Map)

사용자가 AI 진단 결과 보고서를 확인하고 다음 행동(Action Prescription 실행 또는 유료 전환)으로 이어지는 단계를 시각화합니다.

| 단계 (Step) | 사용자 목표 (Goal) | 화면/요소 (Screen/Element) | 핵심 상호작용 (Interaction Focus) | 성공 지표 (Success Metric) |
| :---: | :---: | :---: | :---: | :---: |
| **1** | 진단 시작 및 데이터 입력 | `DiagnosisForm` (입력 폼) | 필수 정보 입력 및 진행 상태 확인 | 폼 제출 성공 |
| **2** | 결과 대기 | 로딩 스피너 / 진행 바 | 시스템 처리 중임을 명확히 인지 | 로딩 완료 시점 |
| **3** | **결과 확인 (핵심)** | `DiagnosisReportView` (리포트 페이지) | AI 진단 결과, Action Prescription, 스토리텔링 요소 시각화 | 보고서 내용 이해 및 공감 |
| **4** | 실행 계획 수용 | Action Prescription 카드 | 각 제안에 대한 구체적인 설명 확인 | 원하는 액션 선택/인식 |
| **5** | 심화 학습/결제 유도 | Upsell/Action Flow 섹션 | 다음 단계(고급 컨설팅, 유료 리포트) 제시 | 전환 행동 시작 (CTA 클릭) |

### 2. 핵심 컴포넌트 구조 정의 (Component Hierarchy)

페이지의 기능적 분리와 재사용성을 극대화하기 위해 필요한 컴포넌트 계층을 정의합니다.

**A. 최상위 컨테이너:** `DiagnosisPage`
*   **역할:** 전체 리포팅 페이지의 레이아웃 관리 및 상태(로딩, 에러) 처리.
*   **내부 구성 요소:**
    1.  `Header`: 페이지 제목 및 주요 CTA 위치.
    2.  `ReportContainer`: 실제 진단 결과와 Action Prescription을 담는 메인 영역.
    3.  `ActionFlowSection`: 실행 계획 단계별 인터랙션을 관리하는 섹션.
    4.  `Footer`: 보조 정보 및 참고 자료 링크.

**B. 핵심 콘텐츠 컴포넌트:**

1.  **`DiagnosisForm` (입력 모듈):**
    *   **목표:** 사용자로부터 필요한 데이터(소상공인 상황, 목표 등)를 수집.
    *   **구조:** 입력 필드 그룹화 (`InputGroup`) -> 동적 필드 추가/제거 기능 명시.
2.  **`DiagnosisReportView` (결과 모듈):**
    *   **목표:** AI 진단 결과를 가장 효과적으로 시각화하여 제공.
    *   **구조:**
        *   `SummaryCard`: 전체 진단 점수 및 핵심 요약 (KPI 중심).
        *   `InsightBreakdown`: 세부 분석 항목별 결과 (데이터 기반 설명).
        *   `ActionPrescriptionList`: 가장 중요한 섹션. 각 Action에 대해 명확한 설명과 실행 단계 제공.
3.  **`ActionCard` (상호작용 모듈):**
    *   **목표:** 사용자가 특정 조언을 선택하고 다음 행동으로 이어지게 유도.
    *   **구조:** 제목, 요약 설명, **'선택하기(Select)' 버튼**, 실행에 따른 예상 결과 표시.

### 3. 디자인 시스템 명세 (Visual System Specification)

개발팀이 일관된 톤앤매너를 적용하도록 구체적인 디자인 변수를 정의합니다.

*   **컬러 팔레트:**
    *   Primary (BDS 브랜드): `#4A90E2` (신뢰, 기술)
    *   Secondary (액션/CTA): `#FFC107` (주의, 기회 포착)
    *   Background: `#F9F9F9` (깔끔한 학습 환경 제공)
    *   Text: `#333333` (가독성 극대화)
    *   Success/Warning: `#28a745` / `#dc3545`
*   **타이포그래피:**
    *   Heading 1 (H1): `Pretendard Bold` (40px 이상) - 강력한 메시지 전달.
    *   Body Text: `Noto Sans KR` (16px, Line-height 1.6) - 장문의 지식 콘텐츠 가독성 확보.
*   **레이아웃 원칙:**
    *   모든 정보는 **'문제(Pain Point) → 진단(Diagnosis) → 해결책(Action)'**의 흐름을 따르도록 계층적으로 배치한다. (Vertical Flow Priority)
    *   핵심 데이터(KPI, Action Title)는 시각적 위계를 최상위로 설정하고, 세부 설명은 확장 가능한 영역에 배치한다.
