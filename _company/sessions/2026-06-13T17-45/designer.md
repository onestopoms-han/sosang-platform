# 🎨 BDS 소상공인 플랫폼 - Trust Widget & PainGauge 최종 UI/UX 스펙 (V4.0)

**적용 버전:** V4.0 (최종 확정본)
**대상 화면:** Pilot 프로그램 온보딩 시나리오 핵심 진단 페이지
**핵심 목표:** 사용자에게 '위험 인지'를 통한 심리적 동의(Buy-in)를 얻고, 데이터 기반의 필요성(Need)을 각인시켜 플랫폼 사용으로 전환 유도.

## 📌 1. 전반적인 디자인 원칙 및 가이드라인 (System Guidelines)

| 항목 | 내용 | 비고 |
| :--- | :--- | :--- |
| **디자인 시스템** | V3.0 준수 (Deep Blue, Growth Green 중심) | 모든 컴포넌트는 폰트 스케일과 컬러 팔레트를 엄격히 지킨다. |
| **UX 흐름 원칙** | **'문제 제시 $\rightarrow$ 데이터 입증 $\rightarrow$ 해결책 시사'** 순서를 절대적으로 유지한다. | 사용자가 '나도 저 문제를 겪고 있다'는 공감대를 느끼도록 설계. |
| **애니메이션** | 진단 결과(Gauge)가 로딩될 때, 마치 측정기가 돌아가듯 부드러운 스캘링/페이드 인 효과를 적용하여 신뢰성을 높인다. | 정적 이미지 사용 지양. 인터랙션을 반드시 부여한다. |
| **데이터 출처 명시** | 모든 통계 및 수치 옆에는 `(출처: 지역 상권 데이터 기반)` 등의 작은 텍스트로 근거를 제시해야 한다. | 신뢰도 확보의 핵심 요소. |

## 📐 2. 컴포넌트별 상세 스펙 (Component Specs)

### A. Trust Widget (신뢰 위젯) - [위험 인지 단계]

**목적:** 사용자에게 '문제가 나 혼자만의 문제가 아니다'라는 집단적인 공감대와 위험감을 동시에 전달한다.
**배치:** 온보딩 페이지 진입 직후, 가장 시선이 집중되는 최상단 또는 메인 이미지 위에 오버레이.

| 요소 | 상세 스펙 및 동작 (Action) | 디자인 지침 (Design Notes) |
| :--- | :--- | :--- |
| **메시지 구성** | 1. 공감 질문: "혹시 이런 고민을 하고 계신가요?" 2. 핵심 통계 수치: `[X]%의 소상공인이 이 문제에 직면했습니다.` 3. 위험 경고: (Deep Blue 배경 + Growth Green 강조) "데이터 기반 진단이 필요합니다." | **폰트:** Headline 크기 사용. 숫자는 가장 크게, 다른 텍스트보다 대비되게 처리한다. |
| **인터랙션** | 스크롤 다운 시, 통계 수치가 마치 카운팅(Counting Up) 되는 애니메이션을 보여주며 등장한다. | 정적인 이미지는 금지. 사용자에게 '살아있는' 데이터를 보여준다. |
| **데이터 추적 로직 연동** | 사용자가 이 위젯의 내용을 읽고 스크롤을 내리는 행위는 `event_id: trust_widget_view`로 기록되어야 한다. |

### B. PainGauge (고통 진단 게이지) - [문제 구체화 단계]

**목적:** 사용자 개인의 비즈니스 데이터를 분석하여, '어떤 부분'에서 '얼마나 큰 문제'가 있는지 직관적으로 시각화한다.
**배치:** Trust Widget 이후, 메인 컨텐츠 영역 중앙 (사용자가 자신의 데이터를 직접 보는 공간).

| 요소 | 상세 스펙 및 동작 (Action) | 디자인 지침 (Design Notes) |
| :--- | :--- | :--- |
| **구조 (3단 비교)** | 1. Ideal State (최적 상태): 목표 값 / 2. Current State (현재 상태): 사용자의 데이터 기반 측정 값 / 3. Gap Analysis (격차 분석): 두 값의 차이와 위험도 표시. | 게이지 바(Gauge Bar)를 사용하여 직관적인 시각화를 최우선으로 한다. |
| **위험 신호 시각화** | 현재 상태가 Ideal State 대비 일정 임계치 이하일 경우, 해당 데이터 블록 전체에 `WARNING` (약한 붉은 계열 또는 Deep Blue의 어두운 변주) 배경을 적용한다. | 위험도가 높을수록 경고 색상이 강해지며, 텍스트와 함께 '개선 필요' 문구를 표시한다. |
| **데이터 추적 로직 연동** | 사용자가 각 게이지 항목(예: '재방문율', '객단가')에 마우스를 올리거나 클릭하면, 해당 데이터의 상세 정의 및 의미를 팝오버로 보여주고, 이 행위는 `event_id: paingauge_drilldown_{metric}`으로 기록되어야 한다. |

## 🚀 3. 최종 사용자 여정 플로우 (Flow Integration)

**[단계] $\rightarrow$ [사용자 감정] $\rightarrow$ [디자인 컴포넌트] $\rightarrow$ [다음 행동 유도]**

1.  **(Start)**: 페이지 진입 $\rightarrow$ **호기심/불안감 인식** $\rightarrow$ **Trust Widget 활성화 (V4.0 스펙 준수)** $\rightarrow$ "데이터를 확인해 봅시다."
2.  **(Process 1)**: 데이터 입력 및 분석 요청 $\rightarrow$ **긴장감/관심 집중** $\rightarrow$ **PainGauge 로딩 애니메이션** (데이터 수집 과정을 시각화) $\rightarrow$ '위험 신호' 발견 직후.
3.  **(Process 2)**: 진단 결과 도출 $\rightarrow$ **문제 확인/공포감** $\rightarrow$ PainGauge의 `Gap Analysis`를 통해 구체적인 문제점 제시 (예: "오후 2시 방문객이 늘고 있지만, 프로모션 부재로 매출 기회를 놓치고 있습니다.")
4.  **(Action)**: 솔루션 제안으로 이동 유도 $\rightarrow$ **희망/해결 기대** $\rightarrow$ 플랫폼의 'AI 챗봇' 또는 '구체적 대안 리스트' 화면으로 자연스럽게 스크롤 전환 및 진입.
