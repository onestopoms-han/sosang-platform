### 🎯 이번 사이클의 핵심 작업: 오류 카드 컴포넌트 최종 확정
**목표:** 플랫폼 전체에서 발생하는 모든 종류의 실패 상황(결제 실패, API 통신 오류, 데이터 누락 등)에 대해 일관적이고 사용자 친화적인 **'Error Card Component'**를 디자인 시스템으로 확정합니다.

**[실행 계획]**

1.  **오류 카드 컴포넌트 스펙 정의:** 단순히 에러 메시지를 보여주는 것을 넘어, **원인 진단 → 안전한 해결책 제시 → 다음 행동 유도(CTA)**의 3단계 흐름을 갖도록 컴포넌트 구조를 설계합니다.
2.  **상태별 목업 제작 (A/B 테스트 준비):** 최소 세 가지 필수 오류 상태(① API 통신 오류, ② 결제 실패, ③ 데이터 누락)에 대한 시각적 목업과 해당 상태에서의 사용자 안내 텍스트 가이드라인을 작성합니다.
3.  **컴포넌트 통합 설계 명세서 업데이트:** 기존의 컴포넌트 가이드북(`Component_Integration_Guidebook_v1.0.md`) 내에 Error Card 섹션을 추가하고, 사용 예시와 상호작용 로직(Interaction Logic)을 구체화하여 최종본으로 완성합니다.

### Error Card Component Spec (v1.0) 주요 내용 구성
**목적:** 시스템 오류 발생 시 사용자의 불안감을 최소화하고, 문제를 인지시키며, 다음 단계를 명확히 제시하여 이탈을 방지하는 것이 핵심입니다.
**톤앤매너:** 경고(Warning)의 톤이지만, 패닉 상태가 아닌 '안전한 진단 및 해결'에 초점을 맞춥니다. (Deep Blue 계열 배경 사용 지양, 밝은 회색 또는 부드러운 주황/노랑 계열 강조)

**1. 컴포넌트 구조 및 레이아웃:**
*   **[제목/헤딩]:** 오류 발생 알림 (`H2` 태그 수준의 명확한 경고 메시지).
    *   예: "죄송합니다. 현재 시스템에 문제가 발생했습니다." (감정적 공감)
*   **[본문]:** 문제 원인 및 영향 범위 설명.
    *   예: "결제 과정에서 네트워크 연결이 불안정하여 거래가 완료되지 않았습니다. 잠시 후 다시 시도해 주세요." (기술적 사실 전달)
*   **[핵심 진단/Solution Box]:** 사용자가 취해야 할 가장 중요하고 즉각적인 행동을 명확히 제시하는 박스 형태의 영역. (이 부분이 가장 강조되어야 함.)
    *   예: "💡 해결 방법: 연결 상태를 확인하시거나, [재시도] 버튼을 눌러주세요."
*   **[액션 버튼]:** 사용자에게 선택권을 부여합니다. 최소 1개 이상의 CTA가 존재해야 합니다. (예: `[재시도하기]` / `[고객센터 문의]`)

**2. 필수 오류 상태별 목업 가이드라인:**

| 오류 타입 | 발생 시나리오 | 핵심 메시지 전략 | 권장 색상/강조점 |
| :--- | :--- | :--- | :--- |
| **① API 통신 오류 (Network)** | 외부 서비스(API)와의 연결 실패. | 원인: '연결 문제'로 포괄적 설명. 해결책: 네트워크 재확인 또는 잠시 후 시도. | 경고 노란색 (#FFC107) / "잠시만 기다려 주세요" 강조 |
| **② 결제 시스템 오류 (Payment)** | PG사 연결 실패, 카드 승인 거절 등. | 원인: '결제 환경 문제'로 구체화. 해결책: 결제 수단 변경 또는 관리자 문의 유도. | 경고 주황색 (#FF9800) / "다른 결제 방법을 시도해 보세요" 강조 |
| **③ 데이터 누락 오류 (Validation)** | 필수 입력 필드 미작성, 서버에서 데이터를 찾지 못함. | 원인: '사용자 입력' 또는 '시스템 데이터 확인 필요'. 해결책: 구체적인 빠진 정보 명시 및 재입력 유도. | 일반 회색/Deep Blue (#004D66) / "필수 항목을 다시 한번 확인해주세요." 강조 |

**3. 상호작용 로직 (Coder & Designer 협의 필수):**
*   **[재시도 버튼 클릭]:** 컴포넌트를 닫지 않고, 백그라운드에서 재요청 API를 실행하며 로딩 스피너(`Loading Spinner`)로 교체됩니다. 성공/실패 여부에 따라 Error Card가 즉시 리렌더링되어야 합니다.
*   **[고객센터 문의]:** 해당 오류 상황과 함께 접수할 수 있는 고유한 트래킹 ID (예: `ErrorID-20260603-A1B2`)를 사용자에게 제공합니다.
