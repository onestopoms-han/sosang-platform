## 1. 목표
전체 BDS 소상공인 플랫폼의 통일된 사용자 경험(UX)을 제공하며, 데이터 상태(Success/Loading/Error/Empty)에 관계없이 사용자가 신뢰하고 다음 행동을 취할 수 있도록 유도하는 공통 컴포넌트 가이드를 확정한다.

## 2. 적용 원칙 (The Core Principle)
*   **신뢰성 유지:** 어떤 상황에서도 플랫폼의 핵심 메시지("대한민국 소상공인의 성공 지원")를 시각적 요소로 은연중에 전달해야 한다.
*   **직관성 극대화:** 에러나 경고는 '실패'가 아닌, '다음 단계가 필요함'이라는 안내로 인식되어야 한다.
*   **일관된 감정 토큰 사용:** 모든 피드백은 핵심 Pain $\rightarrow$ Solution의 감정적 흐름을 역추적하여 디자인된다.

## 3. 컴포넌트별 상세 사양 (Component Specifications)

### A. Empty State (데이터 부재 상태)
*   **발생 원인:** 조회할 데이터 자체가 없을 때 (예: "해당 기간에 등록된 콘텐츠가 없습니다.")
*   **시각 디자인:** 단순히 빈 화면을 보여주는 것이 아니라, **"사용자가 해야 할 다음 행동(Call To Action)"**을 제안해야 한다.
    *   **레이아웃:** 중앙 정렬, 부드러운 아이콘 + 설명 텍스트 조합.
    *   **컬러 토큰:** 메인 배경색(`--color-bg-light`) 사용. 경고/위험 색상 사용 금지.
    *   **메시징 예시:** "아직 등록된 콘텐츠가 없어요. [콘텐츠 만들기] 버튼을 눌러 소상공인의 생생한 이야기를 시작해 보세요!"
    *   **핵심 액션:** 반드시 1차 행동 유도 버튼(Primary CTA)을 포함해야 한다.

### B. Loading State (데이터 로딩 상태)
*   **발생 원인:** API 호출 결과 데이터를 기다리는 동안.
*   **시각 디자인/모션 토큰:** 사용자에게 '기다림'이 아닌 **'처리 중임(Processing)'**이라는 느낌을 주어 지루함을 최소화해야 한다.
    *   **위젯:** 단순 스피너(Spinner) 사용 금지. 데이터의 종류를 추상적으로 표현하는 애니메이션 (예: 점진적인 그리드 패턴 이동, 또는 콘텐츠 카드가 마치 합쳐지는 듯한 모션).
    *   **모션 속도:** 너무 빠르거나 느리지 않도록, **0.8초 간격으로 부드럽게 변화하는 리듬감**을 유지한다.
    *   **사용자 안내:** "정보를 불러오는 중입니다...", 또는 "소상공인의 성공 스토리를 준비합니다..."와 같은 친근한 문구를 함께 표시하여 공백의 느낌을 메운다.

### C. Warning State (주의/경고 상태)
*   **발생 원인:** 시스템에는 문제가 없으나, 사용자가 주의해야 할 데이터 불일치나 미완료 부분이 있을 때. (예: "이 콘텐츠는 아직 승인 절차가 완료되지 않았습니다.")
*   **시각 디자인:** 위험(Danger)과 구별되는 **'노란색 계열의 부드러운 색상 변화'**를 사용한다. 🔴 대신 🟡을 사용.
    *   **아이콘/배지:** 경고 삼각 아이콘 ($\triangle$) 사용 후, 반드시 설명 문구와 함께 배치하여 오해를 방지한다.
    *   **텍스트 토큰:** "주의사항", "검토 필요" 등의 명확한 제목을 사용하고, 하단에 **'왜 이것이 주의가 필요한지'**에 대한 간결한 이유(Reason)를 병기해야 한다.

### D. Error State (치명적 오류 상태)
*   **발생 원인:** API 호출 실패, 네트워크 연결 끊김 등 시스템적인 문제 발생 시. (예: "서버 연결에 문제가 발생했습니다.")
*   **시각 디자인/모션 토큰:** 사용자가 패닉하지 않도록 **차분하고 구조화된 방식**으로 정보를 제공한다.
    *   **컬러 토큰:** `--color-error` (채도 낮은 빨강 계열)을 사용하여 위험도를 낮추되, 경고 상태보다 명확하게 구분한다.
    *   **핵심 메시지:** 문제 상황 설명 + **"다시 시도하기" 버튼 (Primary CTA)** 조합이 필수적이다.
    *   **개발자 안내:** 반드시 에러 코드를 기록하고, 고객센터 문의를 유도하는 텍스트(`[오류 코드: XXXX]`)를 포함하여 기술 지원을 용이하게 한다.
