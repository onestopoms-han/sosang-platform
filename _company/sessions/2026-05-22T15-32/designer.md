# 📊 KPI 대시보드 디자인 가이드라인 (Final Specification)

## 1. 브랜드 컬러 팔레트 (Color Palette)
브랜드 정체성(신뢰, 성장, 경고/행동 유도)을 반영하여 다음과 같은 색상 시스템을 적용합니다.

| 용도 | 색상 이름 | Hex Code | RGB | 역할 및 의미 |
| :--- | :--- | :--- | :--- | :--- |
| **Primary (신뢰)** | BDS Blue | `#007BFF` | (0, 123, 255) | 주요 액션 버튼, 핵심 데이터 라인. 신뢰와 안정감을 부여합니다. |
| **Success (성장)** | Growth Green | `#28A745` | (40, 167, 69) | 긍정적인 지표(수익 증가 등), 성공적인 성장을 나타냅니다. |
| **Warning (경고/손실)** | Risk Orange | `#FFC107` | (255, 193, 7) | 주의가 필요한 지표(잠재적 손실), 즉각적인 검토를 요구합니다. |
| **Danger (위험)** | Loss Red | `#DC3545` | (220, 53, 69) | 심각한 위험 지표(손실 발생), 가장 강력한 행동 유도(CTA)로 사용합니다. |
| **Background** | Light Gray | `#F8F9FA` | (248, 249, 250) | 전체 배경. 시각적 피로도를 낮추고 데이터에 집중하게 합니다. |
| **Text/Accent** | Dark Text | `#343A40` | (52, 58, 64) | 본문 및 라벨 텍스트. 가독성을 극대화합니다. |

## 2. 반응형 레이아웃 구조 (Responsive Layout Structure)
대시보드는 모바일(Mobile-First) 환경을 기본으로 하며, 데스크톱에서는 정보 밀도를 높여 표시합니다.

### A. 전체 그리드 시스템
*   **시스템:** CSS Grid 기반의 12-컬럼 시스템을 사용합니다.
*   **패딩 (Spacing):** 모든 컴포넌트 간의 간격은 `16px` 또는 `24px`를 기본으로 설정하여 시각적 위계를 확보합니다.

### B. 레이아웃 구성 (Desktop View)
| 영역 | 너비 비율 | 내용 및 목적 | 스타일 가이드 |
| :--- | :--- | :--- | :--- |
| **Header/Title** | 100% | 대시보드 제목, 필터/기간 설정 (상단 고정) | `padding-top: 24px; font-size: 2rem;` |
| **KPI Summary Row** | 100% | 핵심 지표(Activation Rate, SCRR 등)를 큰 카드 형태로 배치. | 각 KPI는 별도의 카드(`Widget`)로 분리하여 강조. |
| **Main Data Grid** | 100% | 상세 데이터 시각화 영역 (차트 및 테이블). | `display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));` |
| **Action Panel** | 100% | '손실 최소화'를 위한 다음 행동 제안 섹션. | Risk/Loss 데이터가 높을 경우 이 영역에 **Danger Red** 강조. |

### C. 모바일 레이아웃 (Mobile View)
*   **Stacking:** 모든 카드와 위젯은 세로로 쌓이도록 구성합니다.
*   **KPI Priority:** 가장 중요한 KPI 3가지를 상단에 배치하고, 나머지는 스크롤을 통해 접근하도록 설계합니다.
*   **Interactivity:** 터치 영역(Tap Area)을 충분히 확보하여 모바일 환경에서의 클릭 편의성을 높입니다.
