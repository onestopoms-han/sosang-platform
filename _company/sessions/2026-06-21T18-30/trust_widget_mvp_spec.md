# 🎯 Trust Widget — MVP 명세서 (Version 1.0)

## 1. 실행 배경 및 목적

**[상황]**
- 소상공인플렛폼 초기 사용자(소상공인)는 매출 불안, 공급망 리스크 등으로 인해 고도의 불안을 느끼고 있습니다.
- 개발팀의 기술적 제약사항 보고서에 따르면, 복잡한 데이터 시각화나 실시간 추천 시스템은 초기 개발 비용과 시간이 과도하게 소요됩니다.
- **핵심 목표**: '불안 → 신뢰 → 실행'이라는 감성 흐름 중 첫 단계인 **'불안 해소'**를 최소한의 기능으로 달성하여 사용자 유지율을 높이는 것입니다.

**[목적]**
- 기술적 제약사항을 우회할 수 있는, 가장 직관적이고 효과적인 MVP 기능을 선정합니다.
- 이를 통해 초기 잠재 고객 테스트 시나리오의 성공 확률을 높입니다.
- 개발 리소스를 효율적으로 배분하고, 빠른 피드백 루프를 구축합니다.

## 2. MVP 기능 범위 (Minimum Viable Features)

### 🧩 핵심 기능: `실시간 매출 안심 신호등` (Traffic Light Dashboard)

이 기능은 복잡한 대시보드 대신, 사용자가 한눈에 파악할 수 있는 **가장 중요한 지표 하나**를 시각화하여 제공합니다.

| 기능 | 기술적 구현 난이도 | 예상 효과 | 비고 |
| :--- | :--- | :--- | :--- |
| **매출 안정도 (Revenue Stability)** | ⭐⭐ | 사용자가 자신의 매출 흐름을 '안정적/비난정적'으로 직관적으로 이해하게 합니다. | 개발팀: 평균 변동성(StdDev) + 최근 30 일 매출 추이만 표시 |
| **공급망 리스크 알림** | ⭐ | 공급망 이슈 발생 시, 실시간으로 경고 메시지를 띄웁니다 (예: "원자재 가격 상승 주의"). | 개발팀: 외부 API 연동 최소화. 내부 데이터 로직 기반 |
| **자금 유동성 예측** | ⚠️⚠️ | 복잡하여 MVP 는 제외합니다. | 후속 버전 목표 |

### 📐 시각적 인터페이스 요구사항 (Designer 의 UX Flow 와 호환)

- **컬러 팔레트**: `신뢰 파랑` (#2563EB) 을 기본으로, 위험 신호 시 `주의 주황` (#F59E0B) 으로 전환합니다.
- **레이아웃**: 단일 화면 (Single Page Application) 으로, 로딩 속도를 최적화합니다.
- **애니메이션**: 데이터 변동에 따른 부드러운 그래프 애니메이션 (CSS transition) 만 사용합니다. 복잡한 차트 라이브러리 대신 자바스크립트의 `Chart.js` 를 활용합니다.

### 🛠️ 기술적 제약사항 우회 전략 (코다리 보고서 기반)

| 제약 | 대응 방안 |
| :--- | :--- |
| **실시간 데이터 연동 지연** | 5 분 단위 주기 업데이트로, API 호출 횟수를 줄입니다. |
| **복잡한 차트 라이브러리 부하** | `Chart.js` 와 같은 경량 라이브러리로 대체하고, 차트 항목 수를 최소화합니다 (최대 3 개). |
| **모바일 최적화 필요** | 반응형 그리드 레이아웃과 터치 기반 인터랙션을 지원합니다. |

## 3. 핵심 지표 및 초기 KPI (Key Performance Indicators)

MVP 기능의 효과를 측정하기 위한 초기 지표를 정의합니다.

| 지표명 | 목표치 (1 개월) | 측정 방법 | 비고 |
| :--- | :--- | :--- | :--- |
| **사용자 유지율** | 70% | 일일 활성 사용자 비율 (DAU/MAU) | 기능 사용 후 이탈률이 급감해야 함 |
| **신뢰도 점수** | +1.5 점 | 사용자의 심리 상태 설문조사 (1~10 점 척도) | '안심 신호등' 기능을 사용한 후 조사 |
| **기능 클릭률** | 60% | 메인 대시보드에서 '안심 신호등' 탭 클릭 비율 | 다른 기능보다 높은 우선순위 제공 |
