### 📄 Trust Widget / PainGauge 기능 구현 가능성 검토 및 와이어프레임 테스트 스크립트 (V1.0)

본 스크립트는 사용자가 각 위젯을 상호작용했을 때 시스템이 예상해야 하는 시각적 변화와 데이터 흐름을 정의합니다.

#### 1. 디자인 시스템 제약 조건 확인 (Reference: `sessions/2026-06-28T12-12/designer.md`)

| 컴포넌트 | 상태 | 주요 디자인 규칙 (V3.0) | 테스트 목적 |
| :--- | :--- | :--- | :--- |
| **Trust Widget** | 활성화/비활성화 | 색상: Primary Blue (성공 시), Secondary Red (위기 시). 데이터 표시 형식: 5점 척도 기반의 시각적 바 및 점수. | 신뢰도 변화에 따른 시각적 피드백 검증 |
| **PainGauge** | 활성화/비활성화 | 색상: Warning Orange (높은 고통), Critical Red (심각한 위기). 데이터 표시 형식: Pain Level Score 및 설명 텍스트. | 고통 수준에 따른 경고 레벨 변화 검증 |

#### 2. 인터랙션 플로우차트 기반 테스트 스크립트 (Reference: `sessions/2026-06-28T12-12/designer.md`)

**A. Trust Widget 상호작용 테스트**

| Step | 사용자 액션 (클릭) | 시스템 예상 반응 (Visual State Change) | API 연동 확인 (Expected Output) | 성공 기준 |
| :--- | :--- | :--- | :--- | :--- |
| **T1** | Trust Widget 영역 클릭 (기본 상태) | 위젯이 활성화되고, 현재 데이터(예: 85%)가 시각화되어 표시됨. | `TrustData` API 호출 성공 및 결과 수신 확인. | 데이터 바와 점수가 정확하게 매핑되어 표시됨. |
| **T2** | '데이터 업데이트' 버튼 클릭 (사용자 입력 시뮬레이션) | 로딩 스피너 표시 후, 새로운 데이터가 반영될 때까지 대기 상태 유지. | `TrustData` API 호출 및 성공적인 데이터 쓰기 확인. | UI가 잠시 멈추고, 데이터 반영 후 새 수치가 명확하게 표시됨. |
| **T3** | '데이터 비활성화' (Toggle) 클릭 | 위젯이 비활성화되고, 데이터 시각화 요소가 흐리게(Grayed out) 처리됨. | `TrustData` API 호출에 대한 접근 차단 확인. | UI 요소가 비활성화되어 더 이상 상호작용이 불가능함을 명확히 표시함. |

**B. PainGauge 상호작용 테스트**

| Step | 사용자 액션 (클릭) | 시스템 예상 반응 (Visual State Change) | API 연동 확인 (Expected Output) | 성공 기준 |
| :--- | :--- | :--- | :--- | :--- |
| **P1** | PainGauge 영역 클릭 (기본 상태) | 현재 고통 수준(예: 7/10)에 따라 배경색이 Warning Orange로 표시되고, 설명 텍스트가 활성화됨. | `PainData` API 호출 성공 및 결과 수신 확인. | 색상과 텍스트가 Pain Level Score와 일치하게 시각화됨. |
| **P2** | '고통 수준 상세 보기' (Dropdown/Modal 열기) 클릭 | 상세 모달 창이 오버레이로 나타나며, 고통의 원인에 대한 추가적인 설명(Writer의 가이드라인 기반)이 표시됨. | `PainData`와 연관된 스토리텔링 데이터 로드 확인. | 사용자가 숨겨진 맥락을 이해할 수 있는 상세 정보가 제시됨. |
| **P3** | '고통 측정 재설정' (Reset) 버튼 클릭 | 고통 수준 입력 필드가 초기화되고, 시스템이 새로운 측정을 요청하는 상태로 전환됨. | `PainData`에 대한 리셋 명령 성공 확인. | UI가 깨끗하게 초기화되며, 다음 입력을 받을 준비가 됨. |

#### 3. MVP 우선순위 반영 요약 (Designer의 판단)

MVP 단계에서는 **T1, P1** 테스트에 최우선으로 집중해야 합니다. 이는 핵심 가치인 '신뢰도'와 '위기감'을 사용자가 즉각적으로 인지하는 시각적 피드백이 정확하게 작동함을 검증하는 것이므로, 복잡한 데이터 업데이트(T2)나 상세 설명 모달(P2)은 Phase 2로 미루고 **핵심 가시화** 기능의 안정성을 확보하는 데 집중합니다.
