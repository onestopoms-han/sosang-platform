## 3. [Tech Spec 확정] 기술 아키텍처 및 API 계약
코다리님께서 정의하신 데이터 계약(Data Contract)은 백엔드 개발의 가장 중요한 지침서가 됩니다. 이 스펙을 기준으로 모든 개발이 진행되어야 합니다.

*   **핵심 서비스:** `AnalyticsService` (ROI 계산 로직).
*   **주요 엔드포인트:** `POST /api/v1/analytics/calculate-roi`
*   **데이터 무결성 확보:** 모든 데이터는 엄격한 TypeScript/JSON Schema 기반의 타입을 강제화하여 개발 단계에서 오류를 최소화해야 합니다.

## 4. [UX/UI 확정] 사용자 경험 및 디자인 가이드라인
디자이너님의 최종 개선안을 통해, 플랫폼은 단순한 대시보드를 넘어 **'성장 코치(Growth Coach)'**의 느낌을 주도록 시각적 언어를 재설정했습니다.

*   **🎨 전체적인 흐름:** '현재 위치 $\rightarrow$ 목표 지점 $\rightarrow$ 다음 액션'으로 이어지는 선형적, 단계적(Progressive) 사용자 여정을 설계합니다.
*   **🌈 색상 시스템:** 신뢰도 높은 딥 블루 계열(`Primary`)과 성장을 강조하는 그린/오렌지 그라데이션(`Secondary/Action`)을 메인 컬러로 확정합니다.
*   **📐 시각화 규칙:** 모든 KPI와 로드맵은 수평적 '마일스톤 카드' 형태로 구성하여, 사용자가 다음 스텝에 대한 명확한 기대감을 갖도록 유도해야 합니다.
