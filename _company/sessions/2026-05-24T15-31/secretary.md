| **Phase 1** | **기초 경험 설계 (UX/UI)** | 최종 목업 기반의 사용자 여정 지도(User Flow) 통합 검증. CTA 및 핵심 액션 버튼의 반응형 테스트 완료. | Designer, 코다리 | 확정된 UI 컴포넌트와 인터랙션 가이드라인 |

### ✅ [2] 통합 QA 체크리스트 (Quality Assurance Checklist)
*(목표: 기술적 안정성 확보와 비즈니스 가설(심리적 이득 극대화) 달성 여부를 교차 검증)*

#### ⚙️ 시스템 기능/기술 점검 항목
*   [ ] **데이터 흐름**: 콘텐츠 시청 → Loss Gauge 확인 → 진단 툴 사용률 측정까지의 데이터가 끊김 없이 기록되는지? (KPI 추적 로직)
*   [ ] **사용자 인터랙션**: 모든 버튼과 링크(CTA)가 의도한 페이지와 기능을 정확히 수행하는지? (반응형 웹 검토 포함)
*   [ ] **에러 핸들링**: LLM 호출 실패, API 연결 오류 등 비정상 상황 발생 시 Fallback 화면이 정상적으로 작동하고 사용자에게 혼란을 주지 않는지?
*   [ ] **보안/결제**: 결제 모듈 연동 과정에서 개인 정보가 암호화되고, 트랜잭션 기록에 누락된 부분이 없는지?

#### 📈 비즈니스/사용자 경험 점검 항목 (Writer/Designer 관점)
*   [ ] **심리적 이득 극대화**: 사용자가 '불안감(Loss)'을 느낀 지점에서 '해결책/희망'으로 전환되는 스토리텔링의 흐름이 자연스러운지?
*   [ ] **가독성/명확성**: Loss Gauge 등 핵심 지표 설명이 소상공인 타겟에게 매우 쉽고 명료하게 전달되는지?
*   [ ] **참여 유도**: 진단 툴 사용률(핵심 KPI)을 높이기 위한 동기 부여 장치(예: 무료 컨설팅 제공 등)가 효과적으로 배치되었는지?
