## 📊 '통제권 회복 게이지' 인터랙션 흐름도 명세 (Flowchart Specification)

**핵심 원칙:** 사용자의 **행동(Input)**이 플랫폼의 **EAS 모델($W_{Control}=0.50$)**을 통해 정량화되어, 시각적인 **게이지 상승 값(Output)**으로 변환되는 과정을 단계별로 명세화합니다.

### 1. 시스템 정의 (System Definition)

| 요소 | 정의 | 비고 |
| :--- | :--- | :--- |
| **핵심 가치** | 실행 책임 (Execution Accountability) | 수익 모델의 근거 |
| **모델 변수** | $W_{Control}$ | 0.50 (통제권 회복에 가장 높은 가중치 부여) |
| **게이지 목표** | 사용자가 설정한 목표 달성도와 실제 행동 간의 괴리(Gap) 감소 및 실행 과정 완료 측정 | 시각적 보상 제공 |
| **KPI 산정 기준** | 백엔드 API 스키마 기반의 '실행 책임' 지표 | 데이터의 객관성 확보 |

### 2. 핵심 시나리오별 흐름 상세 명세 (Scenario Flow Details)

다음은 사용자가 특정 행동을 했을 때, 그 결과가 어떻게 게이지에 반영되는지를 보여주는 3가지 핵심 시나리오입니다. 모든 과정은 **[Input $\to$ Process $\to$ Output]**의 논리적 흐름을 따릅니다.

#### 시나리오 1: 재고 파악 성공 (Inventory Management Success)

*   **목표:** 소상공인이 실제 재고 상태를 정확히 파악하고 불필요한 손실을 줄이는 실행을 완료하는 과정 측정.
*   **Input (사용자 행동):** 사용자가 플랫폼의 '재고 연동' 기능을 통해 실제 재고 현황을 입력하거나 시스템이 자동으로 동기화하여 확인 버튼을 누르는 행위.
*   **Process (시스템 계산):**
    1.  **데이터 검증:** 입력된 데이터가 외부 API(혹은 사용자 입력)와 일치하는지 검증 (정확성 측정).
    2.  **차이 분석:** 목표 재고 대비 실제 재고의 차이(실패 지점)를 계산.
    3.  **통제권 산정:** 정확한 파악으로 인해 발생한 불확실성 감소 정도($\Delta U$)를 계산하여 $W_{Control}$에 따라 가중치를 부여.
*   **Output (게이지 변화):** **+15% 통제권 회복 점수.** (예: 목표 대비 98% 일치 달성 시)

#### 시나리오 2: 노동 생산성 증대 (Labor Productivity Gain)

*   **목표:** 사용자가 계획한 작업에 대해 효율적으로 시간을 배분하고, 그 결과로 생산성을 향상시키는 실행 과정 측정.
*   **Input (사용자 행동):** 사용자가 '작업 시작'과 '작업 완료'를 기록하며 소요된 실제 시간 데이터를 입력하는 행위.
*   **Process (시스템 계산):**
    1.  **시간 비교:** 계획된 시간($T_{Plan}$) 대비 실제 소요 시간($T_{Actual}$)을 비교하여 효율성($E = T_{Plan} / T_{Actual}$)을 계산.
    2.  **실행 책임 부여:** 이 효율성 증대가 사용자 통제권의 향상에 기여한 정도를 측정.
    3.  **통제권 산정:** 효율성 개선 폭에 가중치($W_{Control}=0.50$)를 적용하여 최종 점수를 도출.
*   **Output (게이지 변화):** **+25% 통제권 회복 점수.** (예: 계획 대비 20% 시간 단축 달성 시)

#### 시나리오 3: 비용 절감 실행 (Cost Reduction Execution)

*   **목표:** 사용자가 플랫폼을 통해 제시된 컨설팅이나 최적화 방안을 실제로 적용하여 재무적 이득을 창출하는 과정 측정.
*   **Input (사용자 행동):** 사용자가 제안된 절감 방안(예: 원가 절감 방법)을 실행하고, 그 결과를 플랫폼에 피드백으로 제출하는 행위.
*   **Process (시스템 계산):**
    1.  **ROI 측정:** 실행 전후의 비용 및 수익 변화를 분석하여 순수익 증가분($\Delta Profit$)을 계산.
    2.  **실행 책임 연계:** 이 $\Delta Profit$이 플랫폼 가이드라인에 따라 달성되었는지 검증.
    3.  **통제권 산정:** 재무적 성취가 실행의 통제감으로 이어졌는지($W_{Control}=0.50$)를 평가하여 점수를 부여.
*   **Output (게이지 변화):** **+40% 통제권 회복 점수.** (예: 목표 비용 대비 15% 절감 달성 시)

### 3. 최종 UI/UX 디자인 제안 (Design Implication)

이러한 흐름은 게이지의 시각적 표현에 다음과 같은 애니메이션 및 레이아웃을 요구합니다.

1.  **동적 연결:** Input 발생 시, 게이지가 즉시 반응하며 'Processing' 상태를 거쳐 최종 Output 값이 점진적으로 채워지도록 설계합니다 (물리적 결합/조립의 경험 반영).
2.  **색상 변화:** $W_{Control}$에 따라 녹색(성공)이 주를 이루며, 목표 달성 전까지는 회색/파란색으로 표시하여 불안정성($\to$)에서 통제권 회복($\leftarrow$)으로의 여정을 명확히 시각화합니다.
3.  **피드백 강조:** Output 값이 산출될 때마다, 해당 행동(Input)과 결과(Output) 사이에 **'실행 책임(Execution Accountability)'**이라는 텍스트 레이어를 삽입하여, 단순한 수치 변화가 아닌 '내가 이만큼 통제했다'는 인지적 보상을 제공해야 합니다.
