## 📊 KPI 대시보드 및 UX 전환 지점 시각화 가이드라인 (V1.0)

| 요소 | 색상 코드 | 역할 및 심리적 효과 | 적용 범위 |
| :--- | :--- | :--- | :--- |
| **Primary (신뢰/안정)** | `#004D66` (Deep Blue) | 플랫폼의 신뢰성, 데이터의 안정성, 전문성을 표현. 배경 및 주요 구조에 사용. | 대시보드 배경, 메인 내비게이션, 핵심 섹션 구분선 |

| **Neutral (데이터/구분)** | `#F8F9FA` (Light Gray) | 정보의 명확한 구분을 위한 배경색. 시각적 피로도 감소. | 카드 배경, 그래프 보조선, 텍스트 블록 배경 |

### 2. KPI 대시보드 레이아웃 설계 (Loss Minimization Focus)

KPI 대시보드는 사용자가 현재 상태(손실)를 인식하고, 다음 행동(손실 최소화)을 취하도록 유도하는 흐름으로 설계합니다.

**레이아웃 컨셉:** **'불안 → 진단 → 조치 → 회복'의 4단계 흐름**

| 섹션 | 목적 및 내용 | 시각적 디자인 가이드 | Loss Minimization 적용 방식 |
| :--- | :--- | :--- | :--- |
| **A. 핵심 손실 지표 (Loss Focus)** | 현재 플랫폼 사용으로 인해 발생한 잠재적/실제 손실을 가장 먼저 인지시킴. | **KPI 카드:** 각 KPI는 '현재 값'과 '목표 대비 차이'를 명확히 표시. 목표 달성 시 Growth Green, 미달 시 Loss Red 강조. | **Loss Red(#DC3545)**를 사용하여 현재의 부정적 상태(손실)를 즉각적으로 인지시킴. |
| **B. 손실 분석 (Analysis)** | 어떤 지점에서 손실이 발생했는지 데이터 기반으로 진단함. | **트렌드 그래프:** 시간 경과에 따른 손실/이득 추이를 시계열로 표시. 꺾은선 그래프를 사용하여 변화의 흐름을 강조. | Loss Red 선이 하락하거나, Growth Green 선이 상승하는 패턴을 통해 사용자가 '어디서 위험한지' 직관적으로 파악하게 함. |
| **C. 행동 제안 (Action)** | 손실을 최소화하기 위한 구체적인 다음 단계를 제시함. | **액션 블록:** 각 KPI 카드 하단에 '권장 조치(Loss Minimization Action)' 텍스트와 함께 CTA 버튼 배치. | **CTA 버튼:** 가장 중요한 행동은 Growth Green으로 강조하며, 버튼 텍스트는 손실 회피에 초점을 맞춤 (예: "손실 최소화 전략 확인"). |
| **D. 회복 지표 (Recovery)** | 잠재적인 이득 또는 성공적인 행동을 시각화하여 동기 부여함. | **성과 요약:** 누적된 성장률, 절감 금액 등을 긍정적으로 표현. | Growth Green을 주 색상으로 사용하여 사용자가 긍정적인 행동을 했을 때의 보상을 시각화함. |

### 3. 핵심 UX 전환 지점 디자인 가이드 (CTA Focus)

핵심 UX 전환 지점(예: 구독 신청, 컨설팅 요청 등)에서는 '손실 회피' 심리를 극대화하여 사용자의 결정을 유도합니다.

**디자인 목표:** **불안감(Loss Aversion)을 행동(Action)으로 즉시 전환.**

| UI 요소 | 디자인 적용 방법 | Loss Minimization 적용 방식 |
| :--- | :--- | :--- |
| **CTA 버튼 (Primary)** | `Growth Green (#3CB371)` 배경에 흰색 텍스트. 충분한 크기와 대비를 확보하여 시선 집중. | "지금 손실을 최소화하고 성장 시작하기"와 같이, 행동의 결과(성장)를 강조하는 문구를 사용하여 클릭 동기를 부여. |
| **구독/결제 화면** | 가격 플랜 비교 시, 'Standard' 플랜을 기본값으로 설정하고, 'Basic' 플랜 선택 시 잠재적 기회비용(Loss)을 명시적으로 표시. | **기회비용 강조:** "Basic 선택 시, 월 30%의 성장 기회를 놓치게 됩니다"와 같이, 선택하지 않았을 때 발생할 손실을 수치로 제시. |
| **진행 단계 (Journey Map)** | 사용자 여정 지도(A→B→C→D) 각 단계 사이에 '잠재적 위험 알림' 아이콘(`Loss Red`)을 삽입. | 각 단계는 단순한 과정이 아닌, "이 단계를 건너뛰면 발생할 수 있는 손실"에 대한 경고로 기능하도록 설계하여 몰입도를 높임. |
| **데이터 입력 폼** | 필수 항목 미입력 시, 해당 정보가 누락될 경우 발생할 '잠재적 기회 손실'을 실시간으로 팝업 메시지로 경고. | "이 정보를 입력하지 않으면 잠재적 마케팅 효율성 손실이 발생합니다."와 같은 구체적인 손실 메시지를 제공하여 데이터 입력을 필수화함. |
