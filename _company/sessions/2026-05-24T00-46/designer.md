## 🖼️ AI 리스크 예측 대시보드: 최종 비주얼 시스템 및 썸네일 브리프

### 1. 핵심 디자인 원칙 (The Core Principle)
**목표:** 막연한 불안감(Fear)을 구체적인 데이터(Quantification)를 통해 명확히 제시하고, 대시보드가 유일한 통제권(Empowerment)임을 각인시킨다.

**주요 컨셉:** **"경고등에서 생존 전략으로" (From Warning Light to Survival Strategy)**
*   **톤앤매너:** 긴급함(Urgency) + 전문성(Authority) + 희망(Hope).
*   **핵심 시각 요소:** Red/Green 시스템 및 Loss Gauge의 극대화.

### 2. 브랜드 컬러 가이드라인 (Color System)
| 색상 | HEX Code | 용도 (Purpose) | 심리적 연관성 | 적용 위치 |
| :--- | :--- | :--- | :--- | :--- |
| **Red (위협)** | `#E53E3E` | 최대 손실, 경고, 위험 지역 | 공포(Fear), 긴급성 | Max Loss 수치, 위험 영역 배경, 경고 라인 |
| **Green (해결)** | `#38A169` | 안전, 해결책, 기회 | 안도감(Relief), 성장, 통제권 | WTP 시뮬레이션 결과, 성공 영역 강조, CTA 버튼 |
| **Dark Blue (신뢰)** | `#2C5282` | 배경, 텍스트 제목, 시스템 프레임 | 신뢰(Trust), 전문성, 안정감 | 전체 배경, 대시보드 프레임, 핵심 헤드라인 |
| **Light Gray (중립)** | `#F7FAFC` | 보조 배경, 여백 | 명료함, 깔끔함 | 정보 패널 배경, 데이터 표 영역 |

### 3. 썸네일 컨셉 3안 비교 및 상세 브리프

세 가지 컨셉 모두 'Loss Gauge'를 중심으로 불안과 해결을 대비시킵니다.

#### **컨셉 A: 극단적 대비 (The Alarm)**
*   **핵심:** 공포와 해결책의 가장 직접적인 충돌을 시각화.
*   **레이아웃:** 화면을 수직으로 명확하게 분할. 좌측/상단은 붉은색(위협), 우측/하단은 녹색(해결).
*   **주요 요소:** 중앙에 거대한 **Loss Gauge (게이지 바)**가 화면을 가로지르며, 현재 상태는 강렬한 Red로 표시하고, 목표치(WTP)를 녹색으로 설정.
*   **텍스트 배치:** 메인 헤드라인은 상단에 짧고 강력하게 ("당신의 최대 손실액은?") 배치. 하단에는 "AI 리스크 예측 대시보드" 로고와 Green CTA 버튼 강조.
*   **적합 대상:** 유튜브 썸네일, 즉각적인 클릭 유도에 최적화.

#### **컨셉 B: 데이터 중심 (The Analyst)**
*   **핵심:** 데이터의 구체성과 전문성을 강조하여 신뢰도를 구축.
*   **레이아웃:** 다크 블루 배경을 사용하며, 대시보드 UI 스크린샷(MVP)을 중앙에 배치하고 그 위에 핵심 지표만 오버레이.
*   **주요 요소:** Red/Green 시스템은 보조적인 색상으로 사용하되, 데이터 표(Loss Gauge의 구체적 수치)를 가장 크게 강조.
*   **텍스트 배치:** 헤드라인은 전문성을 담아 ("AI 시대, 가게 생존 확률을 90%로 높이는 법") 제시하고, 대시보드 화면 자체를 '해결책'으로 포지셔닝.
*   **적합 대상:** 랜딩 페이지 배너, 상세 콘텐츠 도입부.

#### **컨셉 C: 감성적 스토리 (The Journey)**
*   **핵심:** 소상공인의 여정(Pain $\rightarrow$ Solution)을 서사적으로 표현.
*   **레이아웃:** 왼쪽에는 불안한 모습의 상점 이미지(약간 어둡게), 오른쪽에는 밝고 희망적인 대시보드 화면이 병치(Juxtaposition).
*   **주요 요소:** 경계선에 Red/Green 시스템을 사용하여, '어둠 속에서 빛을 발견했다'는 서사를 시각적으로 표현.
*   **텍스트 배치:** 스토리텔링 기반의 카피를 강조 ("불안함은 끝났다. 이제 AI 리스크를 통제하라.").
*   **적합 대상:** 인스타그램 피드, 감성적인 광고 소재.

### 4. Red/Green 시스템 시각화 가이드라인 (System Guidelines)

이 시스템은 모든 디자인에서 일관되게 적용되어야 합니다.

1.  **Loss Gauge (손실 게이지):**
    *   **구조:** 전체 손실 범위(0%~100%)를 나타내는 바 형태의 게이지를 사용합니다.
    *   **Red Zone (위험 영역):** Loss가 특정 임계치(예: 50% 이상)를 넘으면 배경색을 **`#E53E3E` (Red)**로 전환하고, 게이지 내부 선을 붉은색으로 채웁니다.
    *   **Green Zone (안전 영역):** Loss가 목표 임계치(예: 20% 미만)에 도달하면 배경색을 **`#38A169` (Green)**로 전환하고, 게이지 내부 선을 녹색으로 채웁니다.
    *   **중립 Zone:** 중간 영역은 Dark Blue(`#2C5282`) 또는 Light Gray(`#F7FAFC`)를 유지하여 데이터의 명확성을 확보합니다.

2.  **CTA (Call to Action) 버튼:**
    *   **상태별 색상:**
        *   **위험 상태 시:** Red 버튼 (긴급한 조치를 유도).
        *   **해결 가능 상태 시:** Green 버튼 (안심하고 다음 단계로 진행 유도).
    *   **폰트:** CTA 텍스트는 **Bold, Sans-serif 계열(예: Pretendard Bold)**을 사용하여 즉각적인 행동을 촉구합니다.

3.  **데이터 레이블링:**
    *   모든 수치와 지표는 해당 색상 시스템에 따라 명확히 구분하여 표시합니다. 예를 들어, 'Max Loss' 값 옆에는 작은 Red 아이콘이나 붉은색 텍스트를 사용하여 심각성을 강조합니다.
