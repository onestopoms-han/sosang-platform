# 📝 Action Plan Pro 최종 디자인 사양서 (Design Specification)

## 1. 디자인 목표 및 핵심 원칙
*   **목표:** 사용자(소상공인)가 자신의 현재 상태를 인식하고, 'Action Plan Pro'를 통해 미래의 손실 리스크를 관리하는 것이 가장 쉬운 행동임을 각인시킨다.
*   **핵심 원칙:** **Pain Point First & Loss Cost Emphasis (손실 비용 강조)**
*   **톤 앤 매너:** 신뢰감 있는(Trustworthy), 긴급한(Urgent), 명확한(Clear). 불안감을 자극하되, 해결책을 제시함으로써 안도감을 제공한다.

## 2. 브랜드 시스템 연동 (Color & Typography)
기존 BDS소상공인플렛폼의 디자인 시스템을 유지하되, '리스크'와 '해결'의 대비를 강조하는 보조 색상을 도입합니다.

| 요소 | 역할 | 색상 코드 (Hex) | 설명 |
| :--- | :--- | :--- | :--- |
| **Primary Color** | 브랜드 핵심 (BDS 정체성) | `#007AFF` (Blue) | 신뢰, 안정감 제공 |
| **Accent Color (Loss)** | 리스크/손실 강조 | `#FF3B30` (Red) | 긴급성, 경고, 손실 비용 시각화 |
| **Success Color** | 해결/행동 유도 | `#34C85F` (Green) | 성공적인 행동, 안전함 제공 |
| **Background** | 안정감 확보 | `#F9F9F9` (Light Gray) | 정보의 명확성 극대화 |
| **Typography** | 제목/강조 텍스트 | - | 산세리프 계열 (예: Pretendard, Noto Sans)을 사용하여 데이터의 명료성을 높인다. |

## 3. 핵심 사용자 여정 지도 (Visual Flow)
사용자가 무료에서 유료로 전환되는 과정에 맞춰 시각적 흐름을 설계합니다.

**[단계 1: 문제 인식 (Pain Point)]**
*   **화면:** 랜딩 페이지 상단 및 온보딩 단계.
*   **시각 전략:** 가장 큰 손실 비용(Loss Cost)을 추상적인 불안감이 아닌, **구체적인 수치와 질문**으로 제시한다.
    *   *예시:* "당신이 놓친 행동 하나가 월 매출 50만 원의 기회비용을 의미합니다." (Red Accent 강조)

**[단계 2: 솔루션 제시 (Solution)]**
*   **화면:** Action Plan Pro 소개 페이지.
*   **시각 전략:** '보험료'라는 은유를 사용하여 유료 서비스의 가치를 포지셔닝한다. 데이터 시뮬레이션(Risk Simulator)의 개념을 강조하여, **불안감을 통제 가능성으로 전환**시킨다.
    *   *흐름:* "막연한 불안 $\rightarrow$ 데이터 기반 리스크 측정 $\rightarrow$ 구체적 보험료(Action Plan Pro) 확보"

**[단계 3: 행동 유도 (Conversion)]**
*   **화면:** 가격 비교 및 구매 버튼 영역.
*   **시각 전략:** 무료 기능과 Pro 기능을 명확히 대비시키고, Pro 기능이 제공하는 **'예측 능력(Prediction)'과 '자동화(Automation)'**에 가장 큰 시각적 무게를 둔다.
    *   *CTA 디자인:* '무료 진단' 버튼은 Primary Color로, 'Pro 구독 시작' 버튼은 Accent Color (Red/Green 조합)로 대비시켜 행동을 유도한다.

## 4. 핵심 UI 컴포넌트 상세 사양: Action Plan Dashboard (유료 기능)

**A. 리스크 시뮬레이터 컴포넌트 (Risk Simulator Component)**
*   **목적:** 사용자가 입력한 데이터(매출, 지출, 시간)를 기반으로 잠재적 손실 비용을 계산하여 보여준다.
*   **레이아웃:** 카드형 디자인. 좌측에 현재 상태 요약, 중앙에 리스크 시뮬레이션 그래프/차트, 우측에 '예상 손실 금액(Loss Cost Estimate)'을 가장 크고 굵게 표시한다.
*   **시각 요소:** Red Accent Color를 사용하여 손실 규모를 강조하고, Green 색상을 통해 목표 달성 시 절감 가능성을 시사한다.

**B. AI 기반 우선순위 배열 (AI Prioritization View)**
*   **목적:** AI가 분석한 Action Plan을 가장 중요한 순서대로 제시하여 즉각적인 실행 동기를 부여한다.
*   **레이아웃:** 리스트 뷰(List View) 또는 타임라인 뷰(Timeline View).
    *   **최상단 항목 (Top Priority):** 가장 높은 '손실 비용 감소 효과'를 가진 액션 아이템을 배치한다. 이 항목은 Red/Bold 처리하여 시각적 압박감을 주어 즉시 클릭하게 만든다.
    *   하위 항목들은 일반적인 Task 목록으로 제시한다.

**C. 데이터 연결 및 연동 인터페이스 (Data Link Interface)**
*   **목적:** 외부 데이터(예: 매출, 재고)를 연결하는 과정을 직관적으로 보여준다.
*   **디자인:** 드래그 앤 드롭 또는 명확한 '연결선' UI를 사용하여, 어떤 데이터가 Action Plan에 어떻게 반영되는지 시각적인 흐름을 끊김 없이 전달한다.
