🎨 Designer: 작업 시작합니다.

CEO님의 지시에 따라, 인터뷰 데이터에서 도출될 핵심 지표(손실액, WTP)를 시각화하는 대시보드 구조와 기술적 실행 가능성을 구체적으로 설계하겠습니다. 이는 플랫폼의 '가치 증명'을 위한 핵심 기반이 될 것입니다.

현재 단계에서는 **데이터 구조 정의**와 **비주얼 시스템 초안**에 집중하여, 향후 개발팀(Developer)과 Writer가 즉시 실행할 수 있는 명세를 제공합니다.

## 📊 데이터 시각화 지표 및 대시보드 구조 구체화 (Design Spec)

핵심 목표는 소상공인이 '리스크'를 인식하고 '솔루션'을 선택하게 만드는 **행동 유도형(Actionable)** 대시보드를 설계하는 것입니다.

### 1. 핵심 측정 지표 정의 (KPIs)

인터뷰에서 도출될 데이터를 바탕으로, 다음 세 가지 핵심 지표를 중심으로 대시보드를 구성합니다.

| KPI 명칭 | 정의 (Metric) | 데이터 출처 (Source) | 시각화 목적 (Goal) |
| :--- | :--- | :--- | :--- |
| **잠재 손실액 (Potential Loss)** | 소상공인이 현재 직면한 예상 잠재적 금전적 손실 규모. | 인터뷰 기반 추정치 및 입력값 | 문제의 심각성(Pain Point) 즉각 인지 |
| **WTP 지수 (Willingness to Pay Index)** | 해당 손실을 해결하기 위해 소상공인이 기꺼이 지불할 의사가 있는 금액 또는 가치. | 인터뷰 기반 측정값 | 솔루션에 대한 구매 의향(Value Proposition) 증명 |
| **리스크 회피 가치 (Risk Avoidance Value)** | 잠재 손실액 대비, 플랫폼을 통해 얻을 수 있는 '안정성'의 가치. | 계산된 지표 (WTP / Potential Loss) | 솔루션 도입의 경제적 효용(ROI) 시각화 |

### 2. 대시보드 레이아웃 및 컴포넌트 설계 (Visual System Draft)

**컨셉:** **"Pain $\rightarrow$ Value $\rightarrow$ Action" 흐름**을 따르는 단일 스크롤형 플로우 디자인.

#### A. 메인 화면: 리스크 진단 캔버스 (Risk Assessment Canvas)

*   **레이아웃:** 중앙 집중형(Centered Layout). 사용자가 정보를 입력하고 결과를 즉시 확인할 수 있도록 대화형 카드(Interactive Card) 구조 채택.
*   **핵심 컴포넌트:** **슬라이더 & 입력 필드**를 활용하여 데이터 입력을 유도합니다.

    1.  **Pain Input (왼쪽 패널):** "당신의 예상 잠재 손실은 얼마입니까?" $\rightarrow$ 슬라이더 및 텍스트 입력창으로 **잠재 손실액(Potential Loss)**을 입력하게 합니다.
    2.  **Value Output (중앙 시각화):** 입력된 손실액에 기반하여, 플랫폼이 제공할 수 있는 잠재적 가치와 비교하는 그래프를 표시합니다. $\rightarrow$ **리스크 회피 가치 (Risk Avoidance Value)**를 차트로 보여줍니다.
    3.  **Solution Gateway (오른쪽 패널):** 계산된 WTP 지수에 따라, 가장 적합한 플랫폼 모듈(예: 'AI 리스크 분석' 또는 '컨설팅 연결')로의 **CTA 버튼**을 제시합니다.

*   **색상 팔레트 제안:**
    *   **Primary (Pain/Loss):** 짙은 주황색 계열 (`#FF6B6B`) - 긴급성과 경고의 느낌 강조.
    *   **Secondary (Value/WTP):** 신뢰감을 주는 청록색 계열 (`#4ECDC4`) - 안정성과 기회의 느낌 강조.
    *   **Action (CTA):** 플랫폼의 신뢰성을 반영하는 진한 네이비 또는 딥 그레이 (`#1A2330`).

#### B. 기술적 실행 가능성 검토 (Feasibility Review)

| 항목 | 내용 | 실행 가능성 | 비고 및 제언 |
| :--- | :--- | :--- | :--- |
| **데이터 수집** | 인터뷰 기반의 정량 데이터(손실액, WTP)를 입력받는 Form 구조. | 높음 (UI/UX 구현 용이) | 초기에는 실제 API 연동 대신, **Mockup 단계에서 시나리오 기반 데이터 입력**을 우선 진행하여 디자인 흐름을 확정해야 함. |
| **시각화 로직** | 손실액 대비 가치 지표를 실시간으로 계산하고 차트/게이지로 표현하는 로직. | 높음 (프론트엔드 라이브러리 활용) | $\text{Risk Avoidance Value} = f(\text{WTP}, \text{Loss})$ 공식이 명확히 정의되어야 하며, 이는 Writer/Developer와 최종 확정해야 함. |
| **시스템 연동** | 이 대시보드가 실제 AI 예측 모듈(`AI Risk Predictor`)과 어떻게 연결되는지 명세화. | 중간 (상위 단계 필요) | 초기에는 '가치 증명'에 초점을 맞춘 UI를 먼저 완성하고, 추후 데이터 통합 시점(Phase 2)에 시스템 API 연동을 진행하는 것이 효율적임. |
