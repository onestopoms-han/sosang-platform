## Ⅰ. 핵심 컴포넌트 시각적 일관성 (Visual Consistency)

### TrustWidget
- [ ] **색상 코드:** 명세서에 정의된 `primary_blue`, `alert_red`, `success_green` 등 색상 코드가 실제 개발 로직에서 정확히 적용되는지 확인.
- [ ] **타이포그래피:** `font-family`, `font-size`, `line-height` 등 타이포그래피 스타일이 컴포넌트 전체에서 일관성을 유지하는지 검증.
- [ ] **아이콘 및 이미지:** 디자인 시스템에 포함된 아이콘 및 이미지가 실제 사용 시 품질과 크기가 일관성 있게 표시되는지 확인.

### PainGauge
- [ ] **게이지 애니메이션:** `ease-in-out` 또는 `cubic-bezier(0.4, 0, 0.2, 1)` easing 함수가 실제로 구현되어 부드러운 전환이 이루어지는지 검증.
- [ ] **모달 구조:** 상세 설명 모달의 계층 (Hierarchy) 이 명확하고, 클릭/호버 시 올바른 트리거와 상태 변화를 일으키는지 확인.

### 공통 컴포넌트
- [ ] **반응형 디자인:** 모바일, 태블릿, 데스크톱 등 다양한 화면 크기에 걸쳐 레이아웃과 기능이 정상적으로 동작하는지 테스트.
- [ ] **접근성:** 키보드 네비게이션, 스크린 리더 호환성 등 웹 접근성 가이드라인 준수 여부 확인.

## Ⅱ. UX 흐름 및 비즈니스 로직 구현 적합성 (UX Flow & Business Logic)

### 사용자 여정 (User Journey)
- [ ] **온보딩:** 가락시장 상인을 위한 온보딩 콘텐츠와 실제 사용자 경험(UX)이 통합되어, 사용자가 플랫폼 가치를 쉽게 이해할 수 있는지 검증.
- [ ] **전환 애니메이션:** TrustWidget/PainGauge의 전환 애니메이션이 '신뢰 증진'을 효과적으로 전달하며, 과도한 시각적 요소 없이 명료하고 부드러운 흐름으로 사용자의 불안감을 해소하는지 평가.

### 비즈니스 로직 (Business Logic)
- [ ] **손실 최소화 테마:** '손실 최소화'가 핵심 테마로 설정되어 있어, UI/UX에서 이 감성이 제대로 표현되는지 확인.
- [ ] **기능 일치성:** 백엔드 API 스펙과 프론트엔드 컴포넌트 간의 로직 일치가 완벽하게 이루어지고 있는지 검증.

- [ ] **온보딩 콘텐츠 연동:** 온보딩 콘텐츠가 실제 사용자 경험과 완벽히 통합되어, 사용자가 플랫폼 가치를 쉽게 이해할 수 있는지 테스트.
