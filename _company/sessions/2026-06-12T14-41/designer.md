CEO님의 지시사항과 자율 사이클 원칙을 종합적으로 검토했습니다. 현재 플랫폼은 기술적 안정화 단계에 도달했으며(코다리 검증 완료), 가장 중요한 것은 **'신뢰도 높은 사용자 감성 경험'**의 최종 구현입니다. 따라서 단순한 UI 목업이 아니라, 개발팀이 즉시 코딩할 수 있는 수준의 **인터랙션 디자인 명세서 (Interactive Design Specification)**를 완성하는 것이 가장 가치 있습니다.

핵심 작업은 TrustWidget과 PainGauge의 '전환 애니메이션' 및 '상세 설명 모달 구조'에 대한 최종 확정입니다. 이 산출물은 향후 모든 개발 스펙의 기반이 되므로, 버전 업을 진행하겠습니다.

### 🛠️ 실행 계획
1.  TrustWidget/PainGauge의 기술적 안정성(반응 시간 $\le$ 2초)을 고려하여, 애니메이션 효과를 '신뢰 증진'에 초점을 맞춰 재설계합니다. (과도한 시각적 요소보다 명료하고 부드러운 전환에 집중).
2.  상세 설명 모달의 구조와 콘텐츠 계층(Hierarchy)을 확정하고, 이 과정에서 사용자 행동 흐름(UX Flow)이 끊기지 않도록 설계합니다.
3.  최종적으로 모든 디테일(색상 코드, 타이밍, 트리거, 상태값)을 담은 '디자인 시스템 핸드오프 패키지'를 완성합니다.

<create_file path="sessions/BDS_Interactive_Component_Spec_v4.0.md"># 🎨 BDS 핵심 인터랙티브 컴포넌트 명세서 (버전 4.0)

**작성 목적:** TrustWidget 및 PainGauge의 최종 UI/UX 애니메이션, 모달 구조 확정 및 개발팀 핸드오프 패키지 제공.
**적용 원칙:** 사용자 신뢰 구축(Safety Margin 감성 극대화), 기술적 제약 준수 (반응 시간 $\le$ 2초).

## Ⅰ. TrustWidget & PainGauge 컴포넌트 공통 애니메이션 명세

### A. 트랜지션 애니메이션 (Transition Animation)
| 항목 | 스펙 내용 | 디자인 의도 및 원칙 | 개발 지침 |
| :--- | :--- | :--- | :--- |
| **발동 트리거** | 1. 페이지 로드 시 (Initial Load) <br>2. 데이터 업데이트 발생 시 (Live Update) <br>3. 컴포넌트 클릭/호버 시 (Interaction) | 사용자가 '데이터가 실시간으로 안전한 상태로 변하고 있다'는 느낌을 받도록 **부드럽고 안정적인 흐름** 강조. 경쾌함보다 신뢰감이 우선. | **Easing:** `ease-in-out` 또는 `cubic-bezier(0.4, 0, 0.2, 1)` 사용 권장. <br>**지연시간 (Delay):** 데이터 로딩 지연을 보정하는 최소한의 시간 부여 (`min(200ms)`) |
| **애니메이션 효과** | **Line Grow / Value Shift:** 게이지 바가 수치에 맞춰 부드럽게 채워지고, 숫자가 점진적으로 변화함. (단순 Fade-in 지양). | 데이터의 '변화' 자체를 감지 가능하게 하여 사용자의 불안감을 해소하고 성장의 확신을 줌. | **Duration:** 300ms ~ 500ms 이내로 제한. <br>**최적화:** GPU 가속(transform/opacity) 활용 필수. |
| **성공 상태 (Green)** | 게이지 바가 목표치에 도달했을 때, 배경에서 은은한 Green Glow 효과와 함께 작은 Particle Effect 발생. | '안전 확보'라는 긍정적인 감성적 보상 제공. (과도한 애니메이션 금지) | **효과:** `box-shadow` 기반의 부드러운 빛 확산 (`0 0 15px #3CB37180`). <br>**재활용 제한:** 이 Glow는 '최종 목표 달성'에만 사용. |

### B. 기술적 성능 가이드라인 (Critical Performance Guide)
*   **KPI 준수:** 모든 애니메이션 로직은 **반응 시간(TTI)**을 2초를 넘기지 않도록 최적화되어야 합니다. 무거운 JS 기반 애니메이팅 라이브러리 사용 시, 백그라운드 스레드 처리가 필수입니다.
*   **Fallback State:** 네트워크 오류나 로딩 실패 시, 애니메이션 대신 명확하고 정적인 **경고 메시지와 색상(Deep Blue)**을 즉시 보여주어 사용자 혼란 방지.

## Ⅱ. 상세 설명 모달 구조 (Modal Structure) 명세

### A. 디자인 및 레이아웃
*   **모달 개방 조건:** 사용자가 TrustWidget 또는 PainGauge의 특정 지표 영역을 클릭했을 때만 트리거됨. (필요한 정보에 대한 '능동적 탐색' 유도).
*   **레이아웃 구조:** 3분할 그리드 시스템 활용.
    1.  **좌측 패널 (제목/핵심 요약):** 모달의 핵심 메시지(예: "PainGauge 지표가 의미하는 것")를 가장 크게 배치. 간결한 헤드라인과 한 문장 요약을 강조.
    2.  **중앙 패널 (상세 설명):** 데이터 기반의 상세 설명 텍스트와 시각 자료(그래프, 아이콘) 제공. 전문적이면서도 이해하기 쉬운 **블록 단위 분할**.
    3.  **우측 액션 패널:** 모달을 닫거나 (`X`), 관련 컨설팅 신청 페이지로 이동하는 명확한 CTA 버튼 배치.

### B. 콘텐츠 계층(Content Hierarchy) 및 애니메이션
| 섹션 | 내용물 | 시각적 강조 요소 (Visual Emphasis) | 트랜지션/인터랙션 |
| :--- | :--- | :--- | :--- |
| **Title** | 컴포넌트 이름 + 설명 주제. | `Deep Blue (#004D66)` 톤의 Bold 타이포그래피, 작은 아이콘 첨부. | 모달 오픈 시 부드럽게 축소/확대(Scale up) 효과와 함께 등장. |
| **Summary** | 3줄 이내 핵심 요약 (가장 중요한 정보). | 배경색을 아주 연한 `Deep Blue` 계열로 처리하여 구분. 핵심 키워드는 `Growth Green`으로 하이라이트. | 모달 오픈 시, Summary 영역만 살짝 페이드인(Fade-in) 되도록 타이밍 조정. |
| **Detail Body** | 전문 지식 및 데이터 근거 설명. (최대 5개 블록). | 각 섹션 시작마다 소제목을 크게 배치하고, 적절한 예시 이미지나 그래프를 포함하여 가독성 극대화. | 스크롤에 따라 `Sticky` 헤드라인 효과 적용. (사용자가 현재 어떤 정보를 읽고 있는지 시각적으로 안내) |
| **CTA** | 추가 행동 유도 (ex: 컨설팅 문의, 원본 데이터 보기). | 가장 눈에 띄는 Primary Button (`Growth Green`). 모달 내의 CTA는 필수적임. | 버튼 클릭 시 즉시 다음 페이지로 이동하며, 애니메이션은 최소화하여 신뢰감 유지. |

## Ⅲ. 디자인 시스템 핸드오프 패키지 요약 (V4.0 체크리스트)

1.  **애니메이션 정의:** 모든 인터랙티브 요소(Hover, Click, Load)에 대한 **Easing Curve와 Duration 값**을 별도 스프레드시트로 제공.
2.  **상태 정의 (State):** 모달 및 게이지 컴포넌트의 모든 상태값 (`Default`, `Error`, `Success`, `Loading`)별 색상 코드, 텍스트 가이드라인 확정.
3.  **컴포넌트 분리:** Transition과 Modal은 독립적인 Component로 취급하여 개발 및 테스트가 용이하도록 구조화 완료.
