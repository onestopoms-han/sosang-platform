### ✨ BDS 플랫폼 통합 마일스톤 로드맵 (Mockup → Live MVP)

현재까지의 결과물(랜딩 페이지 목업, 손실 비용 계산 API 구조)을 바탕으로, 다음 3단계에 걸쳐 작업을 진행해야 합니다.

#### 🗓️ Phase 1: 백엔드-프론트 통합 및 핵심 기능 완성 (Target: D+5일)
*   **목표:** Stub API를 실제 데이터 연동 로직으로 전환하고, 랜딩 페이지의 모든 섹션이 실시간 데이터를 받아 작동하는 상태로 만듭니다.
*   **핵심 작업:**
    1.  코다리님: `roiCalculator.ts`와 `action_plan_service.py`를 통합하여 실제 데이터 연동 및 에러 핸들링을 완료합니다.
    2.  디자이너님/프론트엔드: 확정된 API 응답 구조에 맞춰, 모든 UI 컴포넌트를 완성도 높게 구현하고 Red/Green 강조 로직을 코드로 옮깁니다.

#### 🗓️ Phase 2: QA 및 사용자 경험 최적화 (Target: D+10일)
*   **목표:** 내부 테스트(QA)를 통해 데이터 흐름의 오류를 잡고, 소상공인 사용자가 가장 직관적으로 느낄 수 있는 '경험'을 만듭니다.
*   **핵심 작업:**
    1.  전체 팀: End-to-End 시나리오 기반으로 테스트 케이스(Test Case)를 작성하고 실행합니다. (예: "손실 비용이 0원일 때의 사용자 여정", "데이터 로딩 실패 시 메시지 처리")
    2.  Writer님: QA 피드백을 바탕으로, 사용자의 이탈률을 줄일 수 있는 카피라이팅(CTA 문구 수정)을 보강합니다.

#### 🗓️ Phase 3: 베타 서비스 배포 및 초기 사용자 확보 (Target: D+15일~)
*   **목표:** 실제 소상공인에게 플랫폼의 가치를 테스트하고, 개선점을 파악하며 초기 성과 지표(KPI)를 측정합니다.
*   **핵심 작업:**
    1.  개발팀: 배포 환경을 구축하고, 사용자 피드백 수집 채널(FAQ/버그 리포트)을 마련합니다.
    2.  마케팅/콘텐츠: 실제 베타 테스트 결과를 바탕으로 소셜 미디어 마케팅 콘텐츠를 확정하여 홍보에 활용합니다.
