# User Feedback Tracking Template (DRAFT)

**Purpose:** 표준화된 인터뷰 데이터 수집 및 분석을 위한 가이드라인과 체크리스트입니다. 소상공인의 진짜 Pain Point 를 깊이 있게 파악하고, 이를 KPI(재무적 불안정, 운영 비효율 등) 로 명확히 분류하여 실행 가능한 솔루션을 기획하는 데 사용됩니다.

---

## 1. 인터뷰 전 준비 (Pre-Interview Checklist)

| 항목 | 확인 내용 | 비고 |
| :-- | :-- | :-- |
| **대상** | 소상공인/가맹점주 대표자 (업종, 규모 등 기록 필요) | |
| **목표** | 이 인터뷰에서 반드시 알아내야 할 핵심 질문 3 가지 정리 | |
| **데이터** | 기존 고객 설문/피드백 데이터 확인 (반복되는 Pain Point 체크) | |
| **도구** | 녹음기, 필기 도구 또는 노션/노마드 메모 앱 설치 완료 | |

> 💡 *팁: 인터뷰 대상이 "지금 가장 중요한 고민은?" 이라고 먼저 물어보며 시작하세요. 공식적인 질문보다 자연스러운 대화를 유도하는 것이 좋습니다.*

---

## 2. 인터뷰 질문 가이드 (Key Questions for Pain Point Discovery)

### 1 단계: 현재 상황 및 어려움 파악
- **Q1:** "당신의 가게에서 가장 자주 겪는 스트레스나 고민은 무엇인가요?"
  - _관찰:_ 구체적인 사례를 들어달라고 요청하여 실제 상황을 이해합니다.
- **Q2:** "최근 매출이 감소하거나 손님이 줄어든 적이 있다면, 그 이유는 무엇이라고 생각하시나요?"
  - _목표:_ '재무적 불안정' 과 연결 가능한 데이터 포인트 확보.
- **Q3:** "가게 운영 과정에서 가장 불편하게 느껴지는 업무나 시스템은 무엇인가요?"
  - _관찰:_ '운영 비효율' 을 지목하는 키워드 (예: 재고 관리, 원가 계산, 고객 응대) 추출.

### 2 단계: 해결 시도 및 니즈 심층 파악
- **Q4:** "지금까지 이 문제를 해결하기 위해 어떤 방법이나 도구를 사용해 보셨나요? 효과가 있었는지?"
  - _분석:_ 기존 솔루션의 한계점 (가격, 복잡성, 비효율) 을 이해합니다.
- **Q5:** "이상적인 가게 운영 모습은 무엇인가요? 그 모습을 위해 얼마나 비용을 지불하실 의향이 있나요?"
  - _측정:_ Pain Point 와 지불 의사(WTP) 를 연결합니다.

### 3 단계: BDS 플랫폼 제안에 대한 반응 (Qualitative Feedback)
- **Q6:** (BDS 플랫폼의 특정 기능, 예: 'AI 진단 툴' 또는 '재무 관리 대시보드' 언급 시) "이런 기능이 있다면 도움이 될 것 같나요? 왜?"
  - _목표:_ 초기 아이디어 검증 및 개선 포인트 도출.

## 3. 인터뷰 후 데이터 분석 및 구조화 (Post-Interview Analysis)

인터뷰 녹음 및 필기를 바탕으로 다음 항목을 채워주세요. 각 Pain Point 는 아래 KPI 유형 중 하나로 분류합니다.

| 구분 | KPI 유형 정의 | 예시 키워드/사례 |
| :-- | :-- | :-- |
| **A. 재무적 불안정** | 매출 감소, 비용 증가, 자금 조달 어려움 등 금전적 손실과 관련된 문제 | "세금 계산이 복잡해", "원가 관리가 안 돼서 마진만 줄어듦" |
| **B. 운영 비효율** | 반복적인 업무, 잘못된 재고, 고객 응대 시간 낭비 등 프로세스 문제 | "매일 손님이 들어오면 주문을 적어야 해서 바빠", "재고가 부족해서 판매 기회 놓침" |
| **C. 마케팅/고객 유입** | 신규 고객 확보 어려움, 기존 고객 유지율 낮음 등 트래픽 관련 문제 | "광고비를 써도效果가 없어요", "손님이 찾아오지 않아서 고민이에요" |

### 4. 데이터 기록 및 저장 (Data Logging)
- **인터뷰 ID:** `[YYYYMMDD]_[Initials]_[업종]` (예: `260624_JK_카페`)
- **Pain Point 요약:** _한 문장으로 핵심 문제 정의_
  > 예: "재고 관리로 인한 매출 손실과 원가 불명확으로 인한 수익 감소"
- **KPI 분류:** `[A. 재무적 불안정] / [B. 운영 비효율]` (복합일 수 있음)
- **BDS 솔루션 연상 키워드:** _이 Pain Point 를 해결할 수 있는 BDS 기능 제안_

> 💡 *팁: 인터뷰마다 'BDS 플랫폼의 핵심 가치' 와 연결되는 아이디어가 떠오르면 바로 메모하세요. 예를 들어, "재무 불안정" 이라면 'AI 재무 분석 대시보드' 가 도움이 될지 고려해보세요.*
