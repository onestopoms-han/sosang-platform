# 🚀 BDS 소상공인플렛폼 MVP 출시: 최종 GTM 실행 계획 및 KPI 설정서

**작성일**: 2026. 6. 18  
**작성자**: 현빈 (비즈니스 전략가)  
**승인 대기**: CEO / 개발팀 (코다리)

## 1. 목표
MVP 출시 직후, 베타테스터를 통한 초기 사용자 반응을 최적화하고 플랫폼의 핵심 가치(안정적 공급망, AI 기반 실직 대안) 를 검증하는 것입니다. 이 계획은 **베타테스터 모집 캠페인**, **온보딩 UX 전략**, **초기 KPI**로 구성됩니다.

## 2. 베타테스터 모집 및 초기 반응 예측 (GTM Strategy)

### 2.1 대상 및 채널
- **타겟**: 기존 소상공인 커뮤니티(카카오톡 오픈채팅, 네이버 블로그), 지자체 소규모 사업장 (30~50명 기준)  
- **주요 채널**: 
    - Instagram 릴스: "AI 가 당신의 가게를 분석해드립니다" (Reel 15초, 해시태그 #소상공인 #AI #비즈니스)
    - 오픈채팅/카카오스타트: "베타테스터 초청회" 이벤트 (입장료 무료, AI 진단서 발급)
- **인센티브**: 초기 30일 동안 플랫폼 사용 시 `AI 매출 분석 리포트` 무료 제공 + 성공 사례 스터디 참여 기회

### 2.2 초기 반응 예측 시나리오 (Heatmap 기반)
베타테스터들이 온보딩을 완료한 후, 다음 행동이 예상됩니다:
- **핵심 포인트**: AI 가 제안하는 `매출 최적화 팁` 적용 → 1주일 내 매출 변동 추이 확인  
- **예상 Heatmap 패턴**: 
    - **온보딩 페이지**: "AI 진단서 받기" 버튼 클릭률 40% 이상 (반응형)
    - **데이터 입력**: 공급망 데이터 입력 시 `PainGauge` 연동으로 자동 채워지는 경험 → 이탈 방지  
- **예상 반응 지표**: 
    - **Churn Rate(이탈율)**: 15% 이내 (AI 진단서와 초기 성공 사례로 유지)
    - **NPS(추천도)**: 60 이상 (실제 매출 개선 효과 입증 시 70 이상으로 상승)

## 3. 핵심 KPI 설정 및 추적 변수

| KPI | 목표치 | 측정 방법 | 추적 주기 |
|-----|--------|-----------|----------|
| **AOV(평균 주문 가치)** | $150 | 결제 시스템 연동 데이터 | 일별 |
| **온보딩 완료율** | 70% | 사용자 행동 로그 | 실시간 |
| **AI 추천采纳率** | 30% | AI 제안 vs 실제 선택 비교 | 주간 |
| **NPS(추천도)** | 60+ | 온보딩 후 설문조사 | 월간 |

### KPI 추적 변수 명세서 (PM_Dashboard_Status_Variables_v1.md 참조)
- `user_onboarding_complete`: boolean  
- `ai_recommendation_click_rate`: float  
- `pain_gauge_data_sync_status`: enum (synced, pending, failed)  

## 4. 개발팀 스프린트 우선순위 승인 요구사항

### 4.1 PainGauge 데이터 연동 강화
- **요구**: 온보딩 페이지에서 사용자 Pain Point 를 실시간으로 분석하여 `PainGauge` 데이터를 자동 연동  
- **기술적 구현**: 
    - API 엔드포인트: `/api/pain-gauge/v1/sync`  
    - 응답 데이터: `{ "score": 70, "recommendations": ["재고 관리", "마케팅"] }`  
- **연동 우선순위**: MVP 출시 전 100% 연동 완료  

### 4.2 온보딩 UX 개선
- **요구**: AI 진단서 발급 후 `PainGauge` 데이터를 기반으로 맞춤형 가이드 제공  
- **기술적 구현**: 
    - 프론트엔드: 반응형 UI 로 PainGauge 데이터 시각화 (차트, 그래프)  
    - 백엔드: `/api/onboarding/v1/guide/{user_id}` 엔드포인트 추가

# 🛠️ BDS 소상공인플렛폼: 개발 스프린트 우선순위 승인서 (GTM 연계)

**요청자**: 현빈 (비즈니스 전략가)  
**수신팀**: 코다리 (개발팀)  

## 1. 승인 배경
MVP 출시를 위한 GTM 실행 계획과 초기 KPI 설정이 완료되었습니다. 개발팀은 이 계획에 맞춰 **PainGauge 데이터 연동 강화** 및 **온보딩 UX 개선**을 다음 스프린트 우선순위로 선정합니다.

## 2. 구체적 기술 요구사항

### 2.1 PainGauge API 연동 (엔드포인트)
- **GET `/api/pain-gauge/v1/sync`**: 
    - 입력: `user_id`, `business_type`  
    - 출력: `{ "score": int, "categories": ["재고", "마케팅"] }`  
- **POST `/api/pain-gauge/v1/analyze`**: 
    - 입력: `business_data` (JSON)  
    - 출력: `pain_analysis_result`  

### 2.2 온보딩 UX 개선
- **AI 진단서 발급 후**: 
    - 프론트엔드: PainGauge 데이터를 차트로 시각화 (`<div class="pain-gauge-chart">`)  
    - 백엔드: `/api/onboarding/v1/guide/{user_id}` 엔드포인트 추가  
- **반응형 UI**: 
    - 모바일 우선 설계 (사용자 행동 로그 수집)  
    - `PainGauge` 데이터 시각화 시 `d3.js` 또는 `Chart.js` 사용  

### 2.3 MVP 출시 전 필수 작업
- **온보딩 페이지**: AI 진단서 발급 → PainGauge 연동 → 맞춤형 가이드 제공  
- **결제 시스템**: AOV(평균 주문 가치) 추적 변수 추가 (`/api/payment/v1/aov`)  
- **이탈 방지**: Churn Rate(이탈율) 모니터링을 위한 로그 수집
