# 🚀 BDS 소상공인플렛폼: 다음 스프린트 마스터 플랜 (V1.0)

**작성일**: 2026년 6월 18일  
**담당**: 영숙(PA), 현빈(비즈니스), 코다리(개발)  
**목표**: MVP 출시 직후, 베타테스터를 통한 초기 가치 검증 및 핵심 지표(KPI) 달성

---

## 💡 스프린트 목표 요약 (THE WHY)
*   **최종 목표:** 소상공인 대상의 실질적인 '매출 최적화' 경험을 제공하여 서비스 신뢰도와 사용률 증대.
*   **핵심 활동:** 베타테스터 모집 및 온보딩 -> AI 진단서 발급 → 플랫폼 핵심 기능 체험(데이터 연동) -> 매출 데이터 확보.

## 🎯 I. 비즈니스/제품 플랜 (현빈 역할 중심)
| 항목 | 목표 / 액션 아이템 | KPI 측정 변수 | 산출물 기대치 |
| :--- | :--- | :--- | :--- |
| **베타 모집 실행** | 초기 소상공인 커뮤니티(오픈채팅, 네이버 블로그) 채널을 통한 공격적 베타테스터 모집. (인센티브: AI 진단서 무료 제공) | 온보딩 완료율 (목표 70%) | 홍보 콘텐츠 최종본 / 참여자 DB 구축 |
| **온보딩 최적화** | 초기 사용자에게 'AI 진단서'를 통해 즉각적인 성공 경험(Aha Moment)을 제공하는 플로우 확정. | AI 추천 채택률 (목표 30%) | 온보딩 페이지 UX 최종 검토 / 대화 시나리오 DB 업데이트 |
| **수익 모델 준비** | 계약 조건 및 결제 시스템 연동에 필요한 법적, 기술적 요구사항 우선순위 재검토 및 확정. | AOV(평균 주문 가치) ($150+) | 결제 게이트웨이 API 명세서 초안 |

## 💻 II. 개발/기술 플랜 (코다리 역할 중심 - Technical Stack)
| 항목 | 목표 / 액션 아이템 | 우선순위 | 필요 자원/개발 모듈 |
| :--- | :--- | :--- | :--- |
| **핵심 기능 안정화** | Mock API에서 Live Data로의 End-to-End 시나리오 테스트를 완료하고, 통합 위험 요소를 제거하여 MVP 출시 준비 완료. | 🥇 P1 (최우선) | 사용자 데이터 흐름 관리 모듈 / 인증 시스템 개선 |
| **데이터 연동 및 UI/UX** | 캐릭터 컴포넌트의 애니메이션과 Mock API 데이터를 동기화하고, 최종적으로 베타 테스트에 필요한 모든 화면 구현 완료. | 🥈 P2 | 비주얼 일관성 유지 (Pixel Consistency) / 데이터 바인딩 로직 수정 |
| **회원 및 인증 시스템** | 베타테스터 규모 증가를 대비하여, 안정적이고 확장 가능한 회원가입/로그인(OAuth 연동 포함) 모듈을 구축. | 🥉 P3 | 사용자 인증 API 엔드포인트 / 비밀번호 정책 구현 |
