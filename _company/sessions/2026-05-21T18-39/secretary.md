## 📄 개발 로드맵 및 리스크 관리 계획 초안 (CEO 보고용)
**작성자:** 영숙 (비서/PA)
**목표:** Tier 1 핵심 기능을 단계별로 구현하여, 소상공인에게 '실행 가능한 다음 행동'을 제공하는 사이클 검증.

### 🚀 I. 개발 목표 및 원칙 재확인
*   **최우선 가치:** 기능의 완성도보다 **‘진단-액션 제시’가 작동하는 사용자 흐름(Flow) 구현**에 집중합니다.
*   **범위 제한 (Scope):** 오직 현빈님이 확정한 3가지 핵심 기능만 진행하며, 다른 부가 기능은 다음 버전으로 연기합니다.

### 🗓️ II. 단계별 개발 일정 및 목표 (Roadmap)

| Phase | 기간 예상 | 주요 목표 / 스토리텔링 | 핵심 구현 범위 (Deliverables) | 담당 에이전트 |
| :---: | :---: | :--- | :--- | :--- |
| **Phase 0** | 1주 차 | **[기반 구축] 진단 엔진의 데이터 입력 및 구조화**<br>→ 시스템이 소상공인의 데이터를 이해하는 기초 레이어 완성. | 🔸 `Diagnosis Input Schema` 확정 및 백엔드 API Stub 구현.<br>🔸 필수 데이터 필드(업종, 매출 패턴 등)를 받아 처리하는 초기 파이프라인 구축. | Coder (Backend) |
| **Phase 1** | 2~3주 차 | **[MVP 완성] Pain Point 진단 및 시각화**<br>→ 소상공인이 자신의 문제점을 수치로 확인하고 충격받는 경험 제공. (가장 강력한 후크)<br>**Tier 1 - [AI 진단 툴] 구현 완료.** | 🔸 백엔드: 데이터 분석 로직(Rule-based Logic)을 통한 '문제점 도출' API 완성.<br>🔸 프론트: 진단 결과 리포팅 페이지 UI/UX 및 API 연동. | Coder, Designer |
| **Phase 2** | 4~5주 차 | **[액션 제시] 구체적 개선 로드맵 제공**<br>→ 문제점을 인식한 사용자에게 즉시 실행 가능한 해답을 제시하여 플랫폼의 효용성 극대화.<br>**Tier 1 - [액션 플래너] 구현 완료.** | 🔸 백엔드: 진단 결과 기반으로 '개선 행동' 추천 로직 API 완성.<br>🔸 프론트: 액션 로드맵 페이지 및 단계별 체크리스트 UI/UX. | Coder, Designer |
| **Phase 3** | 6주 차 이후 | **[유지 및 확장] 실행 계획 관리 및 데이터 연동**<br>→ 사용자의 지속적인 참여와 플랫폼에 대한 의존성(Sticky Factor) 증가.<br>**Tier 1 - [실행 계획] 구현 완료.** | 🔸 사용자 인증 기반의 '나만의 체크리스트' 기능 (CRUD).<br>🔸 외부 데이터를 연결할 수 있는 API 게이트웨이 설계. | Coder, Designer |
