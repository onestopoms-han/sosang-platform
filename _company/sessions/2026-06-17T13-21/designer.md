### 📄 개발 핸드오버 패키지: 옆집선배 캐릭터 컴포넌트 (Final Handover Package)

이 문서는 '옆집선배' 캐릭터가 플랫폼의 핵심 페이지에 적용될 때 필요한 모든 시각적, 상호작용적 정의를 포함합니다.

#### 1. 디자인 시스템 및 스타일 정의 (Visual & Style Guide)

**캐릭터 이름:** 옆집선배 (Next-door Senior)
**역할:** 데이터 기반 컨설팅을 안내하는 '신뢰성 있는 멘토'
**디자인 원칙:** 미니멀한 라인 아트 준수, 신뢰감 및 성장을 시각화.

| 속성 | 정의 | 상세 설명 |
| :--- | :--- | :--- |
| **주요 컬러 팔레트** | Deep Blue (`#004D66`), Growth Green (`#3CB371`) | 브랜드 시스템의 핵심 색상을 캐릭터의 강조 요소에 적용. |
| **라인 스타일** | 두께 2px, 모서리 둥글림 (8px) | 미니멀 라인 아트 원칙 준수. 시각적 안정성 확보. |
| **상태별 컬러** | `Default`: Deep Blue, `Active/Growth`: Growth Green | 컨설팅 상태(안내 중, 성공 지표 제시)에 따라 색상 변화 정의. |

#### 2. 컴포넌트 배치 및 레이아웃 (Placement & Layout Specification)

캐릭터는 플랫폼의 신뢰도와 성장 로드맵을 시각적으로 강조하는 위치에 전략적으로 배치되어야 합니다.

| 페이지 섹션 | 권장 위치 | 목적 및 효과 | 스타일링 적용 방식 |
| :--- | :--- | :--- | :--- |
| **온보딩 (Onboarding)** | 화면 중앙 상단 또는 시작 시 환영 메시지 근처 | 신뢰 구축, 초기 참여 유도 | 정적인 모습으로 '환영' 메시지와 함께 배치. Deep Blue 배경 위에서 명확하게 부각. |
| **데이터 분석 섹션** | KPI(핵심 지표) 카드 옆 또는 데이터 흐름 설명 박스 내부 | 컨설팅의 결과에 대한 신뢰 부여, 성장 경로 시각화 | Growth Green을 사용하여 긍정적 성과를 강조하는 애니메이션 적용. |
| **성장 로드맵 (Journey Map)** | 여정 단계별 전환점(Milestone) 사이에 배치 | 각 단계의 의미 부여 및 동기 부여 | 각 단계별로 캐릭터가 다음 단계로 안내하는 듯한 순차적 애니메이션 구현. |

#### 3. 인터랙션 및 애니메이션 정의 (Interaction & Animation Definition)

개발팀이 구현해야 할 구체적인 상호작용 로직입니다.

| 이벤트/상태 | 애니메이션 상세 정의 | 기술 요구사항 (Code Trigger) |
| :--- | :--- | :--- |
| **페이지 진입 시** | 캐릭터가 서서히 등장하며(Fade-in), 미니멀한 인사 동작(Head nod) 수행. | `onload` 또는 컴포넌트 마운트 시점 트리거. CSS Transition 사용. |
| **데이터 강조 시** | 특정 KPI 카드에 마우스를 올리거나 데이터가 로드될 때, 캐릭터가 성장 색상(`Growth Green`)으로 전환되며 지표를 가리키는 동작. | API 응답 성공 후 (Success State)에 따라 클래스 변경 및 애니메이션 실행. |
| **온보딩 완료 시** | 최종 안내 메시지와 함께 캐릭터가 미소 짓는(Smile) 마무리 포즈. | 최종 단계 완료 후 트리거. 사용자에게 성취감을 전달. |

#### 4. 개발 핸드오버 요약 (Developer Summary Checklist)

1.  **Asset:** 캐릭터의 기본 라인 아트 및 컬러 정의 파일 (`Character_Integration_Guide.md`) 참조.
2.  **Component:** `NextDoorMentor` 컴포넌트 설계 완료.
3.  **Styling:** Deep Blue/Growth Green을 이용한 동적 스타일링 로직 확정.
4.  **Animation:** 페이지별 배치에 따른 3가지 핵심 애니메이션(등장, 강조, 마무리)의 정확한 CSS/JS 트리거 정의 완료.
5.  **Data Flow:** 캐릭터 상태 변화(`Default` -> `Growth`)가 백엔드 데이터와 완벽히 동기화되도록 API 응답 기반 로직 설계.
