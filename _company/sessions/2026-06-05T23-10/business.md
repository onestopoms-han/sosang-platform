## 1. 모델 개요

StablePay Subscription 은 소상공인에게 **월정액 구독료**를 지불하면 거래 수수료율을 고정 할인해주고, 결제 실패 시 플랫폼이 보상해주는 '안정성 기반 자금 중개' 서비스입니다. 기술 구현 명세 (API 구조, 데이터 포맷) 와 디자인 시스템의 일관성을 확보한 후 개발팀에 전달합니다.

### 핵심 가치
- **기술적 안정성 (Success Rate 95%, Retry Rate 80%)** 을 기반으로 한 신뢰도 제공  
- **심플한 가격 정책** (월정액 + 거래량 할인) 으로 진입 장벽 제거  
- **리스크 분산** (보상금 로직) 으로 소상공인의 불안감 해소

## 2. 수익화 전략 및 초기 KPI

### 2.1 수익 구조
| 항목 | 내용 | 예상 기여도 |
|------|------|-------------|
| 구독료 (월정액) | 기본 5,000원 / 거래량별 할인 (예: 월 거래 > 1백만 원 시 3,500원) | 40% |
| 수수료 할인 수익 | 고정 수수료 1.8%, 변동 수수료 0~0.5% → 차익금 | 30% |
| 포인트 적립 | 거래량 기반 포인트 (예: 1천 원당 0.5%) 를 멤버십 등급으로 전환하여 프리미엄 결제 처리 | 20% |
| 파트너십 중개비 | 금융기관, 카드사, 클라우드 제공자와의 제휴 수수료 | 10% |

### 2.2 초기 KPI 목표 (3 개월)
| 지표 | 목표값 | 측정 방법 |
|------|--------|-------------|
| 활성 구독자 수 (MAU) | 1,000 소상공인 | 플랫폼 내 가입/결제 로그 |
| 평균 거래당 수수료 절약액 | > 200원 | (고정 수수료 - 실제 청구량) × 거래건수 |
| 리텐션율 (월별) | > 85% | 구독 기간 3 개월 이상 유지 비율 |
| 결제 실패 보상 처리 시간 | < 24 시간 | 보상금 지급 요청부터 완료까지 평균 시간 |

## 3. 리스크 관리: 보상금 계산 로직 초안

기술 안정성 지표 (Retry Rate 80%, Success Rate 95%) 와 연동하여, 플랫폼이 대신 지급하는 '보상금'의 공식은 다음과 같습니다.

### 기본 공식
```
보상금 = 실패 거래액 × (1 - 성공률) / 평균 성공률 × 할인율
```

- **실패 거래액**: 특정 기간 (예: 월) 에 발생하지 않은 실제 결제 금액  
- **성공률**: 해당 기간의 기술적 안정성 지표 (예: 95%)  
- **평균 성공률**: 플랫폼 전체 역사적 데이터 기반 (예: 90%)  
- **할인율**: 구독 등급에 따라 1.0 ~ 2.0 배 적용  

### 예시 계산
| 시나리오 | 실패 거래액 | 성공률 | 평균 성공률 | 할인율 | 보상금 |
|----------|-------------|---------|--------------|--------|---------|
| 정상 (95% 성공) | 10,000 원 | 95% | 90% | 1.0 | 500 원 |
| 고난도 (80% 성공) | 20,000 원 | 80% | 90% | 1.5 | 4,500 원 |

### 로직 구현 조건
- **기술 안정성 지표**가 실시간으로 모니터링되어야 하며, 목표치 (Success Rate < 90%) 를 넘을 경우 보상금이 자동 지급됩니다.  
- **보상금 지급 주기**: 월 말 또는 거래 실패 발생 즉시 (최대 24 시간 내)  
- **개발팀 요청 사항**: API 호출 시 'Retry Rate'와 'Success Rate'를 포함하는 JSON 포맷 (`{"retry_rate": 0.8, "success_rate": 0.95}`) 을 전달하고, 해당 지표가 임계치를 넘을 때 보상금 로직이 트리거되도록 설계해야 합니다.
