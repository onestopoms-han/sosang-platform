# Beta Test Scenario Scripts

## Scenario 1: AI 진단서 수령 및 초기 인식 (Trust Widget Focus)
**Goal:** 사용자에게 즉각적인 신뢰감을 제공하고, 다음 단계로 나아가도록 유도한다.
**User Action:** 소상공인 데이터 입력 후 'AI 진단서' 버튼 클릭.
**System Response:** AI 모델이 데이터를 분석하여 핵심 문제점(Pain Point) 3가지와 즉각적인 개선 방향(Actionable Tip)을 제시하는 대시보드 화면 표시.
**Key Interaction:** 결과 페이지에 Trust Widget (예: "XX% 신뢰도 기반", "전문가 검증 완료")를 시각적으로 강조하여 배치.

## Scenario 2: 매출 최적화 팁 수령 및 실행 (PainGauge Focus)
**Goal:** 사용자가 제시된 팁을 실제 행동으로 연결하고, 그 효과를 측정하게 한다.
**User Action:** AI 추천 '매출 최적화 팁' 중 하나 선택 후, 해당 팁을 적용하는 가이드라인 확인.
**System Response:** 구체적인 실행 단계(Step-by-step guide)와 예상되는 매출 변화 시뮬레이션 화면 제시.
**Key Interaction:** PainGauge (예: "현재 상태 대비 잠재적 개선액", "다음 단계 목표 달성률")를 통해 사용자가 감정적/수치적 동기 부여를 받도록 설계.

## Scenario 3: 실제 사례 연결 및 경험 공유 (Storytelling Focus)
**Goal:** 사용자의 성공 경험을 플랫폼에 기여하도록 유도한다.
**User Action:** '내 경험 공유' 버튼 클릭 후, 자신의 성공 사례(매출 증대 과정 등)를 스토리 형태로 입력.
**System Response:** 입력된 내용을 바탕으로 AI가 이를 정제하여 다른 소상공인에게 영감을 줄 수 있는 구조화된 콘텐츠 초안 생성.
**Key Interaction:** 최종 결과물에 시각적 일관성을 유지하며, 사용자가 쉽게 공유할 수 있도록 템플릿 형태로 제공.

 30	
 31	### 목적
 32	AI 진단서 생성 시, 해당 소상공인의 업종별·현황에 맞춘 **즉시 실행 가능한 매출 최적화 팁 (Action Item)** 을 함께 전달하여 사용자의 전환율 (Conversion Rate) 및 사용자 만족도를 극대화합니다. 기존 단순 진단 결과 전달을 넘어 **'고민 → 해결책 제시'** 스토리텔링의 핵심 단계입니다.

 34	### 역할
 35	- **백엔드**: `/api/v1/diagnosis/result` 엔드포인트를 통해 JSON 형태로 `DiagnosisResult` 객체 (진단서 포함) 와 함께 `ActionItemsList` 를 반환.
 36	- **프론트엔드**: 진단서 데이터를 받아 표시함과 동시에, 하단에 '추천 액션' 섹션을 렌더링하여 사용자가 클릭 가능한 액션으로 유도.

 40	## 2. 엔드포인트 정의
 41	
 42	### POST `/api/v1/diagnosis/result`
 43	
 44	**요청 헤더 (Header)**
 45	```json
 46	Content-Type: application/json
 47	Authorization: Bearer {API_KEY}
 48	```
 49	
 50	**요청 바디 (Request Body)**
 51	진단서 생성을 위한 기본 입력 데이터 (업종, 매출 규모 등) 를 포함하며, `diagnosis_id` 를 통해 특정 결과와 연결됨.
 52	
 53	| 필드명 | 타입 | 필수 | 설명 | 예시 |
 54	| :--- | :--- | :---: | :--- | :--- |
 55	| `diagnosis_id` | string (UUID) | O | 진단서 식별자 | `"d-12345678-abcd-efgh"` |
 56	
 57	**응답 바디 (Response Body)**
 58	- **성공 (HTTP 200 OK)**: JSON 형식의 결과 객체.
 59	- **실패 (HTTP 4xx/5xx)**: 오류 메시지 및 상태 코드.

106	## 3. 데이터 스키마 (JSON Schema)
107	
108	### ActionItem 구조체 (`ActionItemsList` 배열 내 객체)
109	```json
110	{
111	  "$schema": "http://json-schema.org/draft-07/schema#",
112	  "title": "ActionItem",
113	  "type": "object",
114	  "properties": {
115	    "id": { "type": "integer" },
116	    "title": { "type": "string", "maxLength": 50 },
117	    "description": { "type": "string" },
118	    "confidence_score": {
119	      "type": "number",
120	      "minimum": 0,
121	      "maximum": 1,
122	      "description": "LLM 이 해당 액션의 효과를 예측한 신뢰도 점수"
123	    },
124	    "expected_roi": {
125	      "type": "object",
126	      "properties": {
127	        "time_saved_per_month_hours": { "type": "number", "minimum": 0 },
128	        "revenue_increase_percent": { "type": "number", "minimum": 0 }
129	      }
130	    },
131	    "steps": {
132	      "type": "array",
133	      "items": {
134	        "title": "Step",
135	        "type": "object",
136	        "properties": {
137	          "step_number": { "type": "integer" },
138	          "action": { "type": "string" },
139	          "estimated_cost": { "type": "number", "minimum": 0 }, // 비용은 0 이 가능 (무료 액션)
140	          "duration_minutes": { "type": "integer" }
141	        }
142	      }
143	    },
144	    "cta_label": { "type": "string", "description": "프론트엔드에서 표시될 버튼 텍스트" }
145	  },
146	  "required": ["id", "title", "description", "confidence_score", "steps"]
147	}
148	```
