#### 1. 외부 데이터 수집 실패 원인 분석 (Technical Analysis)
이전에 시도했던 `comment_harvester.py`와 같은 웹 크롤링 방식은 본질적인 한계에 직면해 있습니다. 특히 소셜 미디어 플랫폼의 경우, 다음과 같은 이유로 단순 스크래핑은 지속 불가능합니다.

| 실패 원인 (Potential Cause) | 상세 설명 및 영향도 | 해결 방안 (Recommendation) |
| :--- | :--- | :--- |
| **CAPTCHA/Anti-Bot Measures** | 플랫폼 측에서 비정상적인 대량 접근을 차단하는 가장 일반적이고 강력한 수단입니다. IP 주소나 브라우저 지문(Fingerprinting) 분석으로 차단됩니다. (높음) | 로봇이 아닌 인간처럼 행동하는 **Headless Browser 서비스**(예: Selenium Grid, Playwright) 사용 또는 유료 API 접근 필요. |
| **API 정책 변경** | 소셜 미디어 플랫폼은 내부 정책을 수시로 변경합니다. 공개된 데이터가 아닐 경우, 공식적인 데이터 추출 경로(API Endpoint) 자체가 막히거나 비용이 발생합니다. (높음) | **공식 파트너십/유료 API 사용**이 유일한 해결책입니다. 비정형 데이터를 얻기 어렵습니다. |
| **데이터 구조 변경 (Layout Shift)** | 웹사이트 디자인이나 HTML 클래스 이름이 사소하게만 바뀌어도, 크롤러가 특정 요소를 찾지 못해 전체 스크립트가 멈추는 경우가 빈번합니다. (중간) | *재활용성 낮은* 크롤링은 포기하고, **구조화된 데이터(Spreadsheet, CSV)**를 제공하는 외부 리포트를 구매/제휴해야 합니다. |

**▶ 결론:** 단순 코딩 기반의 웹 스크래핑으로는 '최신'이고 '대규모'이며 '정확한' 데이터를 확보하기 어렵습니다. **우리는 이제 데이터 수집가(Harvester) 역할에서, 데이터를 구조화하고 가치를 창출하는 분석가(Synthesizer) 역할로 포커스를 전환해야 합니다.**

#### 2. 손실 최소화 Pain Point 및 성공 공식 재수집 요약 (Insight Synthesis)
기존에 확보된 소상공인 데이터와 트렌드를 '손실 최소화'라는 단 하나의 관점으로 필터링하여, 즉시 콘텐츠와 기능으로 연결할 수 있도록 구조화했습니다.

**🔍 핵심 통찰: 소상공인의 가장 큰 손실은 '시간', '자본', 그리고 '신뢰(지속성)'입니다.**
우리가 제공해야 할 것은 단순한 정보가 아니라, 이 3가지 손실을 막아주는 **'시스템적 확신'**입니다.

| Pain Point 카테고리 (손실 지점) | 구체적인 고통 사례 (The Loss) | 관련 데이터/트렌드 근거 | 시스템 솔루션 가치 (Success Formula) |
| :--- | :--- | :--- | :--- |
| **재무적 손실 (Cash Flow)** | 💸 예측 불가능한 매출, 재고 과잉/부족으로 인한 자금 경색. 복잡하고 느린 정부 지원금 신청 과정에서 시간과 기회를 잃음. | *[근거]* 고금리 지속 및 경기 침체 우려 증대 (2026-05-18 보고서 반영). / **WTP 데이터:** '정확한 자금 흐름 예측'에 높은 지불 의사 표출. | **Loss Minimization:** AI 기반의 *선제적* 재고/현금흐름 최적화 제안 (Predictive Analytics). **(Pro Tier 핵심)** |
| **운영 효율성 손실 (Time & Effort)** | 💻 마케팅 채널별 콘텐츠 기획 및 배포에 너무 많은 시간과 인력이 투입됨. 개개인의 역량에 의존하여 시스템화가 어려움. | *[근거]* 디지털 전환의 어려움, 1인 운영자의 과부하 문제 (2026-05-22 보고서). / **Pain Point:** '시간 대비 낮은 마케팅 효율'이 가장 높은 불만점. | **Loss Minimization:** 전 채널에 걸친 *자동화된* 콘텐츠/운영 계획 수립 및 배포 시스템 제공. (AI Native System 핵심) |
| **시장 신뢰 손실 (Trust & Visibility)** | 🌐 온라인 상에서 '나만의 색깔'을 내기 어려움. 대형 플랫폼과 경쟁하며 노출 기회를 놓치거나, 소비자가 진정성을 느끼지 못함. | *[근거]* 로컬 커뮤니티의 중요성 부각(나들가게 트렌드). / **Pain Point:** '브랜드 스토리텔링의 어려움' (2026-05-19 보고서 반영). | **Loss Minimization:** 소상공인의 *진짜 이야기*를 구조화하고, 이를 통해 꾸준히 소비자에게 노출시키는 독점적인 플랫폼 기능.
