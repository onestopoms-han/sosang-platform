# BDS소상공인플렛폼 - 캐릭터 아트워크 및 시스템 구현 Asset Map (V1.0)

## 1. 목적
Writer 와 Designer 가 기획한 '비전(Vision)' 캐릭터를 실제 웹/앱 환경에 적용하기 위한 **최소한의 기술적 요구사항**과 필요한 **디지털 자산(Asset) 목록**을 정의합니다. 개발팀이 프론트엔드 컴포넌트를 구현하고, 디자인팀이 아트워크를 생산할 수 있는 기준입니다.

## 2. 기술 스택 및 구현 방식 (Tech Stack & Implementation)

### 2.1 렌더링 전략
- **프론트엔드 라이브러리**: `Lottie-react` 또는 `react-lottie-web` 사용 권장.
    - **이유**: SVG 기반의 벡터 애니메이션을 효율적으로 렌더링하며, 크기가 작고 확대/축소 시 깨지지 않음. 캐릭터가 단순히 정지한 이미지가 아니라 '말하는' 듯한 표현 (입동작, 눈썹 등) 을 구현할 때 필수적임.
- **대안 (Canvas API)**: 매우 복잡한 배경이나 실시간 데이터 시각화가需要同시进行时 Canvas 를 사용해야 할 수 있으나, 초기에는 Lottie 로 충분함.

### 2.2 이미지 포맷
- **정지 이미지 (Static Assets)**: `WebP` 또는 고품질 `PNG` 사용.
    - WebP 는 PNG 대비 30% 이상 용량 감소 가능하고, 웹 브라우저 지원이 완벽함.
    - 해상도는 `4x` 또는 `2x` 로 제작하여 Retina 디바이스에서 선명하게 표시되도록 함 (`1080px x 1080px` 기준).
- **배경 이미지**: 플랫폼의 전체적인 분위기를 조성하는 배경은 `WebP` (lossless) 또는 `JPG` 사용.

### 2.3 음원 및 음성 처리
- **음향 효과**: 캐릭터가 말하는 동안 발생하는 '입술 움직임'과 연동되는 배경음악 (BGM) 이 필요할 수 있음.
    - `.mp3`, `.wav` 포맷 사용. BGM 은 `low volume` 로 설정하여 텍스트 읽는 목소리에 방해되지 않도록 함.
- **음성 합성**: 캐릭터의 대화 내용은 외부 TTS API 를 호출하거나, 미리 녹음된 음원 파일 (`.mp3`) 을 동적으로 재생할 수 있도록 설계됨.

## 3. 자산 (Asset) 목록 및 명세 (Asset List & Specs)

### 3.1 시각적 방향 가이드라인 (Visual Direction Guidelines)
디자인팀이 아트워크를 제작할 때 반드시 준수해야 할 기준입니다.

- **컨셉 적용**: `The Growth Partner` (친근함 + 신뢰) 컨셉을 기본으로 하되, 상황에 따라 `The Trustworthy Analyst`의 엄격함을 부분적으로 반영 가능.
    - **기본 색상 (Primary)**: `#1A4363` (신뢰), `#FFC759` (지혜/가치). 이 색상을 캐릭터의 옷이나 배경에 활용.
    - **부수색 (Secondary/Accent)**: `#4DB6AC`, `#FFB79D` 등 따뜻하고 부드러운 톤 사용.
- **이미지 스타일**: **미니멀리즘 라인 아트**와 **부드러운 곡선**을 결합. 캐릭터의 외형은 단순하지만, 표정이나 동작으로 감정을 표현할 수 있도록 충분한 디테일을 포함.

### 3.2 필수 아트워크 (Essential Artwork)
다음 자산은 최소한으로라도 제작되어야 합니다.

| 자산명 | 용도 | 형식 | 해상도/규격 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Character_Icon** | 앱 아이콘, 프로필, 로고 | SVG / WebP | 512x512 (WebP), 640x640 (SVG) | 단순화된 버전. 배경 제거 필요. |
| **Character_Idle** | 메인 페이지, 대기 화면 | WebP | 800x600 | 캐릭터가 편안하게 서 있는 모습. |
| **Character_Talk** | 대화 중, 답변 제공 시 | Lottie (JSON) | 1280x720 (Max) | 입동작 포함. 3-5 초 사이클. |
| **Character_Alert** | 리스크 경고, 긴급 알림 시 | Lottie / SVG + GIF | 1280x720 | 눈이 커지거나 손가락을 가리키는 동작. |
| **Character_Success** | 성공 사례, 인증 완료 시 | Lottie / WebP | 1280x720 | 환하게 웃거나 박수치는 동작. |

### 3.3 UI 컴포넌트 연동 (UI Integration)
캐릭터가 단순한 이미지를 넘어 시스템의 일부로 기능하기 위한 요구사항입니다.

- **상태 전환 (State Transition)**: 캐릭터는 `Idle` → `Talking` → `Alert` → `Success` 등 상태별로 다른 애니메이션을 보여야 함.
    - 프론트엔드 코드에서 현재 상태를 (`idle`, `talking`, etc.) 저장하고, 해당 상태를 매칭하는 Lottie 파일의 URL 을 동적으로 로드.
- **상호작용 (Interaction)**:
    - 캐릭터를 마우스/터치로 클릭하면 반응해야 함. 클릭 시 소리가 나거나, 간단한 애니메이션이 실행됨.
    - `onClick` 이벤트 처리 로직은 프론트엔드 컴포넌트에 구현될 것.
- **반응형 (Responsive)**: 모든 화면 크기에서 캐릭터가 잘라지거나 왜곡되지 않도록 CSS Grid/Flexbox 와 Lottie 의 `loop`, `autoplay` 속성 등을 활용.

## 4. 개발 로직 및 API 연동 요구사항 (Dev Logic & API)

### 4.1 프론트엔드 컴포넌트 구조
- **컴포넌트 이름**: `<BDSCharacter />` 또는 `<VisionAvatar />`
- **입력값 (Props)**: `state`, `onSpeak`, `isLoading` 등.
    - `state`: 현재 캐릭터 상태 (`idle`, `talking`, etc.).
    - `onSpeak`: 사용자가 말한 것을 받아서 캐릭터의 입술 움직임을 제어하는 함수.
- **렌더링**: Lottie 파일의 URL 을 props 로 전달하거나, 내부에서 동적으로 로드.

### 4.2 백엔드 API 요구사항 (Backend)
백엔드는 캐릭터가 말하는 내용을 생성하고, 적절한 음원을 제공하는 역할을 할 수 있습니다.

- **음성 합성 API**:
    - TTS (Text-to-Speech) 가 필요한 경우, 외부 API 또는 자체 엔진을 호출.
    - 응답 데이터: `{ text: "오늘 매출이 20% 올랐어요!", emotion: "happy", audio_url: "..." }`
- **상태 업데이트**: 캐릭터의 현재 상태 (`idle`, `alert` 등) 를 프론트엔드에게 전송하기 위한 WebSocket 또는 Polling 이 필요할 수 있음.

# BDS소상공인플렛폼 - 캐릭터 시각적 방향성 가이드라인 (V1.0)

## 1. 개요
이 문서는 '비전(Vision)'이라는 이름의 대표 캐릭터가 플랫폼에서 어떻게 표현되고, 사용자와 어떤 느낌을 줄지 정의합니다. 디자인팀이 아트워크를 제작할 때 참고해야 할 기준입니다.

## 2. 기본 철학 (Core Philosophy)
- **신뢰와 친근함**: 캐릭터는 'AI 컨설턴트'이지만, 소상공인의 고민을 이해하는 따뜻한 파트너처럼 느껴져야 함.
- **기능성 시각화**: 단순한 장식이 아니라, 플랫폼의 '리스크 예측'이나 '성장 제안'이라는 기능을 시각적으로 표현.

## 3. 디자인 스타일 (Style)
- **이미지 스타일**: 깔끔하고 구조적인 **미니멀리즘 라인 아트**를 기본으로 하되, 캐릭터의 표정이나 동작에 따뜻한 **곡선**을 적용.
    - 예: 배경은 단순한 회색/흰색이지만, 캐릭터는 부드러운 색상과 움직임을 가짐.
- **컬러 팔레트**:
    - `Primary`: `#1A4363` (신뢰), `#FFC759` (지혜). 이 색상을 캐릭터의 옷이나 배경에 활용.
    - `Secondary/Accent`: `#4DB6AC`, `#FFB79D`. 따뜻하고 부드러운 톤 사용.
- **폰트**: 기하학적이고 깔끔한 Sans-serif 폰트 (예: Inter, Pretendard Bold).

## 4. 캐릭터 동작 및 표현 (Character Actions & Expressions)
캐릭터는 정지된 이미지가 아니라 '살아있는' 듯한 느낌을 주어야 합니다. 다음 동작들은 필수입니다.

- **Idle (대기)**: 편안하게 서 있거나, 태블릿을 보며 생각에 잠겨 있음.
    - 표현: 여유롭고 친근한 표정.
- **Talking (대화 중)**: 입술이 움직이며, 눈이 생동감 있게 빛남.
    - 표현: 진지하지만 따뜻한 표정.
- **Alert (경고/리스크)**: 눈을 크게 뜨거나, 손가락으로 가리키는 동작을 포함.
    - 표현: 불안함보다는 '주의'를 전달하는 차분한 경고.
- **Success (성공)**: 환하게 웃으며 박수치거나, 엄지손가락을 올림.
    - 표현: 기쁨과 성취감.

## 5. 활용 시나리오 예시 (Usage Scenarios)
이 아트워크가 어떻게 활용될지 구체적인 예를 제시합니다.

- **메인 페이지**: 캐릭터가 인사하며, "오늘 매출이 20% 올랐어요"라고 말함.
    - 컴포넌트: `<BDSCharacter state="talking" />`
- **리스크 경고 팝업**: "경쟁사들이 오늘 가격을 낮췄어요."라는 알림이 오면 캐릭터가 `alert` 상태로 전환됨.
    - 컴포넌트: `<BDSCharacter state="alert" onSpeak={() => ...} />`
- **성공 사례 페이지**: "고객의 매출이 10% 늘었어요!"라고 말하며, 캐릭터가 `success` 상태로 전환됨.
    - 컴포넌트: `<BDSCharacter state="success" />`
