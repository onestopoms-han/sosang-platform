## 📊 데이터-디자인 정합성 검증 프레임 (Data-Design Alignment Framework)

이 프레임은 랜딩 페이지의 각 핵심 전환 지점(Touchpoints)에 대해 **'예상 데이터 상태'**와 **'시각적 표현'** 간의 일치 여부를 측정합니다.

### 1. 검증 목표 (Goal of Validation)
사용자가 특정 행동(CTA 클릭, 정보 입력 등)을 취했을 때, 시스템이 제공하는 예상 결과(KPI 변화, 다음 단계 제시)가 디자인 요소(색상 변화, 레이아웃 변경, 메시지 강조)를 통해 **직관적이고 즉각적으로 전달**되어야 합니다.

### 2. 핵심 검증 영역 (Key Validation Areas)
모든 전환 지점은 다음의 세 가지 축을 기준으로 검증받습니다:

| 축 (Axis) | 설명 (Description) | 관련 디자인 요소 (Design Elements) | 기준 (Metric) |
| :--- | :--- | :--- | :--- |
| **A. 데이터 상태** | 실제 시스템에서 발생할 것으로 예측되는 사용자 행동의 결과 (KPI, 손실 비용 변화 등) | 없음 (Backend Logic) | 정량적 값 (예: $500 절감, 20% 성장) |
| **B. 디자인 의도** | 해당 데이터 상태에 따라 사용자가 취해야 할 다음 행동을 유도하기 위해 의도된 시각적 표현 | CTA 버튼 색상, 섹션의 강조 정도, 레이아웃 변화 | 정성적/구조적 일치 (예: '손실 최소화'를 위한 녹색 강조) |
| **C. 사용자 경험** | 데이터와 디자인이 결합되어 사용자가 혼란 없이 목표 행동을 수행하는 과정 | 폼 필드 유효성 메시지, 로딩 상태 표시, 흐름의 명확성 | 직관성 및 효율성 (예: 입력 오류 시 즉각적인 피드백) |

### 3. 전환 지점별 검증 상세 규칙 (Specific Rules per Touchpoint)

#### 3.1. 핵심 CTA 섹션 (ROI/Action Plan 제시 영역)
*   **목표:** '손실 비용 = 보험료' 프레임의 시각적 효과 극대화 및 행동 유도.
*   **검증 기준:**
    *   **데이터 상태 (A):** 특정 플랜(Basic/Standard) 선택에 따른 예상 절감액이 명확히 제시되어야 함.
    *   **디자인 의도 (B):** **'손실 최소화'** 목표 달성 시, 해당 영역의 배경색이나 텍스트 강조 색상이 **Growth Green** 계열로 명확하게 변화해야 함.
    *   **사용자 경험 (C):** CTA 버튼은 데이터 연동이 완료된 후 즉시 활성화되어야 하며, 클릭 시 다음 단계(예: 상세 컨설팅 신청)로의 흐름이 끊김 없이 연결되어야 함.

#### 3.2. 사용자 입력 폼 (Producer Story Form 등)
*   **목표:** 데이터 수집 과정에서 사용자의 신뢰를 유지하고 이탈을 방지.
*   **검증 기준:**
    *   **데이터 상태 (A):** 모든 필수 필드가 채워졌는지에 대한 실시간 검증(Validation) 결과가 명확해야 함.
    *   **디자인 의도 (B):** 입력 오류 발생 시, 에러 메시지(Error Message)는 **Red** 계열로 강조하고, 해당 필드만 하이라이트하여 사용자에게 즉각적인 수정 요청을 전달해야 함. 성공적으로 채워진 정보는 **Deep Blue** 배경으로 안정감을 제공해야 함.
    *   **사용자 경험 (C):** 데이터 연동 중(Loading)에는 명확한 애니메이션(예: 로딩 스피너, 진행 바)을 표시하여 시스템이 작동 중임을 알려야 함.

### 4. 실행 계획 (Next Step Execution Plan)
1.  **데이터 매핑 테이블 생성:** 개발팀으로부터 받은 최종 API 응답 데이터와 디자인 프레임의 각 섹션(CTA, Form)에 대응하는 변수 관계를 표 형태로 정의합니다.
2.  **시뮬레이션 시나리오 정의:** 최소 3가지 주요 시나리오 (예: 'Basic 플랜 선택', 'Standard 플랜 선택', '데이터 입력 성공')를 선정합니다.
3.  **디자인 적용 및 검증:** 정의된 시나리오별로, **Growth Green**과 **Deep Blue**의 대비를 활용하여 각 상태(A)가 어떻게 디자인(B)으로 변환되어 사용자에게 전달되는지 목업에 최종 반영하고, 이 프레임에 따라 자가 점검합니다.
