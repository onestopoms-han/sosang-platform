### 1. 공신력 있는 외부 데이터 소스 리스트 및 접근 방법

소상공인 매출 및 지역 경제 지표의 신뢰도를 극대화하기 위해 대한민국 정부 기관이 제공하는 공식 데이터를 최우선으로 확보합니다.

| 순위 | 데이터 소스 (API/DB) | 주요 측정 지표 (예시) | 데이터 특성 및 신뢰도 | 접근 방법 및 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **통계청 (KOSIS)** | 지역별 산업별 생산/소비액, 사업체 통계 | 매우 높음 (공식 통계) | KOSIS 국가통계포털 API 또는 데이터 다운로드 서비스 활용. 행정구역(시/도/군/구) 단위 필터링 필수. |
| **2** | **한국은행 경제통계시스템 (ECOS)** | 지역별 물가, 금리, GDP 성장률 등 거시경제 지표 | 매우 높음 (공식 통계) | ECOS API 또는 데이터 다운로드 서비스를 통해 시계열 데이터 확보. 산업별 세부 분류 확인 필요. |
| **3** | **중소벤처기업부 / 소상공인시장진흥공단 DB** | 소상공인 관련 지원 사업 참여율, 매출액 통계 (잠재적) | 높음 (정책 연관성) | 해당 기관의 데이터 공개 API 또는 공고를 통한 접근. 플랫폼 특화 데이터 확보에 유리. |
| **4** | **금융감독원 / 한국거래소** | 특정 산업군 관련 시장 동향, 업종별 거래량 | 중간~높음 (시장 지표) | 금융/증권 관련 API 또는 공개 자료 검색을 통해 간접 지표 확보. |

### 2. 데이터 통합 및 표준화 로직 설계

확보된 외부 데이터를 플랫폼의 요구사항(매출 성장률)에 맞추어 표준화하는 로직은 다음과 같습니다.

1.  **산업군 매핑 (Industry Mapping):**
    *   외부 데이터의 분류 체계(예: 한국표준산업분류)와 BDS 플랫폼 내의 내부 산업군 분류를 **일대일 매핑 테이블**을 구축하여 불일치를 해소합니다.
2.  **지표 통일 (Metric Unification):**
    *   '매출 성장률'을 계산할 때, 사용된 기준 시점(예: 전년 동기 대비, 전분기 대비)과 측정 단위(원화/달러)를 표준화합니다.
3.  **지역 표준화 (Geographic Standardization):**
    *   데이터에서 추출된 지역 코드(시/도/군/구)가 플랫폼 내의 지리적 정의와 일치하는지 확인하고, 필요 시 행정구역 레벨을 조정하여 데이터 포인트를 정규화합니다.

### 3. 확보된 데이터셋과의 비교 및 보고 프레임워크

현재까지 확보된 데이터셋(내부 DB)과 외부 데이터를 비교하여 데이터 신뢰도를 측정하는 검증 단계가 필수적입니다.

**보고 항목:**
1.  **누락 항목 분석 (Missing Data Analysis):** 내부 시스템에서 요구하는 모든 지역/산업 조합에 대해 외부 데이터의 존재 여부를 점검하고, **데이터 공백 지점(Gap Points)**을 식별합니다.
2.  **일관성 검증 (Consistency Check):** 동일한 기간 및 지역에 대해 두 소스 간의 성장률 값이 허용 오차 범위 내에 있는지 비교하여 **비일관성 보고서**를 생성합니다.
3.  **신뢰도 점수 산출 (Reliability Scoring):** 각 데이터 포인트에 대해 데이터 출처(Source)와 업데이트 시점(Timestamp)을 기반으로 **PLRI(잠재적 리스크 감소 지수)**의 기초가 될 신뢰도 점수를 부여합니다.

**결론:**
실제 데이터셋이 제공되지 않았으므로, 위 프레임워크를 바탕으로 실제 데이터 연동 작업 시 적용해야 할 **데이터 파이프라인 설계 명세**를 제시했습니다. 다음 단계에서는 이 계획을 기반으로 구체적인 API 연동 스크립트 및 내부 비교 로직 구현을 진행하겠습니다.
