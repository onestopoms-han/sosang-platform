# ✍️ Writer: 최종 카피라이팅 가이드라인 및 영상 스토리라인

## 📌 개요
**목적:** 비주얼 에셋(썸네일, UI 스냅샷) 과 메시지(신뢰/위기감 극대화) 를 시각적·언어적으로 일관되게 연결  
**대상 채널:** YouTube, Instagram, 블로그  
**핵심 키워드:** 시스템 안정성 (SLA), 데이터 기반 수익화, 소상공인 미래 보장

## 🎨 1. 비주얼 가이드라인 (Designer 와의 연계)
| 요소 | 규칙 | 색상 코드 | 적용 위치 |
|------|------|-----------|----------|
| **메인 배경** | Deep Blue (#004D66) → 신뢰, 안정감 | `#004D66` | 썸네일 좌측 2/3, UI 스냅샷 헤더 |
| **강조 포인트** | Growth Green (#3CB371) → 성장, 해결, CTA | `#3CB371` | 데이터 시각화, 보상 박스, 버튼 |
| **텍스트 색상** | 흰색 (Deep Blue 배경 위), 어두운 회색 (Growth Green 배경 위) | `#FFFFFF`, `#2C2C2C` | 모든 타이포그래피 |
| **폰트 스타일** | 굵은 무세리프 (메인), 세련된 산세리프 (서브) | - | 썸네일 헤드라인, UI 라벨 |

**가시성 규칙:**  
- Deep Blue 배경: 흰색 텍스트 대비 최소 16px 글자 크기  
- Growth Green 배경: 어두운 회색/흰색 텍스트 대비 최적화  
- 데이터 시각화 요소는 Green 박스 안에 제한하여 시선 집중 유도

## ✍️ 2. 카피라이팅 가이드라인 (채널별)
### YouTube 썸네일 (UI-SNAP-01 기준)
| 위치 | 메시지 | 톤 | 길이 |
|------|--------|-----|-----|
| 상단 중앙 | "AI 신뢰도, 이제 데이터로 증명하세요." | 진지하고 확신에 찬 | 9~11 단어 |
| 중앙 하단 | "시스템 안정성으로 소상공인의 미래를 확보" | 안심과 희망을 주는 | 8~10 단어 |
| CTA 버튼 | "베타 참여 신청" | 명확하고 시급한 | 3~4 단어 |

**주의:**  
- "증명하세요", "확보" 같은 동사는 행동 유도 (action-oriented)  
- "데이터로", "시스템 안정성"은 신뢰를 구체화하는 키워드

### Instagram 캡션 / 릴스 스크립트
| 섹션 | 메시지 | 톤 | 해시태그 전략 |
|------|--------|-----|---------------|
| 오프닝 (1~2 문장) | "소상공인의 가장 큰 고민, 시스템 불안정" | 공감과 문제 제기 | #소상공인 #시스템안정성 |
| 중간 (핵심 가치 3 줄) | "1. 데이터 기반 SLA 보상<br>2. 예측 가능한 수익 모델<br>3. 실시간 안정성 모니터링" | 명확하고 신뢰할 수 있음 | #BDS플랫폼 #AI신뢰도 |
| CTA (마무리) | "베타 참여를 원하는 소상공인분은 DM 주세요!" | 친근하고 긴급한 느낌 | #베타참여 #소상공인지원 |

**주의:**  
- 첫 3 줄에 핵심 가치(안정성, 수익화) 포함 → 스크롤 중단 유도  
- 해시태그는 "대중적" + "니치" 조합 (예: #소상공인 + #SLA보상)

### 블로그 / 메일
| 섹션 | 메시지 | 톤 | 링크 전략 |
|------|--------|-----|-----------|
| 제목 | "시스템 안정성과 수익 모델을 연결하는 플랫폼, BDS" | 전문적이고 신뢰할 수 있음 | 링크는 첫 문장 안에 배치 |
| 본론 (3 단락) | 1. 현재 소상공인의 불안정성<br>2. BDS 의 SLA 보상 구조<br>3. 참여를 위한 가이드라인 | 교육적이면서도 설득력 있음 | CTA 버튼은 마지막에 두 번 반복 |

## 🎬 3. 영상 스토리라인 (YouTube, Instagram Reels)
### 공통 구조: **문제 제기 → 해결책 제시 → 증거 제시 → 행동 유도**
| 단계 | 시간 (Reels 기준 30 초) | 메시지 | 비주얼 연계 |
|------|------------------------|--------|-------------|
| **오프닝 (0~5 초)** | "소상공인, 시스템 불안정 때문에 매출 손실?" | 공감과 문제 제기 | Deep Blue 배경 + 흰색 텍스트 |
| **문제 심화 (5~10 초)** | "예상치 못한 다운타임, 막대한 피해" | 위기감 조성 | UI 스냅샷에서 Red 경고 아이콘 |
| **해결책 제시 (10~20 초)** | "BDS 플랫폼: SLA 보상 구조로 안정성 보장" | 해결책 명확화 | Growth Green 박스 + 데이터 시각화 |
| **증거 제시 (20~25 초)** | "시스템 안정성 지표: Uptime 99.9%, MTTR 10 분 내" | 신뢰도 확보 | UI-SNAP-02 에서 Tier 구조 보여줌 |
| **CTA 유도 (25~30 초)** | "베타 참여 신청, 지금 바로 시작하세요!" | 행동 유도 | Growth Green 버튼 애니메이션 |

### YouTube 영상 (3~5 분) 스토리라인
**1. 오프닝 (0:00-0:40):**  
- Deep Blue 배경에서 흰색 텍스트 "소상공인, 시스템 불안정 때문에 매출 손실?"  
- UI-SNAP-01 썸네일 이미지 등장 + "AI 신뢰도, 이제 데이터로 증명하세요." 메시지  
**2. 문제 제기 (0:40-1:30):**  
- 소상공인의 실제 사례(다운타임으로 인한 매출 감소)  
- 시스템 불안정성이 소상공인에게 미치는 영향  
**3. 해결책 제시 (1:30-2:30):**  
- BDS 플랫폼의 SLA 보상 구조 설명  
- Growth Green 박스 안에 데이터 기반 수익화 모델  
- UI-SNAP-02 에서 Tier 구조 시각적 설명  
**4. 증거 제시 (2:30-3:20):**  
- 시스템 안정성 지표(uptime, MTTR) 실시간 데이터  
- 기존 플랫폼과 BDS 의 차이점 비교  
**5. CTA 유도 (3:20-4:00):**  
- "베타 참여 신청" 애니메이션 + Growth Green 버튼  
- 링크는 설명란에 즉시 배치

### Instagram Reels (15~30 초) 스토리라인
**오프닝:** Deep Blue 배경, 흰색 텍스트 "소상공인, 시스템 불안정 때문에 매출 손실?"  
**문제 심화:** UI-SNAP-02 에서 Red 경고 아이콘 등장 + "예상치 못한 다운타임, 막대한 피해"  
**해결책 제시:** Growth Green 박스 + 데이터 시각화 "BDS 플랫폼: SLA 보상 구조로 안정성 보장"  
**증거 제시:** UI-SNAP-02 에서 Tier 구조 보여줌 + "시스템 안정성 지표: Uptime 99.9%"  
**CTA 유도:** Growth Green 버튼 애니메이션 + "베타 참여 신청, 지금 바로 시작하세요!"

## 📊 4. 시각적 일관성을 위한 체크리스트
| 항목 | 규칙 | 위반 시 조치 |
|------|------|--------------|
| **색상 사용** | Deep Blue 배경은 Growth Green 요소만 허용 | 다른 색상 추가 금지 |
| **텍스트 대비** | 흰색/어두운 회색 대비 최소 4.5:1 (WCAG AAA) | 밝기 조정 필요 |
| **데이터 시각화** | 모든 그래프는 Growth Green 박스 안에 제한 | 직접 표시하지 않음 |
| **CTA 버튼** | 항상 Growth Green + 흰색 텍스트 "베타 참여 신청" | 다른 문구 금지 |
| **폰트 크기** | 최소 16px (모바일), 최대 24px (데스크톱) |过小/过大 조정 필요 |
