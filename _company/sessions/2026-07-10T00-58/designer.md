이 사양은 데이터 변화에 따른 시각적 피드백과 애니메이션 로직을 정의하여, 사용자가 단순히 데이터를 보는 것이 아니라 '실행 책임'을 느끼게 하는 데 중점을 둡니다.

# ⚙️ 인터랙티브 와이어프레임 상세 디자인 스펙 (Interaction Spec)

## 1. 핵심 컴포넌트 정의: 통제권 회복 게이지 (Control Gauge Component)

**목표:** 사용자의 진행 과정(S1 $\to$ S2 $\to$ S3)을 '불안'에서 '통제권 회복'으로의 감정적 여정을 시각적으로 표현한다.

### 1.1. 컴포넌트 상태 (Component States)

| State ID | 단계 명칭 (State Machine) | 사용자 행동 (Input Trigger) | 게이지 시각화 목표 (Emotional Goal) | 색상 테마 적용 |
| :---: | :--- | :--- | :--- | :--- |
| **S1** | 진단 (Diagnosis) | 초기 데이터 입력 완료 | 불안정성/미지 상태 표현 | Deep Blue (불안) |
| **S2** | 문제 인식 (Problem Recognition) | 실행 계획(Action Plan) 수립 및 첫 행동 시작 | 전환/노력의 시작 (Initiation) | Blue (집중) |
| **S3** | 행동 계획 (Action Execution) | 목표에 따른 연속적인 행동 완료 | 성취감/통제권 회복 (Achievement/Control) | Growth Green (성장/안정) |

### 1.2. 시각적 표현 및 애니메이션 로직 (Visual & Animation Logic)

**A. 게이지 채움 방식: 파편 결합 (Fragment Assembly)**

*   **기존 방식 (숫자 증감):** 단순한 선형 증가 (Linear Fill).
*   **새로운 방식 (파편 결합):** 전체 진행도를 구성하는 여러 개의 작은 '파편(Fragment)' 요소가 사용자 행동에 따라 순차적으로 결합되는 방식으로 표현한다.
    1.  **S1 $\to$ S2 전환 시:** 초기 파편 3~5개가 불안정하게 분산되어 있다가, 사용자가 첫 번째 실행 계획을 수립할 때 **'첫 번째 핵심 단계(Fragment 1)'**가 Deep Blue에서 Growth Green으로 부드럽게 전환되며 결합된다.
    2.  **S2 $\to$ S3 전환 시:** 이후의 파편들이 순차적으로 결합되며, 각 결합마다 미세한 '성취 보상 애니메이션(Reward Animation)'이 발생한다. 이는 단순히 채워지는 것이 아니라, **'행동에 따른 결과'**가 물리적으로 완성되는 느낌을 주어야 한다.
    3.  **S3 (완료):** 모든 파편들이 완벽하게 결합되어 하나의 안정적인 형태로 고정되며, 게이지는 최대치(100%)를 시각적으로 강조한다.

**B. 트랜지션 로직 (Transition Logic)**

*   **트랜지션 트리거:** 사용자가 'Action Plan' 항목을 클릭하거나, 특정 마이크로카피 단계(예: "첫 번째 행동 실행")를 완료할 때마다 게이지 업데이트가 발생한다.
*   **애니메이션 속도:** S1에서 S2로 넘어갈 때는 다소 느리고 불안정하게 (약간의 흔들림 포함), S2에서 S3으로 넘어갈 때는 목표 지향적으로 빠르고 명확하게 진행되도록 설정하여 **'통제권 회복의 가속화'**를 표현한다.

## 2. 실행 로드맵 연동 스펙 (Execution Roadmap Integration Spec)

### 2.1. 데이터 흐름 매핑

| State Machine | 핵심 KPI 측정 지표 | 시각적 피드백 요소 | API 연동 근거 |
| :---: | :--- | :--- | :--- |
| **S1 (진단)** | 위험 점수(Risk Score) | 게이지 초기 상태, 불안정성 표현 | 백엔드 진단 API 결과 |
| **S2 (문제 인식)** | 계획 완성도 (Plan Completion %) | 행동 항목별 체크리스트 채움 | 사용자 입력 데이터 |
| **S3 (행동 계획)** | 실행 책임 점수 (Execution Accountability) | 파편 결합 애니메이션 진행, ROI 예상치 시각화 | 플랫폼 성과 측정 로직 |

### 2.2. UI/UX 흐름 정의 (Flow Definition)

1.  **Step 1: 진단 화면:** 사용자는 현재의 불안정성(S1)을 확인한다. 게이지는 Deep Blue로 채워지며 '현재 상태 인식'을 유도한다.
2.  **Step 2: 문제 인식/계획 전환 (Trigger Point):** 사용자가 '실행 계획 수립' 버튼을 클릭하면, 시스템은 다음 단계(S2)로 넘어갈 준비를 한다. 이때 게이지는 불안정하게 흔들리며 **'도전의 시작'**을 시각화한다.
3.  **Step 3: 행동 실행 및 성취 (Core Loop):** 사용자가 마이크로카피에 따라 실제 행동(Action)을 기록할 때마다, '파편 결합 애니메이션'이 활성화되어 **'실행 책임'**을 시각적으로 보상한다. 각 파편의 결합은 재무적 ROI 예상치와 동기화된다.
4.  **Step 4: 통제권 회복 (Completion):** 모든 행동이 완료되면, 게이지는 완전한 Growth Green으로 고정되며, 사용자는 '통제권 회복'이라는 감정적 성취감을 경험한다.

## 3. 디자인 시스템 적용 가이드 (Design System Application Guide)

### 3.1. 컬러 스케일 확정

*   **Primary Color (불안/초기):** Deep Blue (`#004D66`) - 불안, 미지
*   **Secondary Color (집중/행동):** Blue (`#4D7C98`) - 집중, 계획 수립
*   **Success Color (성장/보상):** Growth Green (`#3CB371`) - 성취, 통제권 회복

### 3.2. 타이포그래피 및 레이아웃 스펙

*   **헤드라인:** 간결하고 행동 지향적인 서체(예: Pretendard Bold)를 사용하여 명확성을 확보한다.
*   **게이지 영역:** 게이지 자체는 중앙에 배치하되, 파편 결합 애니메이션이 시각적으로 가장 큰 영역을 차지하도록 디자인한다. 주변에는 '실행 책임' 점수와 마이크로카피 텍스트가 명확하게 표시되어야 한다.
