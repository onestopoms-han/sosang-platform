### 1. Tier 1 기능 구현을 위한 초기 코딩 계획 (Implementation Plan)

Tier 1 기능의 핵심은 **'데이터 무결성 확보'**와 **'Actionable Insight 유도'**입니다. 이를 위해 다음 세 가지 모듈에 집중하여 개발을 시작해야 합니다.

#### A. 데이터 모델링 및 API 설계 (Data Layer Focus)
*   **목표:** 소상공인의 Pain Point 데이터를 가장 효율적으로 수집하고, 이 데이터를 기반으로 ROI를 시뮬레이션할 수 있는 최소한의 엔티티를 정의합니다.
*   **핵심 작업:** `ProducerStoryForm`에서 유효성 검증이 통과된 데이터가 DB에 저장될 때 필요한 스키마와 API Endpoints를 확정해야 합니다. 특히, **'Actionable Insight'로 연결되는 핵심 지표(KPIs)**를 어떻게 정의하고 저장할지 설계합니다.
*   **주요 고려사항:** JSON Schema에서 정의한 입력값들이 실제 데이터베이스 필드로 매핑될 때 발생하는 타입 불일치나 누락에 대한 예외 처리 로직을 백엔드에 선반영해야 합니다.

#### B. 프론트엔드 통합 및 상태 관리 (UI/UX Flow Focus)
*   **목표:** Designer가 제시한 6-섹션 구조를 기반으로, 사용자가 '진단 → 계획 → 실행'의 흐름을 끊김 없이 경험하도록 UI 컴포넌트를 구현합니다.
*   **핵심 작업:** `dashboard-kit`의 레이아웃(Two-Column Layout)과 핵심 CTA 버튼(`Growth Green` 색상 적용)이 데이터 변화에 따라 실시간으로 업데이트되는 상태 관리 로직을 최우선으로 구축해야 합니다.
*   **주요 고려사항:** 데이터가 로드되는 시점, API 호출 실패 시 사용자에게 명확한 피드백(Error Handling)을 제공하는 흐름을 정의해야 합니다.

#### C. 시스템 연동 및 자동화 (E2E Focus)
*   **목표:** 프론트엔드의 입력이 백엔드 로직을 거쳐 데이터가 성공적으로 저장되고, 최종 결과가 사용자에게 반영되는 전체 경로를 검증합니다.
*   **핵심 작업:** API Gateway에서부터 DB까지의 모든 트랜잭션 흐름을 시뮬레이션하는 통합 테스트 시나리오를 설계하고, 이를 자동화 스크립트로 구현할 준비를 합니다.

### 2. 초기 통합 테스트 시나리오 (E2E Test Scenario Draft)

이 시나리오는 **"소상공인 데이터 입력 → 시스템 처리 → ROI 지표 계산 및 표시"**의 핵심 흐름에 집중합니다.

#### 테스트 목표:
Tier 1 기능 구현 후, 사용자가 입력한 데이터가 시스템 내에서 정확하게 처리되어 원하는 'Actionable Insight'로 변환되는지 확인한다.

#### 시나리오 1: 성공적인 데이터 입력 및 ROI 산출 (Happy Path)
*   **입력 데이터:** `ProducerStoryForm`에 Designer가 정의한 모든 필수 필드(Pain Point, 현재 상황, 희망 목표 등)를 **정확히 일치**하는 값으로 입력한다. (JSON Schema 준수 확인)
*   **시스템 동작 확인:**
    1.  폼 제출 시, 백엔드가 데이터 유효성 검사(Schema Validation)를 통과했는지 확인한다.
    2.  데이터가 DB에 정확하게 저장되었는지 확인한다.
    3.  저장된 데이터를 기반으로 **ROI 시뮬레이션 로직**이 실행되어 예상 수익 변화 결과가 계산되는지 확인한다.
    4.  프론트엔드 대시보드에서 새로운 ROI 지표와 Actionable Insight가 실시간으로 업데이트되는지 확인한다.
*   **검증 포인트:** 입력값과 최종 출력된 ROI 수치가 수학적으로 일치하는지 검증한다.

#### 시나리오 2: 데이터 유효성 검사 실패 (Negative Testing)
*   **입력 데이터:** 필수 필드 중 하나 이상을 누락하거나, JSON Schema에서 정의한 타입 규칙을 위반하는 값(예: 숫자가 아닌 문자열 입력, 비논리적 범위 설정 등)을 입력한다.
*   **시스템 동작 확인:**
    1.  폼 제출 시, **클라이언트 측 유효성 검사**가 즉시 실패하고 사용자에게 명확한 오류 메시지(`Actionable Insight`에 맞춰 친절하게 제시)를 반환하는지 확인한다.
    2.  서버 레벨에서도 해당 데이터가 DB에 저장되지 않고 트랜잭션이 롤백되는지 확인한다.
*   **검증 포인트:** 시스템이 잘못된 데이터를 받아들이지 않고, **데이터 무결성 규칙**을 철저히 준수하는지 검증한다.

#### 시나리오 3: 경계값 테스트 (Boundary Testing)
*   **입력 데이터:** ROI 계산에 사용되는 변수(예: 초기 투자 비용, 예상 매출 증대율 등)의 **최소값, 최대값, 그리고 논리적 경계값**을 입력하여 시스템의 반응을 확인한다.
*   **시스템 동작 확인:**
    1.  경계값을 초과하거나 벗어나는 값이 입력되었을 때, 시스템이 오류를 반환하거나 합리적인 경고(Warning)를 제시하는지 확인한다.
    2.  음수 값이나 비현실적인 값을 입력했을 때 시스템이 안전하게 처리하고 예외 처리를 수행하는지 확인한다.
*   **검증 포인트:** 시스템의 **예외 처리 로직**이 예상치 못한 입력에 대해 안정적으로 작동하는지 검증한다.
