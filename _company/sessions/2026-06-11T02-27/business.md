# 💼 BDS 플랫폼 핵심 구조 (State-based Architecture) 및 비즈니스 연결성 정의

**작성일**: 2026.06.11  
**작성자**: 현빈 (Head of Business)  
**버전**: v1.0

## 🎯 Executive Summary
본 설계안은 리스크 등급 (Risk Level: A/B/C) 을 플랫폼의 핵심 상태 (State) 로 삼아, 각 상태에 따라 최적화된 수익 모델과 기능 우선순위를 정의합니다. 이를 통해 **Trust Widget** 과 **PainGauge**가 단순히 UI 컴포넌트가 아닌, 사용자의 심리적 리스크를 관리하여 결제를 유도하는 '비즈니스 엔진'으로 작동하도록 설계됩니다.

---

## 1️⃣ State-based Architecture: 리스크 등급별 구조 및 수익 모델
플랫폼의 상태는 **사용자가 인지할 수 있는 리스크 (Risk Perception)**에 따라 정의됩니다. 아래 표는 각 상태별 기능 우선순위와 수익 모델을 명시합니다.

| 리스크 상태 | 정의 | 주요 사용자 감성 | **기능 우선순위** | **수익 모델 (Revenue Model)** |
| :--- | :--- | :--- | :--- | :--- |
| **🟢 A: 안전마진 상태**<br>(Trust Maximized) | 공급망 검증 완료, 실시간 거래 데이터 공유 가능, 정산 100% 보장. | "완전 신뢰", "편의성" | - PainGauge: 0 (최소 노출)<br>- Trust Widget: "인증됨" 표시 유지<br>- 결제 버튼: 즉시 활성화 | **정기 구독료 + 프리미엄 수수료**<br>• 월 5~10 만 원 (플랫폼 유지비)<br>• 거래 건당 2% (수수료 waived for 3 months) |
| **🟡 B: 검증 중 상태**<br>(Risk Mitigation) | 기본 정보 제공, 일부 데이터 검증 중, 신용 점수 기반 평가. | "신중", "호기심" | - PainGauge: '검증 진행' 단계 표시<br>- Trust Widget: 출처 링크 노출<br>- 결제 버튼: 조건부 활성화 (예: 30 일 무료 체험) | ** Freemium 전환 + 데이터 판매**<br>• 무료 체험 기간 후 월 2~5 만 원 (기본 플랜)<br>• 공급망 검증 데이터 리포트 유료 구독<br>• PainGauge 기반 리스크 분석 툴 사용료 |
| **🔴 C: 고위험 상태**<br>(Risk Avoidance) | 정보 부족, 부정 거래 의심, 정산 보장 불가. | "불안", "피해" | - PainGauge: '주의' 경고 표시<br>- Trust Widget: 출처 링크 필수 클릭<br>- 결제 버튼: 지연 또는 차단 | **분리된 수수료 + 보험 파트너십**<br>• 거래 건당 5~10% (고위험 상환 수수료)<br>• 보험 가입 유도 수수료 (보험사 제휴)<br>• PainGauge '위험 분석' 리포트 유료 판매 |

---

## 2️⃣ Trust Widget & PainGauge 의 비즈니스 목표 연결성
각 위젯은 단순한 UI 요소를 넘어, **사용자의 심리적 리스크를 관리하여 결정을 유도**하는 역할을 수행합니다. 아래는 두 위젯이 각 상태 (A/B/C) 에서 어떻게 작동하고, 그것이 곧 수익으로 이어지는지 설명합니다.

### 🔵 Trust Widget: 신뢰 기반의 결제 전환
- **기능**: 플랫폼이 제공하는 인증 정보를 시각적으로 표시하며, 출처 링크를 통해 정밀성을 증명합니다.
- **비즈니스 연결**:
  - **상태 A**: 사용자의 "완전 신뢰"감을 고취시켜, 즉시 결제 행동을 유도합니다. 이는 높은 거래 건수 (High Volume) 와 낮은 수수료 수익을 가져옵니다.
  - **상태 B**: 출처 링크 클릭 시 검증 데이터를 제공하여, 사용자가 '안전하게' 플랫폼을 이용하도록 설득합니다. 이는 Freemium 플랜의 전환율을 높입니다.
  - **상태 C**: "주의" 경고와 함께 출처 링크를 강제 노출시켜, 사용자가 위험을 인지하고도 플랫폼을 신뢰하게 만듭니다. 이는 고위험 수수료 모델 (High Margin) 을 실현합니다.

### 🟢 PainGauge: 리스크 기반의 행동 유도
- **기능**: 사용자의 리스크 수준 (재정적 불안, 공급망 불안 등) 을 실시간으로 시각화하여, 적절한 솔루션을 제안합니다.
- **비즈니스 연결**:
  - **상태 A**: PainGauge 은 "0" 상태로 유지되어, 사용자에게 불필요한 심리적 부하를 줄입니다. 이는 장기적인 플랫폼 충성도를 높여 구독료 수익을 안정화시킵니다.
  - **상태 B**: '검증 진행' 단계가 표시되어, 사용자가 검증 데이터를 유료 리포트로 구매하도록 유도합니다. 이는 데이터 기반의 추가 수익원 (Data Revenue) 을 창출합니다.
  - **상태 C**: '주의' 경고가 표시되어, 사용자가 위험을 인지하고도 플랫폼이 제공하는 보험이나 분산 투자 솔루션을 선택하도록 안내합니다. 이는 고위험 수수료와 파트너십 수익을 가져옵니다.

## 📊 요약 및 결론
본 구조는 리스크 등급 (A/B/C) 을 상태 기반으로 설계하여, 각 상태에 최적화된 수익 모델과 기능을 제공합니다. Trust Widget 과 PainGauge 은 사용자의 심리적 리스크를 관리하여 결정을 유도하는 '비즈니스 엔진'으로 작동하며, 플랫폼의 핵심 가치인 '안전마진'을 실현합니다.

**CEO 에게 전달할 메시지**:
> "리스크 등급 (A/B/C) 을 기반으로 한 State-based Architecture 는, 사용자가 자신의 리스크 수준에 맞는 서비스를 선택하게 하여, 자연스럽게 수익 모델이 연결되도록 설계되었습니다. 이는 단순한 기능 개발이 아닌, 비즈니스 모델을 플랫폼 구조 자체에 내재화하는 것입니다."
