# 💼 BDS 소상공인플랫폼 - MVP 베타 테스트 전략서 (옵션 B 기반)
**작성일:** 2026-06-28  
**작성자:** 현빈 (Head of Business)

## 1. Executive Summary
MVP 베타 테스트는 복잡한 기능보다 **'원가 투명성'**과 **'마진 고정'**이라는 핵심 가치에 집중하여, 초기 자본이 적은 소규모 공급자와 소매점을 연결하는 데 성공해야 합니다. 이를 위해 **Option B (원가 + 마진 고정 가격)** 모델을 적용하고, 3 단계 번들링을 통해 AOV 를 높이는 전략을 수립합니다.

## 2. 타겟 고객 시나리오 (User Stories)
### 2-1. 공급자 (새로운 식품 가공 업체)
> "저는 처음에 원가를 알기 어려운 플랫폼을 쓰고 싶지 않습니다. 저의 비용을 정확히 반영하고, 제가 원하는 마진만 남겨주는 시스템이 필요합니다."

- **프로필:** 신규 설립한 식품 가공소, 2~5 명의従業員, 자본은 1 억원 미만
- **Pain Point:** 중간 유통商的을 제거하고 직접 소매점과 연결하려는 의지, 원가 계산의 투명성 필요
- **Goal:** 초기 재고 회전율 증가, 안정적인 소규모 고객 확보

### 2-2. 구매자 (소규모 소매점)
> "저는 고급 제품을 취급하기에는 비용이 부담스럽지만, 가성비 좋은 제품들을 취급하고 싶어요. 정육점이나 식료품점에서 '가성비'와 '신뢰'를 원하는 고객이 많습니다."

- **프로필:** 1~2 명의従業員, 매출 5 억원 미만, 지역 기반 소규모 점포
- **Pain Point:** 기존 유통渠의 높은 마진 부담, 제품 정보 부족으로 인한 고객 불만
- **Goal:** 원가 투명한 제품을 취급하여 고객 신뢰도 향상, 추가 수익 창출

## 3. 가격 및 번들링 전략 (Option B 기반)
### 3-1. 기본 모델: '원가 투명형' (Entry Tier)
**대상:** 가장 낮은 진입 장벽을 위한 제품  
**특징:**
- **원가 공개:** 공급자의 원가 정보만 노출, 마진 추가 없음
- **마진 고정:** 플랫폼이 정한 고정된 마진율 (예: 10%) 을 적용
- **기능:** 기본 제품 등록, 단순 주문 관리
**가격 구조:**  
`판매 가격 = 공급자 원가 x 1.10`  
*(예: 원가 10,000 원 → 판매 11,000 원)*

### 3-2. 고급 모델: '마진 고정형' (Standard Tier)
**대상:** 마진을 안정적으로 확보하려는 공급자  
**특징:**
- **원가 공개:** 공급자의 원가 정보 노출 + 플랫폼 권장 마진율 적용
- **마진 고정:** 플랫폼이 정한 마진율을 고정 (예: 15%)
- **기능:** 제품 등록, 주문 관리, 간단한 마케팅 도구 제공
**가격 구조:**  
`판매 가격 = 공급자 원가 x 1.15`  
*(예: 원가 10,000 원 → 판매 11,500 원)*

### 3-3. 프리미엄 모델: '유연 마진형' (Premium Tier)
**대상:** 고급 제품이나 브랜드 가치를 인정하는 공급자  
**특징:**
- **원가 공개:** 공급자의 원가 정보 노출 + 플랫폼 권장 마진율 적용 + 고객 리뷰 반영
- **마진 고정:** 플랫폼이 정한 마진율을 고정 (예: 20%)
- **기능:** 고급 제품 등록, 주문 관리, 마케팅 도구 제공, 고객 리뷰 분석
**가격 구조:**  
`판매 가격 = 공급자 원가 x 1.20`  
*(예: 원가 10,000 원 → 판매 12,000 원)*

## 4. MVP 베타 테스트 로드맵
### 4-1. 단계별 목표
| 단계 | 기간 | 목표 | KPI |
|------|------|------|------|
| **Pilot** | 1~2 주 | 초기 공급자 및 소매점 10 개씩 등록, 거래 실행 | AOV: 5 만 원 / 트랜잭션 수: 10 회/일 |
| **Expansion** | 3~4 주 | 추가 기능 테스트 (마케팅 도구, 고객 리뷰) | AOV 증가: 20%, 재방문율: 30% |
| **Scale-up** | 5~6 주 | 초기 수익 모델 검증 및 확장 전략 수립 | 매출: 1 천만 원 / 월, 이탈율: <10% |

### 4-2. 성공 기준 (Go/No-Go Criteria)
- **Pilot:** AOV가 목표 대비 80% 이상 달성 또는 트랜잭션 수 5 회/일 달성 → Expansion 진행  
- **Expansion:** 추가 기능 테스트 시 매출 성장률 100% 이상 또는 재방문율 20% 이상 달성 → Scale-up 진행  
- **No-Go:** AOV 목표 대비 60% 이하 또는 이탈율이 30% 초과 → 전략 수정 후 재도전

## 5. 디자인 및 개발팀 전달 스펙
### 5-1. 기본 모델 (Entry Tier)
- **UI/UX:** 원가 정보와 마진율을 명확히 표시하는 UI 구성  
- **기능:** 제품 등록, 주문 관리, 단순 마케팅 도구 제공  
- **API 스키마:** `GET /product`, `POST /order`, `GET /reviews` 등 기본 API 구현

### 5-2. 고급 모델 (Standard Tier)
- **UI/UX:** 마진율을 고정하며, 고객 리뷰를 분석하는 UI 구성  
- **기능:** 고급 제품 등록, 주문 관리, 마케팅 도구 제공, 고객 리뷰 분석  
- **API 스키마:** `GET /product`, `POST /order`, `GET /reviews` 등 기본 API 구현 + 마케팅 도구 API 추가

### 5-3. 프리미엄 모델 (Premium Tier)
- **UI/UX:** 고급 제품 등록, 주문 관리, 마케팅 도구 제공, 고객 리뷰 분석  
- **기능:** 고급 제품 등록, 주문 관리, 마케팅 도구 제공, 고객 리뷰 분석  
- **API 스키마:** `GET /product`, `POST /order`, `GET /reviews` 등 기본 API 구현 + 마케팅 도구 API 추가
