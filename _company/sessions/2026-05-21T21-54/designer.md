# 🚀 AI 진단 플로우: 프론트엔드 구현 가이드 (Implementation Guide)

## 1. 프로젝트 개요 및 핵심 컨셉
**플랫폼:** BDS 소상공인 플랫폼 - AI 진단 및 액션 플랜 모듈
**핵심 UX 목표:** '혼돈(Chaos)'에서 '질서(Order)'로의 전환을 시각적으로 제공하여, 사용자가 복잡한 데이터를 명확하고 실행 가능한 행동 계획으로 즉시 전환하도록 유도한다. (Action Plan Flow의 핵심 반영)
**디자인 컨셉:** **The Blueprint (청사진)**
*   **톤앤매너:** 신뢰감, 전문성, 명료함, 실행 지향적. (전문 컨설턴트가 제시하는 명확한 로드맵처럼)

## 2. 디자인 시스템 사양 (Design System Specifications)

### 🎨 컬러 팔레트 (Color Palette)
| 역할 | 색상 이름 | HEX 코드 | 용도 |
| :--- | :--- | :--- | :--- |
| **Primary** (액션/강조) | BDS Blue | `#007BFF` | CTA 버튼, 핵심 지표 강조, 활성화 상태 |
| **Secondary** (보조 정보) | Soft Gray | `#E9ECEF` | 배경, 카드 영역, 입력 필드 테두리 |
| **Success** (성공/긍정) | Green Accent | `#28A745` | 성공적인 진단 결과, 긍정적 액션 플랜 항목 |
| **Warning** (주의/경고) | Amber Alert | `#FFC107` | 주의 사항, 실행 전 확인 단계 |
| **Danger** (위험/경고) | Red Alert | `#DC3545` | 리스크 경고, 필수 입력 누락 시 표시 |
| **Text** (본문) | Deep Navy | `#212529` | 본문 텍스트, 헤드라인 |

### ✒️ 타이포그래피 (Typography)
*   **Font Family:** Pretendard (또는 시스템 기본 Sans-serif)
*   **H1 (진단 제목):** 28px, Bold, Deep Navy
*   **H2 (섹션 제목):** 20px, SemiBold, Deep Navy
*   **Body Text:** 16px, Regular, Deep Navy
*   **Action Item:** 15px, Medium, BDS Blue (혹은 강조를 위해 볼드)

### 📐 레이아웃 및 간격 (Layout & Spacing)
*   **Spacing Unit:** 8px (모든 간격은 8px 단위로 설정하여 일관성 유지)
*   **Card/Container:** `padding: 16px;` (내부 여백), `border-radius: 12px;` (모서리 곡률)
*   **Flow Direction:** 위에서 아래로(Top-down). 정보의 중요도 순으로 명확하게 계층 구조를 설정한다.

## 3. 핵심 UI/UX 흐름 상세 정의 (Action Plan Flow Implementation)

플로우는 크게 **[Step 1: 진단 입력] → [Step 2: AI 분석] → [Step 3: 실행 계획 제시]**의 3단계로 구성되며, 각 단계는 명확한 시각적 피드백을 제공해야 한다.

### A. Step 1: 데이터 입력 및 초기 진입 (Input & Entry)
**목표:** 사용자가 심리적 부담 없이 데이터를 입력하도록 유도한다.
*   **컴포넌트:** `ProducerStoryForm.tsx` (데이터 입력 폼)
*   **디자인 사양:**
    1.  **진입 화면:** '당신의 가게를 성공으로 이끄는 청사진을 설계하세요'라는 헤드라인과 함께, 전체 과정에 대한 간결한 설명(3단계 흐름)을 제시한다.
    2.  **데이터 입력 폼:** 각 입력 필드는 `Soft Gray` 배경의 카드 내부에 위치하며, 필수 입력 항목에는 별도의 Red Alert 라벨(`Danger`)을 명시하여 누락을 방지한다.
    3.  **진행 표시줄 (Progress Bar):** 전체 플로우 대비 현재 단계(1/3)를 시각적으로 명확히 표시하며, 각 단계 완료 시 다음 단계로 자연스럽게 넘어가는 애니메이션을 적용한다.

### B. Step 2: AI 분석 및 진단 결과 제시 (AI Analysis & Diagnosis)
**목표:** 복잡한 AI 결과를 신뢰성 있게 전달하고, 사용자의 몰입도를 높인다.
*   **컴포넌트:** `DiagnosisResultView.tsx`
*   **디자인 사양:**
    1.  **AI 진단 결과 박스:** 중앙에 가장 크게 배치하며, 배경은 매우 옅은 `#F8F9FA`로 설정하여 시각적 집중도를 높인다.
    2.  **핵심 요약 (Executive Summary):** AI가 도출한 **'최종 ROI 예측치'**와 **'핵심 리스크 요인'**을 가장 상단에 `Primary Blue` 색상으로 강조하여 제시한다. (가장 중요한 정보)
    3.  **세부 진단 섹션:** '재무 분석', '시장 포지셔닝', '운영 효율성' 등 세부 항목별로 구분하여 제시한다. 각 항목은 별도의 `Card` 형태로 구성하며, 결과 옆에는 해당 결과의 **신뢰도 점수(0-100%)**를 원형 게이지(Gauge) 형태로 표시한다.
    4.  **Action Plan Preview:** 진단 결과 하단에 '실행 계획 요약' 섹션을 배치하고, 가장 중요한 3가지 액션 플랜 항목을 미리 보여준다.

### C. Step 3: 실행 가능한 액션 플랜 제시 (Action Prescription)
**목표:** 분석 결과를 즉시 실행 가능한 단계별 로드맵으로 변환한다.
*   **컴포넌트:** `ActionPlanFlowView.tsx`
*   **디자인 사양:**
    1.  **흐름 시각화:** 전체 Action Plan을 **수평 또는 계단식의 명확한 순서**로 제시한다. (혼돈 → 질서의 전환 강조)
    2.  **단계별 카드 디자인:** 각 실행 단계(예: 1단계: 데이터 정제, 2단계: 전략 수립, 3단계: 실행 계획)는 독립적인 `Card` 형태로 분리하여 보여준다.
        *   각 카드에는 **명확한 목표**와 **필요 자원**이 명시되어야 한다.
        *   **액션 버튼:** 각 단계의 완료를 위한 CTA 버튼(`Primary Blue`)을 제공하며, 다음 단계로 넘어가는 상호작용을 명확히 설계한다.
    3.  **리소스 연결:** 각 액션 항목 옆에 관련 데이터(예: 필요한 컨설팅 유형)와 **'관련 자료 (Link to Knowledge Base)'**를 명시하여, 사용자가 즉시 외부 자원으로 연결할 수 있도록 한다.

## 4. 개발팀 전달 사항 요약 (Developer Checklist)
1.  **API 연동:** Step 2의 결과는 백엔드 API(`/diagnosis`)에서 반환되는 `DiagnosisResult` 스키마를 정확히 매핑해야 함.
2.  **상태 관리:** 전체 플로우의 진행 상태(Step 1 → Step 2 → Step 3)는 전역 상태로 관리되어야 하며, 각 단계 완료 시 UI가 동적으로 전환되어야 함.
3.  **반응형:** 모든 레이아웃은 모바일 환경에서 최적화되도록 설계하며, 특히 데이터 표(Table) 영역은 스크롤을 통해 정보를 제공해야 함.
4.  **디자인 일관성:** 정의된 컬러 팔레트와 타이포그래피 규칙을 100% 준수하여 모든 컴포넌트를 구현할 것.
