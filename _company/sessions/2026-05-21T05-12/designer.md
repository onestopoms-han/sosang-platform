## 📝 ProducerStoryForm.tsx 최종 시각적 및 인터랙션 디자인 확정안

**목표:** 지역 생산자에게 '스토리텔링'을 유도하여 데이터 품질을 높이는 온라인 입력 양식의 최종 디자인 정의.

### 1. 브랜드 스타일 시스템 (Brand Style System)

이 디자인은 BDS소상공인플렛폼의 핵심 가치인 **신뢰(Trust), 지역성(Locality), 연결(Connection)**을 시각적으로 전달합니다.

| 요소 | 설정 값 | 설명 |
| :--- | :--- | :--- |
| **Primary Color (핵심)** | `#4A90E2` (BDS 블루) | 신뢰감과 전문성을 부여하는 메인 색상. |
| **Secondary Color (보조)** | `#F5A623` (골드 옐로우) | 지역성과 따뜻함을 강조하며 CTA에 사용. |
| **Background Color** | `#F9F9F9` (라이트 그레이) | 깨끗하고 집중도를 높이는 중립적인 배경. |
| **Text Color** | `#333333` (다크 그레이) | 가독성을 극대화. |
| **Typography (폰트)** | `Pretendard` (또는 시스템 기본 Sans 계열) | 모던하고 깔끔한 디지털 환경에 적합. |
| **Input Field Style** | Border: `#CCCCCC`, Focus Ring: `1px solid #4A90E2` | 입력 필드에 포커스 시 브랜드 컬러를 적용하여 상호작용 강조. |

### 2. 레이아웃 구조 (Layout Structure)

폼은 세 단계의 스토리텔링 흐름을 반영하여 **'Hooking (관심 유도) → Deep Dive (상세 스토리) → Validation (신뢰 확인)'** 순으로 구성합니다. 전체 너비는 반응형을 고려하여 최대 800px를 기준으로 설계합니다.

| 섹션 | 제목/목표 | 레이아웃 상세 | 인터랙션 디자인 |
| :--- | :--- | :--- | :--- |
| **Header (Hook)** | **[Step 1: 가게의 시작]** - 가장 강력한 첫인상을 유도 | 중앙 정렬. 배경색: `#4A90E2` (Primary Color). 제목은 Bold, 크기 `28px`. | 상단에 스토리텔링 핵심 질문(예: "당신의 가게는 어떤 이야기를 가지고 있나요?")을 배치하고, 아래로 스크롤 시 다음 단계로 자연스럽게 유도. |
| **Section A (Deep Dive)** | **[Step 2: 이야기의 심층]** - 구체적인 배경과 가치 설명 | 입력 필드 그룹(Grid Layout). 각 항목은 카드 형태로 분리하여 배치. 최대 3열 그리드 사용. | 각 스토리 항목(예: 생산자 배경, 핵심 철학) 옆에 작은 아이콘(💡, 🗺️ 등)을 배치하여 시각적 흥미 유발. 입력 필드 하단에 미니멀한 설명(Helper Text) 제공. |
| **Section B (Validation)** | **[Step 3: 신뢰의 증명]** - 데이터 무결성 확인 | 필수 정보 체크리스트 및 데이터 매핑 영역. | 모든 스토리 항목이 완료되면, 자동으로 다음 섹션으로 전환되는 명확한 진행 바(Progress Bar) 표시. 입력 완료 시 '다음 단계로 이동' 버튼 활성화. |
| **Footer (CTA)** | **[Next Action]** - 행동 유도 | 전체 너비에 걸친 CTA 영역. 배경색: `#F5A623` (Secondary Color). | 버튼 텍스트는 Primary Color (`#4A90E2`)로 강조. "스토리 제출하고 잠재 고객을 만나보세요"와 같은 스토리 중심 문구 사용. |

### 3. 핵심 컴포넌트 상세 디자인 (Component Details)

#### A. 스토리 입력 필드 (Story Input Field - 예시)

*   **구조:** `[레이블] + [입력 영역] + [힌트/규칙]`
*   **스타일:** 최소 높이 48px. 테두리는 `#CCCCCC`로 설정.
*   **인터랙션:** 사용자가 필드에 포커스하면 테두리가 즉시 `#4A90E2`로 변경됩니다. (Focus Ring 효과)
*   **JSON Schema 반영:** 모든 입력 필드는 `required: true` 속성을 명확히 표시하며, 누락 시 붉은색 경고(`Error Message`)가 아래에 표시되어야 합니다.

#### B. 데이터 흐름 시각화 (Data Flow Visualization)

*   페이지 상단에 **'스토리텔링 플로우 다이어그램'**을 간략하게 배치합니다.
    *   `[생산자 정보] → [핵심 가치 정의] → [시장 연결점 도출] → [최종 스토리 완성]`
*   이 흐름은 사용자가 현재 어디에 있는지 시각적으로 인지하게 하여, 입력 단계가 단순한 데이터 채우기가 아닌 **'스토리 구축 과정'**임을 강조합니다.

### 4. 코다리 피드백 제안 (Feedback for Coda)

코다리에게는 이 디자인이 단순히 데이터를 수집하는 양식이 아니라, **신뢰를 기반으로 한 가치 창출을 유도하는 스토리텔링 엔진의 입력단**임을 명확히 전달합니다.

> **[피드백 요약]**
> "최종 `ProducerStoryForm.tsx`는 단순한 데이터 폼이 아닌, 지역 생산자의 잠재적 스토리를 구조화하고 신뢰를 구축하는 인터랙티브 스토리텔링 엔진의 입력단으로 설계되었습니다. 핵심은 **'데이터 입력'을 '가치 증명 과정'**으로 포장하는 것입니다.
>
> **요청 사항:** JSON Schema 기반의 유효성 검사 로직(Negative Testing에서 정의된)이 이 시각적 흐름과 완벽하게 일치하도록, 폼 컴포넌트 내부의 상태 관리 및 에러 메시지 시스템을 설계해 주십시오. 특히, 스토리 점수가 낮거나 모순될 경우, 입력 단계에서 즉각적인 **'스토리 보강 가이드(Story Enhancement Guidance)'**를 제공하는 인터랙션을 포함해야 합니다."
