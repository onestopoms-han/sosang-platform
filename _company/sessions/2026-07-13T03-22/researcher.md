## 📊 PLRI 지표: 위험 레벨별 정의 및 데이터 요구사항 상세 분석

PLRI(Platform Risk & Liability Index)를 세 가지 단계로 분류하고, 각 단계에 필요한 **핵심 데이터 변수 3가지**와 이를 수집하기 위한 **API 연동 로직 요구사항**을 아래와 같이 정의합니다. 이 기준은 사용자의 재무적 손실뿐만 아니라 심리적 불안정성(Emotional Risk Management)을 측정하는 데 중점을 둡니다.

### 1. Critical (위기) 레벨: 즉각적인 개입 필요 단계
**목표:** 시스템이 **즉시 경고 및 비상 조치(Actionable Blueprint)**를 요구하여 손실 확산을 막는 것을 목표로 합니다.
**심리적 상태:** 극도의 불안정성, 손실 회피 심리가 최고조에 달함 (Red Tone 시각화 필수).

| 핵심 데이터 변수 | 측정 기준 (Threshold) | API 연동 로직 요구사항 |
| :--- | :--- | :--- |
| **① 이익 변동성 (Margin Volatility)** | 목표 마진 대비 **-25% 이상**의 예측치 발생 또는 실제 달성 시점. | **[API: Real-time Financial Feed]** 원자재 가격, 운영 비용(인건비/임차료)의 실시간 지표를 연동하여 목표 이익과의 편차를 계산하고 즉시 Critical으로 플래그 지정. |
| **② 유동성 위험 (Liquidity Risk)** | 30일 내 예상 현금 흐름이 **마이너스**로 전환되거나, 필수 운영 자금 부족 상태. | **[API: Cash Flow Projection]** 플랫폼 데이터(판매량, 재고 회전율)와 연동하여 향후 30일간의 예측 현금 흐름 시뮬레이션을 제공하고 마이너스 발생 시 즉시 경고 트리거. |
| **③ 공급망 안정성 (Supply Chain Stability)** | 핵심 원자재 또는 필수 부품 수급에 **7일 이상 지연**이 예상되거나, 대체 공급처 확보 실패. | **[API: Supply Chain Tracker]** 계약된 공급업체의 실시간 재고 및 생산 계획 데이터를 연동하여 병목 현상 발생 시 즉시 Critical 알림을 생성. |

### 2. Caution (주의) 레벨: 선제적 관리가 필요한 단계
**목표:** 잠재적인 위험 신호를 감지하고, 사용자가 **행동 가능한 실행 계획(Actionable Blueprint)**을 통해 리스크를 관리할 수 있도록 유도합니다.
**심리적 상태:** 경계심 증가, 의사결정 지연 발생 (Yellow Tone 시각화).

| 핵심 데이터 변수 | 측정 기준 (Threshold) | API 연동 로직 요구사항 |
| :--- | :--- | :--- |
| **① 이익 변동성 (Margin Volatility)** | 목표 마진 대비 **-10% ~ -25%**의 예측치 발생. | **[API: Historical Performance Analysis]** 과거 6개월 평균 데이터와 비교하여 현재 추세가 위험 범위에 진입했음을 통계적으로 제시하고, 개선 방안을 제안하는 모듈 연동. |
| **② 비용 효율성 (Cost Efficiency)** | 예상 운영 비용 대비 실제 지출이 **15% 이상 초과**되었거나, 비효율적인 지출 항목(예: 마케팅 비용)이 임계치를 초과한 경우. | **[API: Expense Tracking & Benchmarking]** 플랫폼 내에서 집계된 비용 데이터와 업종별 평균 데이터를 비교하여 예산 초과 지점을 시각적으로 표시하고 절감 방안을 제시. |
| **③ 시장 수요 변화 (Market Demand Shift)** | 핵심 상품의 판매량 또는 수요가 **전월 대비 10% 이상 감소**했으나, 아직 Critical 수준은 아닌 경우. | **[API: Trend Analysis]** 외부 트렌드 데이터 및 플랫폼 내 거래 데이터를 교차 분석하여 수요 감소 추세를 포착하고 선제적 재고 조정이나 마케팅 전략 변경을 제안. |

### 3. Stable (안정) 레벨: 최적의 운영 단계
**목표:** 모든 지표가 목표 범위 내에 있으며, 플랫폼이 **책임 보증 및 성장 지원** 모드로 전환되어 사용자가 안정적으로 사업을 운영하도록 지원합니다.
**심리적 상태:** 안정감 회복, 통제권 확립 (Green Tone 시각화).

| 핵심 데이터 변수 | 측정 기준 (Threshold) | API 연동 로직 요구사항 |
| :--- | :--- | :--- |
| **① 이익 변동성 (Margin Volatility)** | 목표 마진 대비 **-10% 이하**의 안정적인 범위 내에서 운영됨. | **[API: Baseline Performance]** 과거 12개월 데이터와의 비교를 통해 현재 상태가 성공적인 평균 범주 내에 있음을 확인하고, 지속 가능한 성장을 위한 장기 예측 모델 제공. |
| **② 비용 효율성 (Cost Efficiency)** | 모든 운영 비용이 업계 표준 대비 **5% 이내**로 관리되고 있음. | **[API: Cost Optimization Report]** 최적화된 운영 방식을 유지하도록 지원하며, 추가적인 투자(Scale-up) 기회를 탐색하는 모듈 연동. |
| **③ 공급망 안정성 (Supply Chain Stability)** | 모든 핵심 자원의 수급이 계획대로 이루어지며, 리스크 발생 확률이 **5% 미만**으로 예측됨. | **[API: Predictive Fulfillment]** 실시간 물류 데이터를 기반으로 향후 일정 기간 동안의 공급 안정성을 95% 이상 보장하는 시뮬레이션 결과 제공. |
