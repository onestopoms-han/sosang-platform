# 🛡️ BDS소상공인플렛폼: 컴포넌트 통합 설계 명세서 및 UI/UX 가이드북 (V1.0)

> **목표:** Story Flow API의 모든 오류 상황(Payment Failure, Resource Not Found 등)에서 사용자가 느끼는 불안감을 최소화하고, 플랫폼의 핵심 가치인 '품질 신뢰'를 시각적 인터페이스로 즉시 전달할 수 있는 표준 컴포넌트 라이브러리를 확정한다.
> **참조:** Coder의 Handoff Specification (Payment Failure Screen & Error Handling)

---

## 📐 1. 디자인 원칙: '신뢰 구축(Trust Building)' 우선순위
모든 오류/경고 상황은 사용자에게 *문제*를 지적하는 것이 아니라, *해결책*을 제시하고 *안전한 다음 단계*로 이끄는 과정으로 설계되어야 합니다.

1.  **공감 (Empathy):** "실패" 대신 "잠시 멈춤/재확인 필요"라는 메시지를 사용합니다.
2.  **명료성 (Clarity):** 오류의 원인을 명확하고 구체적인 용어로 제시합니다. (예: '오류 발생' X → '결제 정보가 유효하지 않습니다.' O)
3.  **안정감 (Stability):** 톤앤매너는 경고를 주더라도, 메인 UI와 통일된 안정적인 컬러 팔레트(Primary Blue 계열)를 유지하여 혼란을 방지합니다.

## 🧱 3. 핵심 컴포넌트 라이브러리 (Core Components)

### A. [Error Card Component] - 자원/흐름 오류 처리
*   **용도:** API 호출 실패, 데이터 로드 불가 등 시스템 리소스 관련 문제 발생 시 화면 상단에 표시되는 박스형 알림. (`STORY_NOT_FOUND` 대응)
*   **레이아웃 (Mobile 기준):** `padding: 20px`, 배경색: `--trust-bg-error`.
*   **구조:**
    1.  [Icon]: ⚠️ 느낌표 또는 경고 아이콘.
    2.  [Title]: **(H3)** 오류 발생: [오류 코드 명시] (예: 스토리 리소스 없음)
    3.  [Body]: (P) 사용자가 요청하신 'OOO' 콘텐츠는 현재 시스템에 존재하지 않거나 접근 권한이 없습니다.
    4.  [CTA]: **(Button, Disabled)** 문의하기 / 도움말 보기

### B. [Payment Failure Modal Component] - 결제 관련 오류 처리
*   **용도:** 결제가 최종적으로 실패했음을 알리고 재시도를 유도하는 모달창. (가장 중요함)
*   **레이아웃 (Modal):** `width: 80vw`, 배경색: `--trust-bg-warning`.
*   **구조:**
    1.  [Icon]: ❌ 결제 실패 아이콘.
    2.  [Title]: **(H2)** 결제를 완료할 수 없습니다. (결제 인증 필수)
    3.  [Body]: (P) 입력해주신 카드 정보가 유효하지 않거나, 일시적인 네트워크 문제로 인해 처리에 실패했습니다.
    4.  **[Failure Detail]**: *구체적 원인 안내*: "카드사 정책상 이 지역은 결제 서비스 이용이 제한될 수 있습니다."
    5.  [CTA 1]: **(Primary Button)** 다시 시도하기 (클릭 시, 에러 카운터 로직 연동)
    6.  [CTA 2]: **(Secondary Link)** 결제 방법 변경 (비회원/다른 카드사 선택 유도)

### C. [Network Timeout Toast Component] - 일시적 오류 처리
*   **용도:** 네트워크 연결 문제, 서버 과부하 등 일시적인 비동기 통신 실패 시 화면 하단에 나타나는 짧은 토스트 알림. (`NETWORK_TIMEOUT` 대응)
*   **레이아웃:** `fixed bottom`, 배경색: `--trust-bg-warning`.
*   **메시지:** "잠시만요... 네트워크 연결 상태를 확인해 주세요." (자동 5초 후 사라짐)

## ✨ 4. 통합 UX 흐름 예시 (Story Flow 구매 실패 시나리오)

1.  **사용자 액션:** 사용자가 '품질 인증된 스토리' 구매 버튼 클릭.
2.  **시스템 응답 (Failure):** API 호출 -> `PAYMENT_FAILED` 응답 수신.
3.  **UI 반응:** [Payment Failure Modal Component]이 팝업됨.
4.  **사용자 인지:** "결제를 완료할 수 없습니다."라는 경고를 통해 즉시 문제 파악.
5.  **시스템 안내:** 모달 내부에 구체적인 실패 원인 (예: 카드번호 오류)을 제시하고, **'다시 시도하기'**와 **'다른 방법으로 결제'**의 선택지를 명확히 제공하여 다음 행동을 유도한다.
