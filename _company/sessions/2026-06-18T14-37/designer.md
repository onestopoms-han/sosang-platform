# ✨ Trust Widget & PainGauge: 최종 디자인 시스템 사양 (V1.0)

## 1. 브랜드 컬러 팔레트 확정

이전 논의된 브랜드 아이덴티티(Deep Blue, Growth Green)와 요청하신 핵심 액션 컬러를 통합하여 시각적 계층 구조를 명확히 합니다.

| 역할 | 색상 코드 (Hex) | 용도 및 의미 |
| :--- | :--- | :--- |
| **Primary (신뢰/기반)** | `#4CAF50` (Growth Green) | 성공, 긍정적 결과, 성장 지표(Trust Widget의 긍정 상태). |
| **Secondary (위험/경고)** | `#F44336` (Error Red) | 위험, 부정적 결과, 경고 지표(PainGauge의 위험 상태). |
| **Base (신뢰/안정)** | `#2196F3` (Deep Blue - 기존 시스템 유지) | 기본 배경, 핵심 정보, 플랫폼의 신뢰 기반. |
| **Neutral (텍스트/배경)** | `#FFFFFF`, `#F5F5F5` | 본문 텍스트 및 UI 배경. |
| **Accent (중립)** | `#757575` | 보조 정보, 구분선 등. |

## 2. Trust Widget 설계 사양 (신뢰 지표)

**목표:** 사용자에게 플랫폼과 컨설팅의 신뢰도를 명확하게 전달합니다.

### A. 상태별 시각화 규칙

| 상태 | 설명 | 배경 색상 | 테두리/아이콘 색상 | 텍스트 가이드라인 |
| :--- | :--- | :--- | :--- | :--- |
| **매우 낮음 (Low)** | 신뢰도가 매우 낮음. | `#F44336` (경고 레드) | `#F44336` | "낮은 신뢰도" 또는 "확인 필요" |
| **보통 (Medium)** | 평균적인 신뢰 수준. | `#FFEB3B` (경고 옐로우) | `#FFC107` | "평균 신뢰도" 또는 "데이터 확인 필요" |
| **높음 (High)** | 충분한 신뢰 수준. | `#4CAF50` (성장 그린) | `#4CAF50` | "높은 신뢰도" 또는 "확인 완료" |
| **최고 (Max)** | 최고 수준의 신뢰. | `#4CAF50` (성장 그린) | `#4CAF50` | "신뢰 확보됨" 또는 "검증 완료" |

### B. 컴포넌트 요구사항

*   **레이아웃:** 원형(Circular) 또는 진행 바(Progress Bar) 형태를 기본으로 하며, 중앙에 명확한 숫자와 상태 텍스트를 배치합니다.
*   **상호작용:** 마우스 오버 시 상세 설명 툴팁을 제공하여 데이터의 의미를 즉시 파악할 수 있도록 합니다.
*   **데이터 연동:** 백엔드에서 반환되는 신뢰 점수(0~100%)에 따라 위 표의 색상과 텍스트가 동적으로 변경되어야 합니다.

## 3. PainGauge 설계 사양 (고통/문제 지표)

**목표:** 소상공인이 현재 직면한 어려움의 심각도를 시각화하여 해결의 필요성을 강조합니다.

### A. 상태별 시각화 규칙

| 상태 | 설명 | 배경 색상 | 테두리/아이콘 색상 | 텍스트 가이드라인 |
| :--- | :--- | :--- | :--- | :--- |
| **매우 높음 (Critical)** | 심각한 고통 또는 문제 발생. | `#F44336` (경고 레드) | `#F44336` | "심각한 어려움" 또는 "즉시 해결 필요" |
| **높음 (High)** | 상당한 고통이 존재함. | `#FF9800` (경고 오렌지) | `#FF9800` | "상당한 어려움" 또는 "개선 필요" |
| **보통 (Medium)** | 일반적인 수준의 어려움. | `#FFEB3B` (경고 옐로우) | `#FFEB3B` | "일반적인 어려움" 또는 "관심 필요" |
| **낮음 (Low)** | 어려움이 적거나 해결됨. | `#4CAF50` (성장 그린) | `#4CAF50` | "어려움 완화" 또는 "해결 중" |

### B. 컴포넌트 요구사항

*   **레이아웃:** 리니어 바(Linear Bar) 형태로, 채워진 정도가 고통의 크기를 직관적으로 나타내도록 설계합니다.
*   **상호작용:** 사용자가 슬라이더를 조절하거나 마우스를 올릴 때, 해당 수치에 따른 구체적인 행동 제안(Actionable Insight)이 함께 표시되어야 합니다.
*   **데이터 연동:** 백엔드에서 반환되는 고통 지표 값(예: 0~100%)에 따라 위 표의 색상과 텍스트가 동적으로 변경되어야 합니다.

## 4. 최종 설계 요약 및 개발 지침

1.  **디자인 시스템 통합:** 모든 컴포넌트는 `Deep Blue`를 기본 신뢰의 기반으로 사용하고, **Growth Green (`#4CAF50`)**을 긍정적 결과(Trust)와 해결책(PainGauge 개선)에 할당하여 플랫폼의 목표 지향적인 성장을 강조합니다.
2.  **명확한 대비:** 위험(`F44336`)과 성공(`4CAF50`) 색상을 명확히 분리하여 사용자가 데이터 해석에 혼란을 느끼지 않도록 합니다.
3.  **텍스트 우선:** 시각적 요소 외에, 각 상태에 대한 텍스트 가이드라인(예: "신뢰 확보됨")을 필수적으로 포함하여 AI의 추상적인 수치를 실제 행동 지침으로 전환합니다.
4.  **개발 착수:** 이 사양을 기반으로 **`TrustWidget.tsx`**와 **`PainGauge.tsx`** 컴포넌트의 초기 Mockup 및 API 연동 로직 구현에 착수하십시오.
