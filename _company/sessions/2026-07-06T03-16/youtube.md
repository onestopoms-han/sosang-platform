# 🎥 유튜브 영상 제작 실행 가이드라인 (베타 모집용)

## 1. 기획 및 목표 재확인
**최종 목표:** 플랫폼 베타 참여자 모집을 위한 신뢰 기반 전환 유도.
**핵심 전략:** 감성적 공감(Pain) → 데이터 기반 해결책 제시(Solution) → 행동 유도(CTA).

## 2. 촬영 준비 (Shooting Prep) 상세 지침
### A. 비주얼 시스템 적용 가이드라인 (Trust Widget/PainGauge V3.0)
*   **Tone & Manner:** 불안함에서 확신으로의 시각적 전환을 극대화해야 합니다. 전체 영상 톤은 어둡고 답답한 골목 분위기에서 시작하여, 데이터 제시 구간에서는 **청명하고 신뢰감 있는 파란색/녹색 계열**로 명확하게 전환되어야 합니다.
*   **Trust Widget (신뢰 위젯):** 매출 불안정 구간(문제 제기) 직후와 솔루션 제시 직전에 배치하여 시각적 앵커 역할을 수행합니다.
*   **PainGauge (고통 측정 게이지):** 데이터 기반 예측이 필요한 지점(예: '수요 예측 부재' 언급 시)에 맞춰 불안정함을 극대화하여 보여줍니다.

### B. 시간대별 촬영 체크리스트 및 샷 구성
| 시간대 | 내용 (스크립트 기반) | 비주얼 요구사항 | 촬영 중 강조할 감정/연출 |
| :--- | :--- | :--- | :--- |
| **0:00 - 0:03** | [강렬 후크] 매출에 대한 질문 던지기 | 🟥 빨간색 필터, 답답한 골목 풍경 + 사장님 고개 숙인 모습 | 불안감, 공포 (긴박함) |
| **0:03 - 1:30** | [문제 제기/공감] 직관의 한계 지적 | 손으로 계산하는 장면 vs 데이터 화면 대비 | 좌절, 혼란 (불안정성 강조) |
| **1:30 - 3:00** | [핵심 이론/지식 자본화] 수요 예측 부재 지적 | 공식 1, 2, 3 그래픽화 | 이해, 깨달음 (인지의 전환) |
| **3:00 - 5:00** | [솔루션 시연/MVP 데모] AI 리스크 예측 시스템 소개 | Loss Gauge 애니메이션(🟢→🟡→🔴), 대시보드 화면 클로즈업 | 희망, 확신 (해결책 제시) |
| **5:00 - End** | [결론/행동 유도] 실행 가능한 다음 스텝 제시 | 명확한 CTA 버튼 강조, 플랫폼 로고 노출 | 행동 촉구, 자신감 (실행 동기 부여) |

## 3. 편집 방향 설정 (Editing Direction)
*   **템포 조절:** 초반부(0~1:30)는 공감과 불안감을 전달하기 위해 **약간 느리고 감성적인 BGM**을 사용하되, 문제 제기 부분에서는 시각적 대비를 통해 템포를 살짝 높여 긴장감을 유지합니다.
*   **데이터 시연 구간:** AI 리스크 예측 시스템(3:00~) 설명 구간에서는 BGM을 **명료하고 신뢰감 있는 미니멀한 음악**으로 전환하며, 모든 데이터 애니메이션은 빠르고 명확하게 제시하여 시청자가 정보를 소화하는 데 집중하도록 유도합니다.
*   **텍스트 오버레이:** 핵심 키워드(안전 마진, 예측, 시스템)는 화면 중앙에 크고 명확하게 배치하고, 디자인 시스템의 컬러 팔레트(빨강→노랑→파랑/초록) 변화를 시각적 흐름과 완벽히 일치시킵니다.
