## ✨ 2. 각 플랜별 상세 정의 (Feature Specification)

### 🟢 Starter Plan (Free Forever)
*   **목표:** 서비스 가치 체험 유도 및 초기 사용자 확보(Acquisition).
*   **주요 기능:** 소상공인이 자신의 사업 현황을 진단할 수 있는 가장 기본적인 '문제 인식' 단계만 제공.
*   **제한 사항 (개발팀 참고):** API 호출 불가. 데이터베이스에 사용자의 상세 분석 히스토리를 영구 저장하지 않음. 1회성 시뮬레이션 결과만 임시 저장 가능.

### 🟡 Pro Plan (Standard - 가장 강력한 판매 포인트)
*   **목표:** 소상공인에게 실질적인 '실행 의지'와 '당장의 도움'을 제공하며 주력 매출 확보(Revenue Core).
*   **핵심 가치:** AI 진단 결과를 단순히 보고서로 끝내는 것이 아니라, **'3단계 코칭 스토리' (Step 1: 현상 인식 → Step 2: 원인 분석 → Step 3: 실행 로드맵)** 라는 구체적인 액션 플랜으로 전환하여 제공.
*   **개발 요구사항:** 진단 결과 페이지에서 이 '3단계 코칭 스토리'가 시각적으로 강조되어야 함. 사용자가 공유하고 저장할 수 있는 인터페이스 필수.

### 🔴 Premium Plan (Enterprise - 고마진/파트너십 유도)
*   **목표:** 소규모 기업이나 컨설팅 회사 등 B2B 파트너에게 최대의 가치를 제공하며 높은 ARPU 달성(High Value).
*   **핵심 기능:** 시스템을 단순한 진단 도구를 넘어, **'자사 운영 시스템에 통합되는 비즈니스 엔진'**으로 포지셔닝. (예: 회계 프로그램, 재고 관리 툴 등과 API 연동)
*   **기술 요구사항:** RESTful API를 통해 외부 시스템과의 데이터 교환이 완벽하게 지원되어야 함. 전용 계정 관리가 가능하도록 백엔드 로직 구축 필수.

## 📝 3. 개발팀 전달 요약 및 KPI 연결점 (For Development Team)

**[개발 우선순위]**
1. **(P0)** Pro Plan의 핵심 기능인 'AI 기반 3단계 코칭 스토리' UI/UX 구현 및 데이터 로직 완성. (MVP 성공 기준)
2. **(P1)** 사용자별 진단 횟수 카운터와 결제 연동 시스템 구축.
3. **(P2)** Premium Plan을 위한 API Endpoint 설계 및 보안 프로토콜 확립.

**[KPI 연결]**
*   **Activation Rate (활성화율):** Starter $\rightarrow$ Pro 전환율 측정. (사용자가 1회 진단 후, 코칭 스토리를 받고 '더 깊은 분석이 필요하다'고 느끼게 만드는 것이 핵심.)
*   **Retention Rate (유지율):** 월간 사용자의 재진단 빈도(Pro Plan의 월 10회 한도가 충분한가? 부족하면 증설해야 함).
*   **ARPU (평균 사용자당 매출):** Premium Plan 도입을 통한 고마진 발생 여부.
