🎨 Designer: 작업 시작합니다.

CEO의 지시를 확인했습니다. 확정된 Trust Widget 및 PainGauge 디자인 시스템(V3.0)을 기반으로, 개발팀이 제공할 기술적 사양과 시각적 레이아웃 가이드라인을 통합하여 콘텐츠에 최적화된 비주얼 흐름을 최종 확정하는 것이 현재의 핵심 작업입니다.

저는 이 작업을 위해 다음 세 가지 핵심 요소를 중심으로 검토 프레임을 준비하겠습니다.

### 1. 디자인 시스템 재확인 (V3.0 기반)
모든 시각적 결정은 아래 V3.0 시스템의 원칙을 절대적으로 준수해야 합니다.
*   **톤앤매너:** 신뢰(Trust)와 긴급성/위험(Pain)의 감성적 균형 유지.
*   **컬러 팔레트:** (V3.0에서 정의된 Primary, Secondary, Accent 컬러 사용 명세).
*   **타이포그래피 계층:** 정보의 중요도에 따른 폰트 크기, 굵기, 간격의 엄격한 위계 설정.

### 2. 기술 사양 vs. 시각적 레이아웃 매핑 (Mapping Framework)
개발팀으로부터 제공받을 기술 스펙(API 응답 구조, 데이터 흐름 정의 등)을 다음의 비주얼 요소와 1:1로 매핑하여 최종 비주얼 흐름을 확정합니다.

| 기술 입력 항목 (개발팀 제공) | 디자인 변환 목표 (Designer 역할) | 적용되는 V3.0 컴포넌트 | 기대하는 시각적 결과물 |
| :--- | :--- | :--- | :--- |
| **Trust Score** (예: 0-100%) | 신뢰도 측정의 즉각적인 인지 | Trust Widget (V3.0) | 동적 레벨 바 또는 게이지 시각화 |
| **Pain Level** (예: High/Medium/Low) | 위험도에 따른 긴급성 전달 | PainGauge (V3.0) | 색상 기반 경고 신호 및 대비 강조 |
| **StoryFlow Step** (현재 단계) | 사용자 여정의 명확한 위치 표시 | Navigation Bar / Contextual Frame | 현재 위치를 강조하는 시각적 포커스 |
| **Data Transition Logic** (전환 조건) | 논리적 흐름의 시각화 | Flow Connector / Animation Rule | 다음 단계로 이동할 때 발생하는 시각적 효과 |

### 3. 최종 비주얼 흐름 확정 (Visual Flow Finalization)
기술 사양에 따라, 사용자가 플랫폼을 경험하는 과정(StoryFlow)이 **'감성적 신뢰 형성'**과 **'데이터 기반 코칭 제공'**이라는 두 가지 목표를 달성하도록 시각적으로 설계합니다.

*   **흐름 확정 원칙:** 데이터는 숨기지 않고, 감성은 데이터를 포장하여 전달해야 합니다. (Data-Driven Empathy)
*   **최적화 포인트:** Trust Widget과 PainGauge는 단순한 수치가 아니라, 사용자가 다음 행동을 취하도록 유도하는 **'코칭 신호'**로 기능하도록 디자인합니다.
