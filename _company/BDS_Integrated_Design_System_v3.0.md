## 💡 Part 1: 브랜드 코어 & 시각적 언어 (Brand Core & Visual Language)

### 1. 컬러 팔레트 및 역할 정의
| 이름 | HEX Code | 용도 (Role) | 사용 규칙 (Usage Rule) |
| :--- | :--- | :--- | :--- |
| **Primary Green** (Solution) | `#2ECC71` | 성공, 안정성, 행동 유도(CTA), Solution 영역 강조. | 긍정적 결과를 나타내는 핵심 색상. CTA 버튼 배경색으로 우선 사용한다. |
| **Warning Red** (Pain) | `#FF4D4D` | 위험, 문제 발생, 경고 알림, Pain 시나리오 제시. | 사용자에게 즉각적인 주의(Attention)가 필요함을 인지시키는 데만 제한적으로 사용하며, 장시간 노출을 피한다. |
| **Neutral Blue** (Trust/Info) | `#3498DB` | 정보 전달, 시스템 기능 안내, 신뢰 구축 영역. | '안전한 거래 경험' 및 플랫폼의 전문성을 강조하는 보조 색상으로 활용한다. |
| Background | `#F7F9FA` | 주 배경색. 모든 콘텐츠는 이 색상을 기준으로 가독성을 확보해야 한다. |

### 2. 타이포그래피 (Typography)
*   **메인 헤드라인:** Pretendard Bold, Size: 32px / Weight: 700 (가장 중요한 메시지)
*   **서브 제목:** Pretendard SemiBold, Size: 18px / Weight: 600
*   **본문 텍스트:** Pretendard Regular, Size: 14px / Weight: 400
*   **규칙:** **'핵심 메시지(H1)'는 무조건 가장 크게, '행동 유도 문구(CTA)'는 대비가 강한 색상으로 처리한다.**

### 3. 비주얼 트랜지션 (Pain $\to$ Solution Flow)
| 단계 | 시각적 표현 | 의미 부여 | 적용 범위 |
| :--- | :--- | :--- | :--- |
| **Phase 1: Pain** | `#FF4D4D` 필터, 불안한 구도(Low Angle), 혼란스러운 이미지. | '문제의 인식' 및 '현상 유지의 위험성'. 감정적 동요 유발. | 모든 마케팅 콘텐츠와 진단 초기 단계 UI (예: 손실 비용 계산 화면). |
| **Transition** | 노랑/주황색 경고 애니메이션 (⚠️), 데이터 그래프 등장. | '근본적인 원인 파악' 및 '새로운 해결책의 필요성'. 지적 호기심 유발. | 진단 과정, 시스템 분석 리포트 제공 시. |
| **Phase 2: Solution** | `#2ECC71` 필터, 안정된 구도(Eye Level), 명확한 데이터 대시보드 UI. | '안정성', '명확한 해결책 제시'. 신뢰 및 행동 개시 유도. | 플랫폼의 핵심 기능 화면, 성공 스토리 콘텐츠, 최종 CTA 영역. |

## 🧩 Part 2: 컴포넌트 시스템 정의 (Component System)

개발팀과 마케팅 에셋 제작 시 반드시 참조해야 할 표준화된 요소들입니다.

### 1. 버튼 (Button Component - `BDS_Btn`)
*   **기본형 (Primary CTA):** 배경 `#2ECC71`, 텍스트 흰색, 그림자(Shadow) 적용 권장. (사용자가 가장 먼저 클릭해야 하는 '다음 단계'에만 사용.)
    *   *개발 지침:* 모든 버튼 컴포넌트는 `:hover` 시 미세한 애니메이션 변화(Scale-up 또는 Shadow Deepening)를 포함해야 한다.
*   **보조형 (Secondary CTA):** 배경 투명, 테두리 `#3498DB`, 텍스트 `#3498DB`. (부가 기능이나 정보 확인에 사용.)

### 2. 카드 컴포넌트 (Card Component - `BDS_Card`)
*   **목적:** 콘텐츠의 구조화 및 가독성 확보.
*   **Pain Card 예시:** 상단 배경을 `#FFEDED`로 처리하고, 경고 아이콘(🚨)과 함께 배치하여 시각적 위기감을 조성한다.
*   **Solution Card 예시:** 성공 아이콘(✅)과 함께 배치하며, 충분한 여백(Whitespace)을 확보하여 안정감 있는 느낌을 준다.

### 3. 데이터 시각화 (Data Visualization - Loss Gauge)
*   **Loss Gauge 표준:** 위험도를 나타내는 게이지는 반드시 **빨간색(🔴)**에서 시작해 **노란색(🟡)**을 거쳐 **녹색(🟢)**으로의 명확한 색상 변화와 함께 애니메이션 효과를 제공해야 한다. (단순 정지 이미지는 금지.)
*   **데이터 레이블:** 모든 데이터 차트에는 반드시 '현재 상태'와 '개선 목표치' 두 가지 수치를 병기하여 제시한다.

## 📱 Part 3: 마케팅 에셋 가이드라인 (Marketing Asset Guidelines)

(Instagram, 유튜브 등 외부 채널에 적용 시 지침)

1.  **시각적 일관성:** 광고 콘텐츠 제작 시에도 **Part 2의 컴포넌트와 컬러를 따르는 것처럼 보이게** 디자인해야 한다. 즉, 아무리 광고라 하더라도 'BDS 시스템'을 사용하는 느낌이 나야 한다.
2.  **CTA 위치 고정:** 모든 형태의 마케팅 에셋(릴스 마지막 장면, 피드 이미지)은 **화면의 우측 하단 또는 좌측 상단에 플랫포머 로고와 함께 CTA 배너 영역**을 확보해야 한다. (예: "7일 무료 진단 받고 시작하기")
3.  **폰트 통일:** 마케팅 카피가 아무리 짧더라도, 시스템 내에서 사용되는 Pretendard 계열의 굵기 및 크기 비율(헤드라인:본문 = 2:1)을 유지해야 한다.
