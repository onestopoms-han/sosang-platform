# 🔎 BDS 소상공인 플랫폼: 최종 시각적 검토 체크리스트 (V3.0)

**목표:** 플랫폼의 핵심 가치인 '기술적 안정성'과 '금융적 안전망(안정성 보험료)'이 사용자 인터페이스 전반에 걸쳐 명확하고 신뢰감 있게 전달되는지 최종 점검한다.
**작성자:** 🎨 Designer (Lead Designer)
**최종 검토 대상 영역:** 대시보드 메인 화면, 결제 실패 처리 플로우

## 💔 섹션 2: 결제 실패 화면 UX/UI 검토 (Payment Failure Screen Flow)

| 구분 | 체크 항목 | 상세 점검 내용 (기준) | 담당 에이전트 | 완료 여부 | 비고/조치 사항 |
| :--- | :--- | :--- | :---: | :--- | :--- |
| **① 실패 메시지의 톤** | 감정적 공감 및 진단 | "결제에 실패했습니다"라는 단순 오류 문구 대신, 소상공인의 불안감을 읽고 *진단*해주는 공감형 문장으로 시작하는가? (예: "혹시 결제 수수료나 기술적인 문제로 어려움을 겪으셨나요?") | Writer/Designer | [ ] | |
| **② 해결책 제시의 흐름** | Fail $\rightarrow$ Solution 전환 유도 | 실패 메시지 직후, 단순히 재시도를 요청하는 것이 아니라, '안정성 보험료'라는 명확한 *해결 경로*와 함께 다음 행동을 제안하는가? (강제적으로 Solution으로 유도) | Designer/Writer | [ ] | |

| **④ 오류 코드의 처리** | 기술적 상세 정보 노출 최소화 | 사용자에게는 '해결책'에 집중시키고, 복잡한 오류 코드나 기술적인 디테일은 숨기거나 FAQ를 통해 접근하도록 가이드하는가? (UX 보호) | Developer/Designer | [ ] | |
