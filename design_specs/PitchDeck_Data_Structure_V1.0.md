# BDS 소상공인 플랫폼: 투자자 대상 피치 덱 데이터 구조 및 디자인 가이드라인 V1.0\n\n## 🎯 목표 요약 (CEO 지시사항 반영)\n본 문서는 시스템 안정성 검증 매트릭스(`System_Stability_UX_Verification_Matrix_V1.0.md`)를 기반으로, 기술적 성능(PRSR, ERT 등)을 **'소상공인의 심리적/경제적 통제감 회복'**이라는 비즈니스 스토리로 전환하는 방법을 정의합니다.\n\n## 📈 데이터 흐름 및 구조화 원칙\n1.  **기술 지표의 인간화:** 기술 용어(PRSR, ERT)를 직접 제시하지 않고, 이 수치가 의미하는 **사용자의 행동 변화나 감정 상태(예: 불안 $\rightarrow$ 안심)**로 번역하여 슬라이드에 배치해야 합니다.\n2.  **Before-After 대비 필수:** 모든 문제 해결 사례는 반드시 '문제 발생 상황 (좌측)'과 'BDS를 통한 해결 과정 및 결과 (우측)'의 명확한 분할 구조를 가져야 합니다.\n3.  **핵심 메시지 중심:** 슬라이드당 하나의 가장 강력한 스토리(One Core Message)만 전달합니다. 장황한 기능 나열은 금물입니다.\n\n## 🎨 디자인 및 시각화 요구사항 (Visual Requirements)\n*   **컬러 활용:** 신뢰/성장 계열 컬러를 중심으로 사용하며, 문제 상황을 설명하는 부분에서만 경고색(Red)을 제한적으로 사용하여 임팩트를 극대화합니다. \n*   **인포그래픽 우선:** 데이터는 숫자가 아닌 **흐름도(Flow)**나 **여정 지도(Journey Map)**로 변환되어야 합니다.\n\n## 📚 스토리보드 핵심 목차 (Slide Breakdown)\n1.  **(Problem) 고통의 깊이:** 소상공인의 불안정성 → '불확실성에 노출된 사장님' 이미지 사용.\n2.  **(Solution) BDS가 제공하는 가치:** 플랫폼을 단순 결제 시스템이 아닌, **'안전망(Safety Net)'**으로 포지셔닝.\n3.  **(Proof Point 1: 안정성 증명)** : (PRSR/ERT 활용) '실패도 관리 가능한 경험'. 오류 상황의 대응 과정을 애니메이션화하여 보여줍니다.\n4.  **(Proof Point 2: 통찰력 제공)** : AI 대시보드가 사장님에게 주는 **'선제적 행동 가이드'** 제시. (기술이 컨설팅 역할을 한다는 점 강조).\n5.  **(Vision) 미래와 비전:** 플랫폼 완성 이후, 소상공인 생태계에 가져올 파급 효과(KPIs) 제시.\n\n*작업 배분: 스토리텔링 구체화는 **Writer**에게 요청하고, 최종 발표 자료의 세부 목차 및 발표 스크립트 정리는 **영숙** 에이전트를 통해 진행합니다.*
