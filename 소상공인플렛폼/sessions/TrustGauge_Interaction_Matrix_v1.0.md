# 🛡️ Trust & Pain Gauge 통합 상태 및 상호작용 매트릭스 V1.0

## 📜 목적
본 문서는 BDS 플랫폼의 핵심 가치 지표(신뢰도, 고통점)를 시각화하는 Trust Widget과 PainGauge 컴포넌트가, 백엔드 API (V2.0 스키마 기반)로부터 어떤 데이터를 받든 일관되고 안전한 사용자 경험을 제공하도록 하기 위함입니다. 모든 상태별 시각적/기술적 요구사항 및 QA 체크리스트를 포함합니다.

## 📊 공통 컴포넌트 기준
*   **사용 색상 변수:**
    *   `--color-success`: `#3CB371` (Growth Green) - 목표 달성, 안정적 상태
    *   `--color-warning`: `#FFD700` (Amber Yellow/Light Orange) - 주의 필요, 모니터링 필요
    *   `--color-danger`: `#DC143C` (Crimson Red) - 위험 수준, 즉각적인 개입 필요
    *   `--state-background-default`: `#F7F9FA` (배경색)

## 🧭 컴포넌트별 상태 매핑 및 요구사항

### 1. Trust Widget (신뢰 지표)
**[데이터 스키마 연관 항목]:** `trust_score`, `data_completeness_ratio`

| 상태 | 데이터 조건 (V2.0 기반) | 시각적 표현/행동 정의 | 기술적 요구사항 및 QA Point |
| :--- | :--- | :--- | :--- |
| **🟢 성공 (Optimal)** | `trust_score` > 0.8, `data_completeness_ratio` > 90% | 배경: `--color-success`. 아이콘: 체크 마크 + '안정적'. 설명: "데이터가 충분하며 플랫폼 신뢰도가 높습니다." (최대 길이 명시) | **[QA]** 스코어 변화 애니메이션(Green $\rightarrow$ Green)이 부드럽게 작동하는가? |
| **🟡 주의 (Monitoring)** | 0.5 < `trust_score` $\le$ 0.8, 또는 데이터 부족 경고 발생 시 | 배경: `--color-warning`. 아이콘: 삼각형 + '모니터링 필요'. 설명: "데이터를 보완하면 더 높은 신뢰도를 확보할 수 있습니다." (구체적 행동 유도) | **[QA]** 스코어 변화 애니메이션(Green $\rightarrow$ Yellow) 시, 색상 전환 타이밍이 0.5초 이내인가? |
| **🔴 위험 (Critical)** | `trust_score` $\le$ 0.5 또는 핵심 데이터 누락 시 | 배경: `--color-danger`. 아이콘: 느낌표 + '개입 필요'. 설명: "신뢰도를 높이기 위한 즉각적인 조치가 필요합니다." (가장 강력한 Call to Action 유도) | **[QA]** 에러 상태 진입 시, 주변 컴포넌트의 UI 흐름을 차단하는지 확인. 경고 메시지의 중요도가 충분히 강조되는가? |
| **⚙️ 로딩/오류** | API 호출 전 또는 실패 시 (Timeout 등) | Skeleton Loader(`Loading...` 텍스트). 오류 시: 간결하고 친절한 에러 메시지 (`API 연결에 문제가 발생했습니다. 잠시 후 다시 시도해주세요.`). | **[QA]** 로딩 상태가 길어질 경우, 경과 시간(예: '데이터 불러오는 중...')을 표시하여 사용자 이탈 방지. 5초 이상 실패 시 자동 재시도 로직 구현 여부 확인. |

### 2. PainGauge (고통점 지표)
**[데이터 스키마 연관 항목]:** `pain_score` (0~1), `improvement_potential`

| 상태 | 데이터 조건 (V2.0 기반) | 시각적 표현/행동 정의 | 기술적 요구사항 및 QA Point |
| :--- | :--- | :--- | :--- |
| **🟢 개선됨 (Resolved)** | `pain_score` < 0.2, 또는 지난 측정 대비 현저히 하락했을 때 | 게이지 바: Green 채움(높음). 아이콘: '⬇️ 감소'. 설명: "최근 어려움이 줄어들었습니다." (성공적 경험 강조) | **[QA]** Gauge가 목표치에 도달하는 애니메이션(점진적 색상 변화)의 부드러움 확인. 과거 대비 개선폭 수치를 반드시 명시해야 함. |
| **🟡 모니터링 (Stagnant)** | 0.2 $\le$ `pain_score` < 0.6, 또는 변동 폭이 적을 때 | 게이지 바: Yellow 채움(중간). 아이콘: '⚠️ 주의'. 설명: "현재 상황은 안정적이나, 추가적인 노력이 필요합니다." (유지 및 관리 강조) | **[QA]** Gauge가 변화하지 않을 경우, 경고 메시지가 너무 부정적으로 느껴지지 않도록 톤앤매너를 유지해야 함. |
| **🔴 심화 위험 (Crisis)** | `pain_score` $\ge$ 0.6 또는 특정 위기 키워드(예: 폐업, 고금리) 포함 시 | 게이지 바: Red 채움(위험). 아이콘: '🚨 긴급'. 설명: "현재 어려움이 매우 심각합니다. 전문 컨설팅을 받아야 합니다." (가장 강력한 CTA 유도) | **[QA]** 위험 상태 진입 시, 플랫폼의 핵심 솔루션(AI 컨설턴트 등)으로 즉시 연결되는 버튼/위젯을 최상단에 노출해야 함. |
| **⚙️ 로딩/오류** | API 호출 전 또는 실패 시 | Skeleton Loader (Gauge 바가 점진적으로 채워지는 듯한 애니메이션). 오류 메시지: 간결하고 명확하게 표시. | **[QA]** 게이지의 값이 측정되는 순간, 단순히 숫자가 뜨는 것이 아니라 '측정 중'이라는 과정을 보여주는 시각적 장치가 필수적임. |

### 📚 최종 QA 및 다음 단계 정의
이 매트릭스는 이제 디자인 시스템과 API 명세를 연결하는 **최종 검증 문서**가 되었습니다. 이 문서를 바탕으로 개발팀은 컴포넌트 구현을, 저는 프로토타입의 애니메이션 디테일링에 집중할 것입니다.
