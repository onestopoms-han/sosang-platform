# 🎨 BDS 디자인 시스템 통합 가이드라인 v2.0: 오류 및 전환 경험 설계

**목적:** 플랫폼 전 영역(결제, 마케팅 콘텐츠, 온보딩)에서 'Pain $\rightarrow$ Solution'의 시각적/경험적 일관성을 확립하고, 모든 에러 상황을 기회로 재정의합니다.
_작성일: 2026-06-01 | 버전: 2.0 (최종 승인 전)_

## I. 핵심 디자인 토큰 및 컬러 시스템 재확인

| 요소 | 역할/메시지 | 컬러 코드 | 사용 범위 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Pain Red** | 위기, 오류, 부족함, 불안 (Initial State) | `#D94F3A` (따뜻한 적) | 에러 메시지, 경고 팝업 배경, 초기 콘텐츠 후킹. | 심리적 압박감 유도에 사용 (마케팅). |
| **Solution Green** | 해결책, 안정, 성장, 성공 (Final State) | `#4CAF50` (신뢰의 초록) | CTA 버튼, 성공 메시지 배경, 최종 스토리 전개. | 긍정적 행동을 강력히 유도 (기술/마케팅). |
| **Neutral Gray** | 정보 전달, 기본 레이아웃 | `#F5F5F5` | 본문 배경, 카드 구분선 등 안정적인 바탕색. | 가독성을 극대화합니다. |

## II. 🚨 컴포넌트 통합 설계: Error Card Component v2.0 (가장 중요)

단순 오류 메시지 표시를 넘어 **'Pain 진단 $\rightarrow$ Solution 제시 $\rightarrow$ 재도전 유도'** 의 3단계 경험을 제공해야 합니다.

| 단계 | 목표/메시지 | 시각적 적용 (CSS/UI) | 코다리(개발) 반영 요구사항 |
| :--- | :--- | :--- | :--- |
| **1. Pain 진단** (오류 발생) | "무엇이 문제인가요?" 사용자에게 실패 원인에 대한 이해를 돕는다. | 배경: `Pain Red`의 낮은 채도 버전 (`#E8C4C0`). 아이콘: 경고(⚠️). 메시지: **"결제 처리에 오류가 발생했습니다."** (구체적 에러 코드 제시). | *필수*: 실패 원인에 대한 일반화된 설명을 제공해야 함. (e.g., "카드사 정책 변경으로 인한 일시적 오류일 수 있습니다.") |
| **2. Solution 전환** (해결책 안내) | "어떻게 해결할 수 있나요?" 기술적 해결책을 제시하여 무력감을 해소한다. | 배경: 중립적인 색상 (`Neutral Gray`와 `Solution Green`의 경계). 메시지: **"다음 시도를 위해 다음 옵션을 확인해주세요."** (해결 가이드 제시). | *필수*: 실패 원인과 함께, 재시도할 수 있는 구체적 행동(예: '다른 결제 수단 사용', '일정 시간 후 재시도')을 텍스트로 노출. |
| **3. Action 유도** (재도전) | "무엇을 해야 하나요?" 명확한 다음 단계와 CTA를 제공한다. | 버튼: `Solution Green` 배경, 흰색 글씨. 메시지: **[🔄 재시도하기]** 또는 **[💡 다른 결제 수단으로 전환]**. | *필수*: 재시도 시의 로직(자동 리트라이/팝업) 명확화 및 CTA 활성화 상태 구현. |

## III. 📸 마케팅 콘텐츠 일관성 점검 (Instagram / YouTube 등)

| 콘텐츠 유형 | 핵심 메시지 구조 | Pain $\rightarrow$ Solution 연결점 | 디자인 에셋 적용 원칙 |
| :--- | :--- | :--- | :--- |
| **릴스/피드** | 불안(Pain) 자극 $\rightarrow$ 플랫폼 가치 발견(Solution). | "매출이 줄어요" (Pain Red) → "AI로 분석해서 성장할 수 있어요!" (Solution Green). | 텍스트 오버레이 시, Pain 영역은 `Pain Red` 계열의 다크톤 사용. Solution 전환점에서는 밝고 활력 있는 `Solution Green`을 압도적으로 사용하여 감정적 반전을 극대화. |
| **웹사이트** | 문제 정의 $\rightarrow$ 기능 설명 $\rightarrow$ 등록 유도. | (UX 흐름) 가입 버튼 직전까지는 Pain Red 계열의 톤으로 긴장감 유지 → '가입 완료' 페이지에서 폭발적인 Solution Green을 사용하여 안도감 부여. | 모든 핵심 CTA(Call To Action)는 `Solution Green` 토큰만을 사용하고, 다른 어떤 요소와도 색상이 혼합되지 않도록 합니다. |
