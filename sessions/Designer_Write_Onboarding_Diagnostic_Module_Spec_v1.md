# 🛠️ Onboarding/Pre-Diagnostic Module 사양서 (V1.0)\n\n**[작성 목적]** 소상공인의 Pain Point를 극대화하여, BDS 플랫폼의 필요성을 인지시키는 첫 진입점 콘텐츠 설계.\n**[핵심 원칙]** Pain $\rightarrow$ 공감/진단 $\rightarrow$ Solution 기대감 증폭 (스크롤에 따른 감정적 흐름 변화).\n\n## 🎨 디자인 사양 및 구조 (Designer Scope)\n\n### 1. 전체 레이아웃 컨셉: '공감형 대화 인터페이스' (Conversational UI)\n*   **톤:** 신뢰감, 따뜻함, 전문성.\n*   **컬러 사용:** 초기 스크롤 섹션은 **경고색(Warning Red/Orange 계열)**의 뉘앙스를 사용하여 불안감을 자극하고, 진단 질문이 쌓일수록 점진적으로 BDS의 메인 컬러인 **안정감 있는 블루(Primary Blue)**로 전환되어야 함.\n*   **핵심 컴포넌트:** 대화창 형태의 UI (User Query / System Response)\n\n### 2. 섹션별 상세 와이어프레임 및 인터랙션 정의\n\n#### A. [Hero Section] - 시선 강탈 및 공감 유도 (The Hook)\n*   **배경:** 빈티지한, 하지만 세련된 '직장인/사장님의 하루' 느낌의 이미지와 텍스트 조합.\n*   **Headline (Writer 협업):** "혹시 이런 고민으로 밤잠 설치고 계신가요? 😥" (질문형)\n*   **CTA:** 버튼 대신, **[나의 어려움 진단하기]** 라는 부드러운 시작 유도 버튼.\n*   **인터랙션:** 스크롤 시, 배경 이미지가 흐릿하게 처리되며 '진단의 문'으로 들어가는 느낌을 줌 (Depth/Focus 효과).\n\n#### B. [Pain Recognition Section] - Pain Point 제시 및 질문 루프 (The Diagnosis)\n*   **구성:** 사용자가 직접 상호작용하는 Q&A 형태의 카드 컴포넌트.\n*   **요소:** '질문 팝업/카드' + '선택지 버튼(Multiple Choice)'\n*   **진행 방식:** 질문을 던지고, 선택할 때마다 다음 질문으로 넘어가며 *이전 답변 내용을 요약하여* 화면에 노출시킴. (ex: "사장님은 **인력난**도 고민하시고, **마케팅 비용** 문제도 있으시군요.")\n*   **기술 지침:** 각 선택지 클릭 시, 해당 Pain 키워드가 데이터로 수집되는 로직이 필요함.\n\n#### C. [System Bridge Section] - 해결책 기대감 증폭 (The Turn)\n*   **배경 변화:** 경고색 뉘앙스 $\rightarrow$ **Primary Blue** 계열의 안정적인 배경으로 급격히 전환.\n*   **내용:** "당신의 어려움, BDS가 구조적으로 해결합니다."라는 확신을 주는 메시지. \n*   **시각화:** 지금까지 사용자가 진단한 Pain 키워드 3~4개가 모여서 하나의 **'시스템 대시보드(Dashboard)'** 형태의 그래픽으로 재구성되어야 함 (KPI Gauge와 연관성 확보).\n\n## ✍️ 카피라이팅 및 메시지 흐름 가이드 (Writer Scope)\n\n*   **톤앤매너:** 비난이나 해결책을 제시하기보다, **'경청하고 공감하는 컨설턴트'**의 목소리여야 함.\n*   **핵심 목표:** 사용자가 스스로 '이 문제를 시스템적인 방법으로 풀어야겠다'는 절박함을 느끼게 하는 것. (Emotionally resonant).\n\n---
**[참고] 이 디자인 사양서는 코다리가 구현할 백엔드 진단 로직과 연동되어, 단순한 랜딩 페이지가 아닌 상호작용하는 '진단 툴'로 기능해야 합니다.**
