# 💎 BDS 플랫폼 통합 시스템 디자인 명세서: 신뢰 구축 및 오류 처리 (V1.0)

## 🎯 목표
플랫폼 이용 과정에서 발생 가능한 모든 실패 시나리오(결제, 인증, API 통신 등)에 대해 일관되고 사용자 친화적인 경험을 제공하고, 이를 통해 플랫폼의 궁극적인 가치인 '신뢰'를 강화한다. 단순한 오류 메시지가 아닌, **문제 해결의 시작점**으로 설계한다.

## 🎨 디자인 시스템 통합 원칙
1.  **톤(Tone):** 경고 (Warning) $\rightarrow$ 안심/지침 (Reassurance & Guidance).
2.  **컬러 코드:**
    *   **Primary Trust Color (성공/안정):** Deep Blue (#004D66) - 신뢰와 전문성을 유지한다.
    *   **Failure Warning Color (주의/오류):** Muted Orange (#FF9800) 또는 Soft Red (#E53935). *직관적이지만 공포감을 주지 않는 톤을 사용.*
    *   **Guidance Color (행동 유도):** Growth Green (#3CB371) - 다음 액션을 안내하며 희망을 준다.
3.  **타이포그래피:** 실패 메시지의 헤드라인은 명확하고 간결한 산세리프 폰트를 사용하며, 핵심 정보는 **굵게(Bold)** 강조한다.

## 🖼️ 섹션 1: 결제 실패 화면 (Payment Failure Screen) 디자인 스펙
*(트리거: 결제 API 오류 또는 카드사 인증 실패)*

### A. UI/UX 레이아웃 구성
*   **제목:** [Warning] 결제가 완료되지 않았습니다.
*   **메인 메시지 (Pain Point):** "결제 정보가 일치하지 않거나, 시스템에 일시적인 문제가 발생했습니다."
*   **문제 해결 안내 (Solution Focus):** 실패 원인을 추정하여 2~3가지의 구체적 행동 옵션을 제시한다.
    1.  (Most Likely) 카드사/은행 앱을 통해 결제 가능 여부를 먼저 확인해주세요.
    2.  (Technical) 네트워크 연결 상태를 점검하고 다시 시도해 주세요.
    3.  (Alternative) 다른 결제 수단을 이용하시거나, [고객센터 문의] 버튼을 눌러 도움을 받으세요.

### B. 핵심 컴포넌트 명세
*   **재시도 CTA:** 가장 크고 눈에 띄는 Growth Green 버튼으로 배치하며, "다시 시도하기" 문구를 사용한다. (클릭 시 API 재연결 로직 발동)
*   **정보 입력 필드:** 실패 사유와 무관하게 모든 결제 정보 입력 필드는 명확히 보이도록 유지하여 사용자 혼란을 최소화한다.

## 🔒 섹션 2: 인증 및 보안 오류 화면 (Authentication Failure Screen) 디자인 스펙
*(트리거: 비밀번호 불일치, 이메일 미존재 등)*

### A. UI/UX 레이아웃 구성
*   **제목:** [Warning] 인증에 실패했습니다.
*   **핵심 메시지:** "아이디 또는 비밀번호가 일치하지 않습니다. 보안을 위해 2회 이상 오류 발생 시, 잠시 후 다시 시도해 주세요." (추측 불가 원칙 명시)
*   **보조 안내:** '비밀번호 재설정' 기능을 가장 눈에 잘 띄게 배치한다.
    *   [Forgot Password] 링크를 명확하게 제공하며, 클릭 시 인증 절차 페이지로 이동하는 흐름을 유지한다.

### B. 실패 로직 연동 (코다리 API 기반)
만약 코다리가 정의한 `AuthService`에서 특정 에러 코드(예: `AUTH_USER_NOT_FOUND`)가 반환될 경우, 일반적인 '오류' 대신 해당 에러에 특화된 친절한 메시지를 제공해야 한다.

*   **[Error Code 401 - User Not Found]:** "존재하지 않는 아이디입니다. 회원 가입을 먼저 진행해주세요." (직접적이고 명확하게 안내)
*   **[Error Code 429 - Too Many Requests]:** "너무 자주 시도하셨습니다. 잠시 후(5분 뒤) 다시 접속해 주세요." (시간 기반의 구체적인 지침 제공)

## 📡 섹션 3: 일반 API 통신 오류 화면 (General API Error Screen)
*(트리거: 백엔드 시스템 문제, 네트워크 끊김 등)*

### A. UI/UX 레이아웃 구성
*   **제목:** [Attention] 연결 상태를 확인해주세요.
*   **메시지:** "현재 BDS 서버와 통신이 원활하지 않습니다. 저희가 문제를 해결할 수 있도록 잠시만 기다려 주시거나, 새로고침을 눌러주세요." (책임감 있는 태도 유지)
*   **Action Plan:** '새로고침' 버튼과 함께, 만약 지속될 경우를 대비해 **[상태 확인 페이지]** 링크를 제공하여 사용자 불안감을 해소한다.
