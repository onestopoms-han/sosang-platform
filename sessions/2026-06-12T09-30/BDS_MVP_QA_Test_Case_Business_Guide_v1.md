# BDS 플랫폼 MVP: QA 테스트 비즈니스 가이드 초안

## 1. 개요 및 성공 기준 (Definition of Done)
MVP 의 핵심 기능인 "대출 비교"와 "AI 상담"은 단순한 기능 구현을 넘어, 소상공인의 실질적인 금융 니즈를 해결하는 **신뢰 기반의 도구**여야 합니다. 따라서 QA 테스트는 **기능적 정확성 (Functional Accuracy)** 과 **감성적 안전마진 (Emotional Safety Margin)** 두 축으로 설계됩니다.

**🏆 MVP 성공 기준 (Definition of Done, DoD):**
- [ ] **사용자 만족도 (CSAT) ≥ 4.5 점 (5 점 만점)**: 첫 10 명 베타 사용자의 피드백을 기반으로 측정.
- [ ] **오류율 < 1%**: AI 상담 시 금리, 이자 계산, 대출 한도 산출 등 핵심 데이터가 99% 이상 정확.
- [ ] **반응 시간 ≤ 2 초**: AI 상담 응답 및 대출 비교 결과 로딩이 평균 2 초 이내 완료.
- [ ] **안전마진 달성**: 사용자가 금융 정보 제공 중 불안감을 느끼지 않고, '안전을 보장한다'는 메시지가 체감됨 (PainGauge 지표 < 0.3).

## 2. 핵심 기능별 QA 테스트 방향 (Business-Oriented Test Scenarios)

### A. 대출 비교 기능 (Loan Comparison Engine)
**테스트 목표:** 사용자가 여러 은행/대리점의 대출 정보를 한눈에 비교하고, 자신의 상황 (신용등급, 담보 등) 에 맞는 최적의 대출을 추천받을 수 있어야 함.

| 테스트 시나리오 | 성공 기준 (Pass Criteria) | 비즈니스 영향도 |
| :--- | :--- | :--- |
| **시나리오 1: 다양한 조건 입력**<br>사용자가 신용등급, 담보유무, 목적 등을 정확히 입력하고, AI 가 맞춤형 대출 상품을 추천함. | - 추천된 상품이 실제 금융기관의 공식 정보와 95% 이상 일치.<br>- 추천된 상품의 이자율과 원금 상환 조건이 명확히 표기됨. | **신뢰도 향상**: 사용자가 플랫폼을 통해 정확한 정보를 얻을 수 있어 신뢰 형성. |
| **시나리오 2: 비교 결과 정확성**<br>여러 은행의 대출 정보를 동시에 표시하고, 사용자의 조건에 따라 최적의 결과를 제공함. | - 모든 비교 대상의 금리, 상환 기간, 초기 비용이 정확히 반영됨.<br>- 사용자의 신용등급 변화에 따라 추천 결과가 실시간으로 업데이트됨. | **전환율 증가**: 사용자가 원하는 대출을 쉽게 찾을 수 있어 플랫폼 이용 빈도 증가. |
| **시나리오 3: 안전한 정보 제공**<br>사용자가 금융 정보를 입력할 때, 개인정보 유출 우려 없이 안전하게 처리됨. | - 사용자의 개인정보가 암호화되어 저장되고, 권한 없는 접근이 차단됨.<br>- AI 가 불필요한 질문을 최소화하여 사용자 피로감 감소. | **안전마진 확보**: 사용자가 플랫폼의 안전성을 신뢰하고 정보를 공유함. |

### B. AI 상담 기능 (AI Consulting Assistant)
**테스트 목표:** 소상공인이 금융, 마케팅, 운영 등 다양한 주제를 물어볼 때, AI 가 정확하고 유용한 답변을 제공하며, 사용자에게 '안전을 보장한다'는 느낌을 심어줌.

| 테스트 시나리오 | 성공 기준 (Pass Criteria) | 비즈니스 영향도 |
| :--- | :--- | :--- |
| **시나리오 1: 금융 관련 질문 처리**<br>사용자가 "대출 금리는 어떻게 되나요?"와 같은 금융 관련 질문을 하면, AI 가 정확한 정보를 제공함. | - AI 가 제공한 정보의 정확도가 95% 이상.<br>- 답변이 사용자의 신용등급이나 상황에 맞춰 맞춤화됨. | **신뢰도 강화**: 사용자가 AI 를 통해 신뢰할 수 있는 금융 정보를 얻을 수 있음. |
| **시나리오 2: 비상상황 대응**<br>사용자가 "은행 문을 닫았어요, 대출 신청은 어떻게 하나요?"와 같이 긴급한 질문을 하면, AI 가 즉각적이고 구체적인 해결책을 제시함. | - AI 가 은행의 공식 웹사이트 정보를 빠르게 참조하고, 사용자에게 가장 적합한 대안을 제안.<br>- 답변이 불필요한 불안감을 유발하지 않고, '안전을 보장한다'는 메시지가 체감됨. | **안전마진 확보**: 사용자가 긴급 상황에 대비할 수 있어 플랫폼에 대한 의존도 증가. |
| **시나리오 3: 감정적 안전마진**<br>사용자가 "대출을 못 받을 것 같아요"와 같이 불안한 감정을 표현하면, AI 가 공감과 위로의 메시지를 제공함. | - AI 가 사용자의 감정을 인지하고, 적절한 위로나 정보를 제공.<br>- 답변이 사용자가 플랫폼을 신뢰하게 하는 '안전'을 심어줌. | **고객 충성도**: 사용자가 플랫폼을 친구처럼 느끼고 장기적으로 이용함. |

## 3. 개발팀 (코다리) 에게 전달할 것
1. **테스트 데이터셋 준비:** 실제 금융기관의 공식 데이터를 기반으로 한 테스트 데이터셋을 제공.
2. **API 연동 확인:** AI 상담 및 대출 비교 기능을 위한 API 연동이 정상 작동하는지 확인.
3. **UI/UX 검토:** 사용자가 입력하고 결과를 확인하는 과정에서 불편함이 없는지 UI/UX 측면에서 검토.
