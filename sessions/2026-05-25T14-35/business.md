| 옵션 | 가격 정책 | 초기 CAPEX | OPEX/월 | 예상 전환율 | MVP 적합도 |

## 🔧 개발 모듈 구현 계획 (코다리 전달용)
### 1️⃣ 우선순위 1: 결제 및 구독 연동
- **기술 스택**: PayPal REST API / Stripe SDK (기존 도구 실패 시 대체)
- **MVP 기능**: 
  - 사용자 등록 시 이메일 인증 + 결제 정보 수집
  - 월정액 구독 자동 갱신/취소 로직
  - 대시보드: 구독 상태 실시간 표시
- **개발 시간**: 2~3 일 (Figma 와 연결된 UI 목업 참조 필요)

### 2️⃣ 우선순위 2: 초기 대시보드 UI
- **핵심 컴포넌트**: 
  - `Revenue Summary` (월/년 수익, 전환율 추적)
  - `User Growth Chart` (MAU 추이, 목표 대비 달성률)
  - `AI Risk Alert` (실시간 경고 메시지)
- **기술 스택**: React + Chart.js / D3.js

### 3️⃣ 우선순위 3: 데이터 수집 및 분석 파이프라인
- **데이터 흐름**: 
  - 결제 이벤트 → PostgreSQL/Redis 캐시
  - 대시보드 조회 → GraphQL API
  - 로그 분석 → ELK 스택 (비용 절감을 위해 무료 티어 활용)
- **보안 요구사항**: PCI DSS 준수를 위한 결제 정보 암호화
