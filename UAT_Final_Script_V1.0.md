## 📋 1. 테스트 준비 및 앱 최초 실행 방법

> [!IMPORTANT]
> **첫 실행 시 보안 경고 대처 방법 (코드 서명 전)**
> 배포 파일인 **`Connect-AI-0.1.0-win-x64-Setup.exe`** 실행 시, 아직 코드 서명(Code Signing) 완료 전이므로 최초 실행 시 OS 보안 경고가 발생할 수 있습니다. 
> 
> *   **윈도우 (Windows) 환경:**
>     *   경고창에서 **[추가 정보]** 클릭 ➡️ 하단의 **[실행]** 버튼 클릭
> *   **맥 (macOS) 환경:**
>     *   경고창 발생 시 확인 클릭 ➡️ **[시스템 설정] → [개인정보 보호 및 보안]** 이동 ➡️ 하단의 **[확인 없이 열기]** 클릭

## 👥 2. UAT 테스트 시나리오 명세

### 시나리오 1: 회원가입 및 로그인 플로우 (UAT-FL-01)
*   **목적:** 사용자가 정상적으로 계정을 생성하고 로그인하여 메인 플랫폼에 접근할 수 있는지 확인합니다.
*   **테스트 절차:**
    1. 앱을 실행하고 우측 상단의 **[회원가입]** 버튼을 클릭합니다.
    2. 필수 항목(이메일, 비밀번호, 소상공인 사업자 등록번호)을 입력합니다.
    3. 약관 동의 후 가입을 완료하고 로그인 화면으로 이동합니다.
    4. 가입한 정보로 로그인을 시도합니다.
*   **기대 결과:**
    *   정확한 사업자 정보 입력 시 즉시 회원가입이 완료됩니다.
    *   로그인 후 첫 대시보드 화면으로 매끄럽게 진입하며, 사용자명이 화면에 정상 표시됩니다.

### 시나리오 2: 대시보드 데이터 연동 및 지표 시각화 (UAT-DB-02)
*   **목적:** 백엔드에서 제공되는 실시간 비즈니스 데이터 및 핵심 신뢰 지표가 대시보드에 정확하게 연동되는지 검증합니다.
*   **테스트 절차:**
    1. 로그인 후 메인 대시보드 화면에 접속합니다.
    2. 상단 핵심 지표 영역의 '거래 성공률'과 '자동 복구율'을 확인합니다.
    3. 지난달 평균 데이터와 비교하여 수치가 오차 없이 연동되는지 모니터링합니다.
*   **기대 결과:**
    *   **거래 성공률 (Success Rate):** **95%** 수준의 수치가 경제적 비용 절감 가치 헤드라인과 함께 표시됩니다.
    *   **자동 복구율 (Retry Rate):** **80%** 수치와 함께 복구 프로세스 상태가 시각적으로 노출됩니다.
    *   그래프 및 카드가 Deep Blue (`#004D66`) 브랜드 컬러로 조화롭게 디자인되어 신뢰감을 줍니다.

### 시나리오 3: 결제 모듈 및 복구 플로우 (UAT-PAY-03)
*   **목적:** 결제 실패 상황에서 사용자가 불안감을 느끼지 않고, 대안을 선택하여 문제를 해결할 수 있도록 돕는 감성적 복구 플로우를 검증합니다.
*   **테스트 절차:**
    1. 플랫폼 내 유료 서비스 결제를 시작합니다.
    2. 테스트 결제 창에서 일부러 '한도 초과' 혹은 '일시적 통신 오류'를 발생시켜 실패 상황을 만듭니다.
    3. 로드되는 결제 실패 화면(Payment Failure Screen)의 디자인과 안내 문구를 검토합니다.
    4. 제시되는 3가지 옵션을 하나씩 클릭해 봅니다.
*   **기대 결과:**
    *   결제 실패 화면에 불안감을 조성하는 **붉은색(Red 계열) 색상이 전혀 사용되지 않아야 합니다.**
    *   Deep Blue 배경과 부드러운 Warning Orange (`#FF9800`) 조합의 프레임이 적용됩니다.
    *   **3가지 대안 버튼**이 정상 작동해야 합니다:
        1. **자동 재시도 (가장 권장 - 복구율 80% 강조)**
        2. **다른 결제 수단 사용**
        3. **고객센터 문의 (Deep Blue 강조)**
