# 🌟 BDS소상공인플렛폼 최종 QA 성공 기준 체크리스트 (V1.0)

## 🎯 목표: '안전마진' 메시지 전달 및 사용자 심리 흐름 검증
**최종 목적:** 소상공인이 위기(Pain)를 인지하고, 플랫폼을 통해 실질적인 해결책과 희망('안전마진')을 느끼도록 유도하는가?

---

## 🛠️ I. 기술 기능 (Functionality Check - 코다리/디자인팀 검증)
| 항목 | 테스트 내용 | 합격 기준 (Pass Criteria) | 비고 |
| :--- | :--- | :--- | :--- |
| **1. 상태 전환 로직** | PainLevel(0-100) 변화에 따른 컴포넌트의 색상/라벨 변경이 API 스키마와 일치하는가? (A $\to$ B $\to$ C 순서) | `constants.ts` 기준 3가지 리스크 등급(Green, Yellow, Red)과 라벨이 정확히 매핑되어야 함. | **[필수]** |
| **2. 사용자 입력 처리** | 사용자가 수동으로 값을 조작하거나 잘못된 데이터를 넣었을 때의 예외 처리(Error Handling). | 시스템 오류 메시지(`400 Bad Request` 등)가 친절하고 명확하며, 해결 방법을 제시해야 함. | 💡 에러 경험 최소화 |
| **3. 데이터 일관성** | `PainGauge`와 `TrustWidget`에 표시되는 핵심 지표(숫자/등급)의 값이 다른 곳에서 상충되지 않는가? | 모든 위젯이 동일한 백엔드 데이터를 기반으로 계산 및 표시해야 함. | |

## 💖 II. 비즈니스 경험 (Emotional & UX Check - CEO/기획팀 검증)
| 항목 | 테스트 내용 | 합격 기준 (Pass Criteria) | 비고 |
| :--- | :--- | :--- | :--- |
| **1. 위기 인식 단계 (Pain Gauge)** | 사용자가 '위험'을 느끼는 순간, 시각적 자극(색상/애니메이션 등)이 적절하게 작동하는가? | Red 등급에서 발생하는 *시각적 긴장감*과 그에 대한 공감이 느껴져야 함. 단순 경고를 넘어선 **위기감** 전달이 핵심. | 🚨 감정 이입 유도 |
| **2. 해결책 제시 단계 (Trust Widget)** | 플랫폼의 솔루션(컨설팅, 자금 등)을 접했을 때, 사용자가 '안전하다'는 심리적 안정감을 느끼는가? | 추상적인 문구가 아닌, 구체적인 행동 계획과 수치로 **실질적 안전마진**이 느껴져야 함. (예: "월 매출 15% 상승 예상") | ✅ 희망 제시 |
| **3. 플로우의 완성도** | Pain Gauge $\to$ 진단 결과 확인 $\to$ Trust Widget(해결책)으로 이어지는 전체 흐름에 논리적/감정적 비약이 없는가? | '위기 인지'에서 '해결 가능성 인지'로 자연스럽게 전환되어야 함. (사례: "지금 위험하지만, 이렇게 하면 돼!") | 🚀 플로우 매끄러움 |
