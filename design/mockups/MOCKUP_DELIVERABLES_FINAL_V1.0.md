# 📊 BDS 소상공인 플랫폼: 최종 목업 디자인 자산 (V1.0)

## 💡 컨셉 요약: 데이터 기반 신뢰 확보
*   **핵심 메시지:** "막연한 희망이 아닌, 증명된 안정성으로 성장합니다."
*   **디자인 시스템 적용:** Deep Blue (#004D66) - 전문성/신뢰, Growth Green (#3CB371) - 성장/긍정, Warning Orange - 위험 경고 및 주목 유도.
*   **최대 강조 요소 (USP):** Trust Widget (기술 안정성 지표).

### 🔷 [Pitch Deck Mockup] 슬라이드 구성 상세 디자인 명세서

#### 📄 Slide 3: 문제 제기 & 전환 (The Data Shock)
**(목표: 고객의 막연한 불안감을 구체적인 '데이터 위험'으로 인식시키기)**

*   **레이아웃:** 좌측 70% (문제점 나열), 우측 30% (해결책 도입 예고).
*   **Trust Widget 통합 방식:**
    *   슬라이드 하단 중앙에 전체 폭을 사용하는 대형 위젯 배치.
    *   **제목:** "현재 소상공인 시장의 데이터 리스크 분석"
    *   **시각화 내용:** 성공률(Success Rate)과 장애 빈도(Failure Frequency)를 그래프가 아닌, **'실시간 모니터링 게이지' 형태**로 구현. (예: Success Rate 95% $\rightarrow$ 'Good', 80% $\rightarrow$ 'Warning').
    *   **카피라이팅:** "단순히 높은 성공률을 제시하는 곳이 아니라, **위험 발생 시 복구 능력(Retry Rate)**까지 증명해야 합니다." (→ 다음 슬라이드로 자연스럽게 연결)

#### 📄 Slide 7: 핵심 가치 & CTA 전 단계
**(목표: 서비스가 '필수적인 방어막'임을 각인시키기)**

*   **레이아웃:** 신뢰성 지표를 중심으로 하는 대시보드 형태의 섹션.
*   **Trust Widget 통합 방식:**
    *   **제목:** "BDS 플랫폼만의 차별점: 기술적 안정성의 증명"
    *   **구조:** 3개의 카드로 구성 (데이터 확보 $\rightarrow$ 분석 컨설팅 $\rightarrow$ 위험 관리 솔루션).
    *   가장 큰 카드에 Trust Widget을 배치하고, 이 위젯이 **'우리가 제공하는 서비스의 기반'**임을 강조.
    *   **UX 흐름:** 사용자가 이 지표를 보고 안심하면, 다음 단계로 '컨설팅 신청 (Growth Green CTA)'으로 시선을 유도.

### 🌐 [Landing Page Mockup] 섹션별 구현 상세 명세서

#### 🚀 Hero Section (최상단)
**(목표: 스크롤을 내리기 전에 신뢰를 확보하고 이탈 방지)**

*   **핵심 배치:** 메인 헤드라인 바로 아래, Sub-CTA와 함께 Trust Widget을 배치합니다.
*   **Trust Widget 구현:** 크고 간결한 3개의 숫자 표시기 (Digital Display Style).
    *   `Success Rate: 95%` (Deep Blue 배경)
    *   `Avg. Recovery Time: <10분` (Growth Green 강조)
    *   `Monthly Uptime: 99.99%` (텍스트로 신뢰성 보강)
*   **CTA 연계:** "기술적 위험성을 관리하는 컨설팅을 받아보세요." 버튼과 직접 연결되어, **데이터 $\rightarrow$ 문제 인식 $\rightarrow$ 솔루션 탐색**의 플로우를 즉시 완성합니다.

#### 📈 ROI 시뮬레이션 섹션 (Conversion Point)
**(목표: 지식 기반 판매(Know-how Selling)로 전환)**

*   **개선된 UI:** 기존의 단순 결과창이 아닌, **"RISK GAP 분석기"**라는 명칭을 부여합니다.
*   **Trust Widget 연계:** 사용자가 가상의 '낮은 안정성' 데이터를 입력했을 때, 시스템이 자동으로 Trust Widget 기반의 시뮬레이션 결과를 보여줍니다.
    *   **결과 메시지 예시:** "현재 플랫폼의 불안정성은 월평균 **X%의 잠재적 손실 리스크(Deep Blue)**를 발생시키고 있으며, 이를 해결하기 위해 컨설팅 서비스가 필요합니다." (→ 명확한 Pain Point 제시)
