# BDS 플랫폼 애니메이션 & 인터랙션 최종 스펙 명세서 (V1.1)
## 🎯 목표: 신뢰도 변화(Red $\to$ Yellow $\to$ Green)의 시각적 계층화 및 기술 표준 정의

### 1. 전반적인 디자인 원칙 (Visual Philosophy)
*   **톤앤매너:** 차분함, 데이터 기반의 객관성, 점진적 개선 (Progressive Improvement).
*   **애니메이션 스타일:** 부드러운 이징(Easing), 과도한 움직임 배제. '깜빡거림' 방지 및 상태 변화의 명확성이 최우선.
*   **핵심 메커니즘:** 값 변화 시, 이전 상태에서 다음 상태로 **점진적으로 전환**되는 과정 자체를 애니메이션화하여 '안전마진 확보 과정'을 스토리텔링한다.

### 2. 핵심 컴포넌트별 스펙 (Component Specification)

#### A. PainGauge / Trust Widget (플랫폼 UI 컴포넌트)
| 상태 | 값 범위 | 색상 코드(Hex) | 애니메이션/변화 디테일 | 기술 명세 (Dev Notes) |
| :--- | :--- | :--- | :--- | :--- |
| **Red (위험)** | 0.0 ~ 0.3 | `#D9534F` (경고 레드) | - 배경색과 게이지 바가 빨간색으로 급격히 채워짐.<br>- 게이지 수치가 낮을수록 '떨림(Jitter)' 효과를 미세하게 적용하여 위기감을 시각화. | `CSS Transition: 0.5s ease-out` 사용. 값 변화에 따라 게이지 폭이 비례적으로 증가/감소해야 함. |
| **Yellow (주의)** | 0.3 ~ 0.7 | `#F0AD4E` (경고 옐로우) | - Red에서 Yellow로 전환 시, 색상이 부드럽게 블렌딩(Color Blending).<br>- 게이지 수치가 안정화되는 느낌을 주기 위해 움직임이 다소 완만해짐. | `CSS Transition: 1s cubic-bezier(0.42, 0, .58, 1)` (EaseInOut) 사용. 시각적 속도 저하 강조. |
| **Green (안정)** | 0.7 ~ 1.0 | `#5CB85C` (성장 그린) | - 배경색과 게이지 바가 녹색으로 완전히 채워지며, 안정감을 주는 부드러운 광원(Glow) 효과 적용.<br>- 수치 변화 시 '점진적 상승'의 느낌을 주기 위해 미세한 오버슈트(Overshoot) 애니메이션 추가. | `CSS Transition: 1s cubic-bezier(0.68, -0.55, 0.265, 1.55)` (Bounce/Spring 효과). 최소 변동률 유지를 통해 '안정성' 강조. |

| Scene | 내용 | 시각화 요소 | 전환 방식 / 타이밍 | 참고 지침 (Tone & Manner) |

| **Intro** | 소상공인의 불안감 제시 | 텍스트 오버레이 + Red Gradient 필터 | 느린 페이드 인(Fade In). 배경 음악은 낮고 긴장감 있는 사운드 사용. | 감성적 공감 유도. 시청자가 '우리 이야기'라고 느끼게 할 것. |
| **PainGauge 소개** | 문제점 정의 및 측정 도구 제시 | PainGauge 컴포넌트 (Red $\to$ Yellow) | 데이터가 빨간색부터 점진적으로 노란색으로 '정화(Purification)' 되는 애니메이션을 보여줌. | 수동적 불안감 → 능동적 인지 과정의 전환 시각화. **시간차(Delay)**를 활용하여 중요성을 강조할 것. |
| **Solution (안전마진)** | BDS 솔루션 제시 및 가치 설명 | Green 영역 확장 + Growth Curve 그래프 | 게이지가 Yellow $\to$ Green으로 매우 부드럽게, 하지만 확신에 차서 상승하는 모습을 보여줌. | 가장 중요한 순간이므로, 애니메이션 속도를 늦추고 (Slow Motion), 시청자가 수치를 따라갈 시간을 충분히 부여해야 함. |
| **CTA/End** | 행동 촉구 및 Call to Action | 최종 Green 상태의 인터페이스 Mockup | 모든 요소가 부드럽게 제자리로 돌아오며(Settle In), 긍정적인 '완성' 느낌을 주는 명확한 CTA 버튼이 등장하며 영상 종료. | 결론은 반드시 밝고 확신에 찬 톤으로 마무리. |

### 3. 디자인 시스템 적용 및 검증 목록 (Checklist)
*   [ ] 모든 애니메이션은 `BDS_Primary_Font`의 가독성을 최우선으로 고려해야 한다.
*   [ ] Red $\to$ Yellow $\to$ Green의 색상 전환 비율(Ratio)과 시간(Time)은 톤앤매너를 유지하며 일관되어야 한다. (예: Red-Yellow는 긴장감 유발에 초점을 맞추고, Yellow-Green은 안도감/확신에 중점을 둠).
*   [ ] 모든 인터랙션 지점(Hover, Click)에서 애니메이션이 끊기지 않도록 `CSS Transition`을 전역적으로 적용한다.
