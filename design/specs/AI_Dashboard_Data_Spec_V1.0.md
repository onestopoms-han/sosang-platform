# 🛡️ AI 리스크 예측 대시보드 - 최종 데이터 요구사항 정의서 (V1.0)

**목표:** 소상공인이 직면한 잠재적 위험(AI 리스크)을 정량화하고, BDS 플랫폼의 해결책에 대한 경제적 가치(WTP)를 제시하여 행동 변화를 유도한다.
**핵심 측정 지표 (KPI):** Max Loss (최대 손실액), WTP (지불 의사), ROI (투자 대비 수익률).

## 1. 핵심 데이터 시나리오 정의: '위협'에서 '가치'로의 전환

| 단계 | 개념 (사용자 경험) | 측정 지표 | 설명 및 스토리텔링 원칙 |
| :--- | :--- | :--- | :--- |
| **[Phase 1] 위험 인식** | "내가 만약 AI 리스크에 노출되면 얼마나 손해일까?" | Max Loss (최대 예상 손실액) | 소상공인의 공포와 불안감을 자극하며, 문제를 명확히 정의하는 단계. 데이터는 '위협(Threat)'의 형태로 제시되어야 한다. |
| **[Phase 2] 해결책 탐색** | "이 문제를 막으려면 얼마나 비용을 들여야 할까?" | 리스크 감소율 (Risk Mitigation %) | 플랫폼 또는 대안 서비스가 제공할 수 있는 구체적인 방어 메커니즘의 효용성을 제시한다. |
| **[Phase 3] 가치 증명** | "이 해결책에 기꺼이 얼마까지 지불할 의향이 있나?" | WTP (지불 의사) / ROI (투자 대비 수익률) | 플랫폼을 통해 얻게 되는 경제적 이득(기회비용 절감 등)을 제시하고, 이를 바탕으로 **가장 높은 가치**를 산출하여 '구매 동기'로 전환한다. |

## 2. 핵심 지표별 상세 정의 및 계산 로직 (개발 요구사항)

### A. Max Loss (최대 예상 손실액) - [🚨 경고/Red Zone]
Max Loss는 플랫폼 이용자가 직접 입력한 정보와 외부 경제 데이터를 결합하여 산출되는 가장 중요한 '위협' 지표이다.

*   **[Input Data Sources]**:
    1.  $L_{Revenue}$: 월 평균 매출액 (소상공인 입력)
    2.  $D_{Impact}$: 리스크 발생 시 비즈니스 중단 예상 일수 (소상공인 입력/산업 통계 기반 초기값 설정)
    3.  $C_{Operating}$: 월 고정 운영 비용 (임대료, 인건비 등) (소상공인 입력)
*   **[Calculation Formula]**:
    $$MaxLoss = (L_{Revenue} \times D_{Impact}) + C_{Operating}$$
*   **[Output Format]**: 원화 단위 (₩). 대시보드에서 Loss Gauge의 주요 값이 된다.
*   **[Validation Rule]**: $D_{Impact}$가 0일 경우, MaxLoss는 0으로 처리하며 경고 메시지를 출력한다.

### B. WTP (지불 의사) - [💰 가치 측정/Green Zone]
WTP는 리스크 감소를 통해 얻을 수 있는 효용에 기반하여 사용자가 자발적으로 지불할 용의가 있는 금액이다. 이는 단순한 '비용'이 아닌, '투자'로 정의되어야 한다.

*   **[Input Data Sources]**:
    1.  $MaxLoss$: (위 A에서 산출됨)
    2.  $R_{Mitigation}$: 플랫폼 솔루션으로 감소시킬 수 있는 리스크 비율 (%) (예: 30% 리스크 감소 시 $R_{Mitigation}=0.3$)
*   **[Calculation Formula]**:
    $$WTP = MaxLoss \times R_{Mitigation} \times K_{Confidence}$$
    *(여기서 $K_{Confidence}$: 플랫폼 신뢰 계수 (1~1.5). 초기에는 1.2로 설정하여 잠재적 가치를 과소평가하지 않도록 조정)*
*   **[Output Format]**: 원화 단위 (₩).

### C. ROI (투자 대비 수익률) - [📈 최종 판단/Action Zone]
ROI는 WTP를 바탕으로 플랫폼의 유료 기능(Subscription Fee, Consulting Package 등)을 도입했을 때의 경제적 효용성을 나타낸다. 이는 소상공인에게 '돈이 된다'는 명확한 메시지를 준다.

*   **[Input Data Sources]**:
    1.  $WTP$: (위 B에서 산출됨)
    2.  $Cost_{BDS}$: BDS 플랫폼의 월 구독료/서비스 비용 (개발팀 결정값)
*   **[Calculation Formula]**:
    $$ROI = \frac{WTP - Cost_{BDS}}{Cost_{BDS}} \times 100 (\%) $$
*   **[Output Format]**: 백분율 (%)와 절대 금액(₩). **ROI가 0% 이상일 때만 '필수 도입' 상태로 표시되어야 한다.**

## 3. 대시보드 시각화 요구사항 (UI/UX 연동)

| 지표 | 데이터 유형 | 표현 방식 | 색상 코드 및 의미 |
| :--- | :--- | :--- | :--- |
| **Max Loss** | 금액 (₩) | **Loss Gauge / 경고 게이지** | Red (위험): 값이 높을수록 빨갛게 표시. Max Loss가 높은 경우 즉시 시각적 경고 발생. |
| **리스크 감소율** | 백분율 (%) | 막대 그래프 또는 수치 오버레이 | 회색 $\to$ 녹색으로 변화하는 정도를 직관적으로 표현. (개선 폭 강조) |
| **WTP / ROI** | 금액/백분율 | 명확한 CTA 버튼 및 텍스트 강조 | Green (기회): 높은 ROI와 WTP는 성공적인 기회로 포장하고, '클릭'을 유도하는 형태로 배치. |
